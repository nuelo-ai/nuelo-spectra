## Integration Issues

### 1. Hardcoded localhost:8000 in Invite Page (MAJOR)

**Files:** `frontend/src/app/(auth)/invite/[token]/page.tsx` lines 41, 74
**Impact:** Invite registration flow breaks in any non-localhost environment (staging, production, Docker)
**Fix:** Replace `fetch("http://localhost:8000/...")` with `apiClient.get(...)` or relative URL `fetch("/auth/...")`
**Affected:** INVITE-04, INVITE-05

### 2. credit_reset_policy UI is Informational Only (MINOR)

**Files:** Settings form saves `credit_reset_policy` to DB; scheduler reads from `user_classes.yaml`
**Impact:** Admin changing reset policy in Settings UI has no effect on actual scheduler behavior — misleading
**Affected:** SETTINGS-05

### 3. Dead Route — PUT /api/admin/tiers/users/{user_id} (TRIVIAL)

**Files:** `backend/app/routers/admin/tiers.py` line 58
**Impact:** None — frontend uses the users router equivalent. Both call `change_user_tier`
**Affected:** TIER-06 (functionally satisfied via users router)
