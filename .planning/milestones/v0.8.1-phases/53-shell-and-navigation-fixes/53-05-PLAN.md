Key insight for LBAR-02: The button cva base has `p-2` (padding on all sides). In collapsed icon
mode it forces `size-8! p-2!` (important). When `asChild` is used with Slot.Root, the button's
className merges onto the Link/anchor child. Adding `pl-1` to the child produces competing
left-padding values (`p-2` sets pl-8px, then `pl-1` = 4px is also present but Tailwind merges
unpredictably). More critically, `p-2` is `padding: 8px` on all sides — with `pl-1` added to
the child, the left padding becomes Tailwind's winner (last class wins in output CSS order, which
is unpredictable).

The correct fix:
- Remove `className="pl-1"` from the Link and anchor children entirely
- Add `className` to the `SidebarMenuButton` wrapper instead, using `[&>svg]:mx-auto` for
  collapsed centering plus `justify-center` when the sidebar is collapsed
- BUT: The cva already handles icon centering via `items-center` (vertical). The horizontal
  misalignment in collapsed mode is caused by the child's own padding overriding the button's
  padding. Removing `pl-1` from the child lets the button's `p-2` (8px on all sides) be the
  sole padding, and `flex items-center` + `size-8` in collapsed mode will center the icon.
- For expanded mode alignment: the default `p-2` means icon starts at 8px from left edge.
  The Logo button at top uses `size="lg"` which is `h-12` with no special left offset — so
  the icon position in the lg button may differ from default-size nav buttons. If alignment
  with the logo icon is the goal, add `pl-2` on the SidebarMenuButton className (not the child)
  to make it explicit, OR simply remove `pl-1` and let the default `p-2` render cleanly.

Safest approach (removes the bad fix, applies no new override — lets shadcn default render):
1. Remove `className="pl-1"` from Link (line 142) and anchor (line 120) in UnifiedSidebar.tsx
2. If the default p-2 still looks misaligned after removal, the issue is with the logo button
   size="lg" having different padding — that is a separate concern outside LBAR-02 scope.
   The requirement is "menu items aligned with their icons" — not logo alignment.
