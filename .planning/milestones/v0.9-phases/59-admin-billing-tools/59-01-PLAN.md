**2. Add service methods to SubscriptionService** (`backend/app/services/subscription.py`):
Add three new static methods:

`admin_force_set_tier(db, user_id, new_tier, reason, admin_id)`:
- Get current user (from User table), get current tier
- If current tier == new_tier, raise CheckoutValidationError("Already on this tier")
- If new_tier is "on_demand" or "free_trial" and user has active subscription: cancel Stripe sub immediately using `client.v1.subscriptions.cancel(stripe_subscription_id)` (NOT cancel_at_period_end — admin override is immediate). Update local Subscription status to "canceled".
- If new_tier is "standard" or "premium" and user has NO active subscription: just update user_class locally (no Stripe sub creation — admin override does not create billing). Set stripe_synced=True, stripe_action=None.
- If new_tier is "standard" or "premium" and user HAS active subscription on different tier: use `client.v1.subscriptions.update()` to change price, same as change_plan but with `proration_behavior="none"`.
- Update `user.user_class = new_tier`
- Log via `log_admin_action(db, admin_id, action="force_set_tier", target_type="user", target_id=user_id, details={"previous_tier": old_tier, "new_tier": new_tier, "reason": reason})`
- Return ForceSetTierResponse dict

`admin_cancel_subscription(db, user_id, admin_id)`:
- Reuse existing `cancel_subscription` logic but also log admin action
- Log via `log_admin_action(db, admin_id, action="admin_cancel_subscription", target_type="user", target_id=user_id, details={})`

`admin_refund(db, user_id, payment_id, amount_cents, admin_id)`:
- Look up PaymentHistory row by id, verify user_id matches
- If amount_cents is None, refund full amount_cents from the payment
- Validate amount_cents <= original payment's amount_cents
- Call `client.v1.refunds.create(params={"payment_intent": payment.stripe_payment_intent_id, "amount": amount_cents, "reason": "requested_by_customer"})` — if full refund, omit amount param
- Calculate credit deduction: `credits_to_deduct = (refund_amount / original_amount) * original_credit_amount`. Use Decimal arithmetic. Clamp to min 0.
- Deduct from purchased_balance: `credit.purchased_balance = max(Decimal("0"), Decimal(str(credit.purchased_balance)) - Decimal(str(credits_to_deduct)))`
- Record CreditTransaction with transaction_type="refund", balance_pool="purchased", negative amount
- Update PaymentHistory status to "refunded" (full) or "partial_refund" (partial)
- Add new PaymentHistory row with payment_type="refund", negative amount_cents, status="succeeded"
- Log admin action
- Return RefundResponse
