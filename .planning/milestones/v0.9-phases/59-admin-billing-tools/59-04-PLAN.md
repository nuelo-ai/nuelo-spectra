**1. Create useDiscountCodes.ts** (`admin-frontend/src/hooks/useDiscountCodes.ts`):
Follow the exact hook pattern from useUsers.ts. Use `adminApiClient`.

```typescript
"use client";
import { useMutation, useQuery, useQueryClient } from "@tanstack/react-query";
import { adminApiClient } from "@/lib/admin-api-client";
```

Hooks:
- `useDiscountCodes()` -- `useQuery` fetching `GET /api/admin/discount-codes`. queryKey: `["admin", "discount-codes"]`.
- `useCreateDiscountCode()` -- `useMutation` posting to `POST /api/admin/discount-codes` with body `{ code, discount_type, discount_value, max_redemptions?, expires_at? }`. On success: invalidate `["admin", "discount-codes"]`.
- `useUpdateDiscountCode()` -- `useMutation` PUT `/api/admin/discount-codes/${codeId}` with body `{ max_redemptions?, expires_at? }`. On success: invalidate `["admin", "discount-codes"]`.
- `useDeactivateDiscountCode()` -- `useMutation` POST `/api/admin/discount-codes/${codeId}/deactivate`. Invalidate.
- `useDeleteDiscountCode()` -- `useMutation` DELETE `/api/admin/discount-codes/${codeId}`. Invalidate.

Each mutation: check `res.ok`, throw `new Error(body.detail || "Operation failed")` on failure.

**2. Create CreateDiscountDialog.tsx** (`admin-frontend/src/components/discounts/CreateDiscountDialog.tsx`):
Per D-16, D-17, D-18, D-19. This dialog supports BOTH create and edit modes.

Props:
```typescript
interface CreateDiscountDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  initialValues?: {
    id: string;
    code: string;
    discount_type: "percent_off" | "amount_off";
    discount_value: number;
    max_redemptions: number | null;
    expires_at: string | null;
  } | null;  // If provided, dialog is in EDIT mode
}
```

- Determine mode: `const isEditMode = !!initialValues;`
- State: `code`, `discountType`, `discountValue`, `maxRedemptions`, `expiresAt`
  - In create mode: all start empty/default
  - In edit mode: pre-fill from `initialValues` via `useEffect` when `initialValues` changes
- Uses `useCreateDiscountCode()` for create mode, `useUpdateDiscountCode()` for edit mode
- **In edit mode:** code, discount_type, and discount_value fields are DISABLED/READ-ONLY (Stripe coupons are immutable for these). Only `max_redemptions` and `expires_at` are editable.

Dialog layout:
- DialogTitle: `isEditMode ? "Edit Discount Code" : "Create Discount Code"`
- Fields:
  - Code: Input, placeholder "e.g. SUMMER2026", auto-uppercase. **Disabled in edit mode** with text-muted-foreground styling.
  - Type: Select with options "Percentage Off" (value: "percent_off") and "Fixed Amount Off" (value: "amount_off"). **Disabled in edit mode.**
  - Value: Input type="number". If percent_off: label "Percentage (%)", min=1, max=100. If amount_off: label "Amount (cents)", min=1. **Disabled in edit mode.**
  - Max Redemptions: Input type="number", optional (placeholder "Unlimited if empty"), min=1. **Editable in both modes.**
  - Expiry Date: Input type="date", optional (label "Expires At (optional)"). **Editable in both modes.**
- Validation (create mode):
  - code required, alphanumeric + hyphens + underscores only
  - discount_type required
  - discount_value required, > 0 (if percent_off: <= 100)
  - expiry date if set must be in the future
- Validation (edit mode):
  - only max_redemptions and expires_at validated
- DialogFooter: Cancel + primary button (`isEditMode ? "Save Changes" : "Create Discount Code"`)
- On submit:
  - Create mode: call `useCreateDiscountCode()` with full payload. On success: `toast.success("Discount code created")`, close dialog, reset form.
  - Edit mode: call `useUpdateDiscountCode()` with `{ codeId: initialValues.id, max_redemptions, expires_at }`. On success: `toast.success("Discount code updated")`, close dialog.
- On error: `toast.error(e.message)`
