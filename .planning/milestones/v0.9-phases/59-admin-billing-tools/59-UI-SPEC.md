## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (size-1), inline padding |
| sm | 8px | Compact element spacing, gap-2, dense info rows (space-y-2) |
| md | 16px | Default element spacing, card padding (space-y-4, gap-4, mt-4) |
| lg | 24px | Section padding within cards (space-y-6, gap-6) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (py-12) |
| 3xl | 64px | Page-level spacing (not used in admin -- pages are compact) |

Exceptions: Tab content offset uses mt-4 (16px) per established UserDetailTabs pattern. Card internal spacing uses space-y-2 (8px) for dense info rows, matching compact admin layout.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / Label | 14px (text-sm) | 400 (normal) | 1.43 (20px) |
| Heading / Display | 16px (text-base) / 30px (text-3xl) | 600 (semibold) | 1.5 (24px) / 1.2 (36px) |

Note: Admin UI is predominantly text-sm (14px) body with text-base (16px) CardTitle headings. Display size (30px) at weight 600 is used only for stat counters (e.g., credit balance "150.0"). Labels use the same 400 weight as body text. Monospace (font-mono) used for currency amounts, credit values, and Stripe event IDs.

|------|-------|-------|
| Dominant (60%) | var(--background) / oklch(1 0 0) light, #2E3440 dark | Page background, table backgrounds |
| Secondary (30%) | var(--card) / oklch(1 0 0) light, #3B4252 dark | Cards, sidebar, dialog surfaces |

Accent reserved for: primary CTA buttons ("Force Set Tier", "Issue Refund", "Create Discount Code", "Save Settings"), active tab underline, focus rings on inputs, links to Stripe Dashboard.

Semantic colors for inline data:
- Positive amounts: text-emerald-600 / dark:text-emerald-400 (credits added, successful payments)
- Negative amounts: text-red-600 / dark:text-red-400 (credits deducted, refunds)
- Warning/at-risk: text-amber-600 / dark:text-amber-400 (subscription at risk, payment failed)

## Interaction States

### Loading States
- All data tables: 3 Skeleton rows (h-10 w-full) matching existing pattern in UserDetailTabs
- Subscription status card: Skeleton block (h-32)
- Settings form: Skeleton blocks for each field group
- Button pending: Text changes to "Processing..." with disabled state (matches existing adjustCredits pattern)

### Validation
- Force-set tier: Reason field required (min 1 character), new tier must differ from current
- Refund amount: Required, must be > 0, must be <= original payment amount, numeric input only
- Discount code: Code required (alphanumeric + hyphens, auto-uppercase), type required, amount required (percentage: 1-100, fixed: > 0), expiry date optional (must be future if set)
- Billing settings: All numeric fields require positive integers, prices require positive numbers

### Pagination
- Payment history table: Client-side, show last 20 payments initially, "Load More" button pattern (not page numbers)
- Stripe event log: Client-side, show last 50 events initially, "Load More" button pattern
- Discount codes table: No pagination needed (expected < 50 codes)
