   ```python

       api_key_id = getattr(request.state, 'api_key_id', None)
   ```

**2b. Create `backend/app/middleware/api_usage.py`:**

Implement `ApiUsageMiddleware` as a Starlette middleware (using `BaseHTTPMiddleware` from `starlette.middleware.base`):

```python
import time
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

class ApiUsageMiddleware(BaseHTTPMiddleware):
    """Log all /v1/ API requests with timing, user, and status."""

    async def dispatch(self, request: Request, call_next):
        # Only log API v1 requests
        if not request.url.path.startswith("/v1/") and not request.url.path.startswith("/api/v1/"):
            return await call_next(request)

        # Skip health endpoint (noisy)
        if request.url.path.endswith("/health"):
            return await call_next(request)

        start_time = time.monotonic()
        response = await call_next(request)
        elapsed_ms = int((time.monotonic() - start_time) * 1000)

        # Extract user info from request.state (set by get_authenticated_user dependency)
        user_id = getattr(request.state, 'user', None)
        api_key_id = getattr(request.state, 'api_key_id', None)

        # Log asynchronously — don't block the response
        # Use a background task attached to the response
        # Get credits_used from request.state (set by query endpoint)
        credits_used = getattr(request.state, 'credits_used', 0.0)
        error_code = getattr(request.state, 'error_code', None)

        # We need a DB session for logging — use async_session_maker
        # to avoid the request-scoped session (may be closed/committed)
        try:
            from app.database import async_session_maker
            from app.services.api_usage import ApiUsageService

            async with async_session_maker() as db:
                await ApiUsageService.log_request(
                    db=db,
                    user_id=user_id.id if hasattr(user_id, 'id') else user_id,
                    api_key_id=api_key_id,
                    endpoint=request.url.path,
                    method=request.method,
                    status_code=response.status_code,
                    credits_used=credits_used,
                    response_time_ms=elapsed_ms,
                    error_code=error_code,
                )
                await db.commit()
        except Exception:
            # Never let logging failure break the API response
            import logging
            logging.getLogger(__name__).warning(
                "Failed to log API usage for %s %s",
                request.method, request.url.path,
                exc_info=True,
            )

        return response
```

**Important design notes:**
- The middleware uses `async_session_maker()` (check `backend/app/database.py` for the exact name of the session factory) — NOT the request-scoped `db` — because the request session may already be committed or closed by the time we log.
- Wrap logging in try/except — logging failures must NEVER break API responses.
- Store `credits_used` on `request.state` from the query endpoint so the middleware can read it. Update query.py to set `request.state.credits_used = float(cost)` after successful deduction.
- For user_id: The `get_authenticated_user` dependency sets the user on `request.state` implicitly via FastAPI's dependency injection. However, `BaseHTTPMiddleware` runs BEFORE dependencies. So the user_id may not be available.

**Alternative approach if request.state.user is not available:** Instead of middleware, use a dependency-based approach where each endpoint calls the logging function explicitly. OR use a simpler approach: add structured Python logging (not DB logging) in the middleware, and do DB logging explicitly in endpoints that need credit tracking.

**Recommended hybrid approach:**
1. Middleware: Structured Python logging (using `logging` module) for ALL /v1/ requests — captures timing, endpoint, method, status_code. This satisfies APIINFRA-04 (structured request/error logs).
2. Explicit DB logging: In the query endpoint (and any credit-consuming endpoint), call `ApiUsageService.log_request()` directly after the operation completes. This satisfies APISEC-04 (per-user, per-key usage logging with credits).
