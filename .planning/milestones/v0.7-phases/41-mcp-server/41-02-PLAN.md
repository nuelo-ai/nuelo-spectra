**main.py changes:**

1. **Modify the trailing slash middleware** to exclude `/mcp` paths. The MCP Streamable HTTP transport may rely on specific path patterns, and stripping trailing slashes can break it:
```python
@app.middleware("http")
async def strip_trailing_slash(request: Request, call_next):
    path = request.url.path
    if path != "/" and path.endswith("/") and not path.startswith("/mcp"):
        request.scope["path"] = path.rstrip("/")
    return await call_next(request)
```

2. **Add MCP mounting inside the existing `if mode in ("api", "dev"):` block**, after the API v1 router inclusion:
```python
# API v1 routes (api and dev modes) — external API access and API key management
if mode in ("api", "dev"):
    from app.routers.api_v1 import api_v1_router
    from app.middleware.api_usage import ApiUsageMiddleware

    app.include_router(api_v1_router)
    app.include_router(api_v1_router, prefix="/api")
    app.add_middleware(ApiUsageMiddleware)

    # MCP server (Streamable HTTP at /mcp/)
    from app.mcp_server import create_mcp_app
    mcp_app = create_mcp_app()
    app.mount("/mcp", mcp_app)
```

3. **Lifespan coordination with combine_lifespans:**

This is the most critical change. The existing `lifespan` function manages checkpointer, scheduler, and LLM validation. The MCP app has its own lifespan for session manager initialization. These must be merged.

The approach depends on how FastAPI's `app.mount()` handles lifespans. There are two strategies:

**Strategy A (preferred — try first):** Use `combine_lifespans` to wrap the existing lifespan:
```python
from fastmcp.utilities.lifespan import combine_lifespans
```

Modify the FastAPI app creation to conditionally use the combined lifespan:
- Before the `app = FastAPI(...)` line, check if MCP will be mounted (mode check)
- If yes, create the mcp_app early, extract its lifespan, and combine
- If no, use the existing lifespan as-is

This requires restructuring the mode check to happen BEFORE `app = FastAPI(...)`. The current code creates the app first, then mounts conditionally. To combine lifespans, we need:

```python
# Determine mode early (settings already loaded above)
mode = settings.spectra_mode

# Build the app lifespan — conditionally combine with MCP lifespan
app_lifespan = lifespan
if mode in ("api", "dev"):
    from fastmcp.utilities.lifespan import combine_lifespans
    from app.mcp_server import create_mcp_app
    _mcp_app = create_mcp_app()
    app_lifespan = combine_lifespans(lifespan, _mcp_app.lifespan)

app = FastAPI(
    title="Spectra API",
    version="0.1.0",
    description="AI-powered data analytics platform backend",
    lifespan=app_lifespan,
    redirect_slashes=False,
)

# ... (exception handler, middleware, router setup) ...

# Later in the api/dev mode block:
if mode in ("api", "dev"):
    # ... existing api_v1_router and middleware ...
    app.mount("/mcp", _mcp_app)
```

Move the `mode = settings.spectra_mode` line and its validation check ABOVE the `app = FastAPI(...)` line. Currently mode is set on line 331 after app creation on line 272. The mode variable and validation need to move up.

**Strategy B (fallback):** If `combine_lifespans` does not work as expected (import error, API mismatch), mount the MCP app WITHOUT lifespan merging. Since we use `stateless_http=True`, the MCP session manager may not need initialization. Add a comment noting the fallback and that lifespan coordination should be verified.

**Important:** Do NOT modify the `lifespan` function itself — only how it is passed to `FastAPI()`. The existing lifespan logic (checkpointer, scheduler, LLM validation) must remain unchanged.
