# Milestone v0.10: Streamline Pricing Configuration

**Status:** ✅ SHIPPED 2026-04-26
**Phases:** 60-61
**Total Plans:** 7

## Overview

Config-driven pricing and admin pricing management — subscription pricing and credit packages are fully defined in config files, automatically provisioned on startup, and manageable from the admin portal with view/edit/reset workflows.

## Phases

### Phase 60: Config-Driven Pricing & Startup Sync

**Goal**: Subscription pricing and credit packages are fully defined in config files and automatically provisioned in both the database and Stripe on first startup — zero manual setup required
**Depends on**: Phase 59 (v0.9 Monetization complete)
**Plans**: 3 plans

Plans:
- [x] 60-01: Extend user_classes.yaml with pricing fields and add get_credit_packages() loader
- [x] 60-02: Create pricing_sync.py service with seeding, Stripe sync, readiness check, and unit tests
- [x] 60-03: Wire pricing sync into lifespan() and add monetization toggle guard

**Requirements**: SUB-01, SUB-02, SUB-03, SUB-04, PKG-01, PKG-02, PKG-03, SAFE-01, SAFE-02
**Success Criteria**:
  1. user_classes.yaml includes `has_plan` and `price_cents` fields for subscription tiers
  2. A config file defines default credit packages
  3. On first startup, subscription pricing and credit packages seeded from config
  4. Stripe Products/Prices auto-created for missing items
  5. Existing admin-customized Stripe Price IDs never overwritten

### Phase 61: Admin Pricing Management UI

**Goal**: Admin can view, edit, and reset both subscription pricing and credit packages from the admin portal
**Depends on**: Phase 60
**Plans**: 4 plans

Plans:
- [x] 61-01: Backend API — schemas, billing-settings extensions, credit-packages router
- [x] 61-02: Config features + dynamic /subscriptions/plans endpoint
- [x] 61-03: Admin frontend — PasswordConfirmDialog + extended hooks
- [x] 61-04: Admin frontend — billing-settings page refactor with view/edit/reset modals

**Requirements**: SUB-05, SUB-06, SUB-07, PKG-04, PKG-05, PKG-06, UI-01, UI-02
**Success Criteria**:
  1. Admin can view subscription pricing with config defaults side by side
  2. Admin can edit subscription pricing (auto-creates new Stripe Prices)
  3. Admin can reset subscription pricing to config defaults
  4. Admin can view all credit packages from database
  5. Admin can edit credit packages
  6. Admin can reset credit packages to config defaults
  7. Plan Selection page dynamically renders from config
  8. Billing page displays credit packages from database

## Milestone Summary

**Key Decisions:**
- Config-driven pricing: user_classes.yaml gets has_plan + price_cents fields
- Startup sync fills gaps only, never overwrites admin-customized Stripe Price IDs
- Lazy import SubscriptionService inside Stripe functions to avoid circular imports
- Pricing sync uses dedicated async session to avoid scope conflicts
- Features lists added only to plan-displayed tiers (on_demand, standard, premium)
- Password verification required on all pricing mutations

**Issues Resolved:**
- Zero manual Stripe configuration needed on first deployment
- Hardcoded plan features replaced with config-driven dynamic rendering
- No admin UI for credit package management (now available)

**Issues Deferred:**
- Admin creating new credit packages from scratch (config-defined only)
- Admin deleting credit packages (deactivate via is_active flag instead)

**Technical Debt Incurred:**
- None identified
