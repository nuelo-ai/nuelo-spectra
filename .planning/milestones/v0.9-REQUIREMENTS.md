# Requirements: Spectra Monetization

**Defined:** 2026-03-18
**Core Value:** Accurate data analysis through correct, safe Python code generation

**Edition:** All requirements in this milestone are commercial-only (PRO). Community/commercial separation is a future milestone.

## Milestone Requirements

Requirements for monetization milestone. Each maps to roadmap phases.

### Tier & Credit

- [x] **TIER-01**: System supports 4 consumer tiers: Free Trial (temporary), On Demand, Basic, Premium
- [x] **TIER-02**: Existing `free` tier dropped, `standard` renamed to `basic`, `on_demand` tier added to user_classes.yaml
- [x] **TIER-03**: Existing users on `free` tier migrated to appropriate tier with admin-configurable migration path
- [x] **TIER-04**: User credit balance split into subscription credits and purchased credits with independent tracking
- [x] **TIER-05**: Credit deduction consumes subscription credits first, purchased credits second
- [x] **TIER-06**: Purchased credits persist across billing cycles (never reset)
- [x] **TIER-07**: Subscription credits reset on each billing cycle via Stripe `invoice.paid` webhook
- [x] **TIER-08**: APScheduler credit reset skips subscription-tier users (Stripe handles their resets)

### Trial

- [x] **TRIAL-01**: New user registration assigns Free Trial state with configurable credit amount
- [x] **TRIAL-02**: Trial duration configurable via `trial_duration_days` platform setting (default 14 days)
- [x] **TRIAL-03**: Trial expiration enforced in auth middleware (covers web, API, and MCP paths)
- [x] **TRIAL-04**: Trial banner on dashboard showing days remaining and credit balance
- [x] **TRIAL-05**: Trial banner becomes urgent (amber) at 3 days remaining or 10 credits remaining
- [x] **TRIAL-06**: Full blocking overlay when trial expires — no access until plan selected
- [x] **TRIAL-07**: Trial credits not eligible for top-up purchase

### Stripe Integration

- [x] **STRIPE-01**: Stripe Python SDK integrated in backend with API key configuration via environment variables
- [x] **STRIPE-02**: Stripe Checkout Session creates subscription for Basic/Premium plans
- [x] **STRIPE-03**: Stripe Checkout Session creates one-time payment for credit top-up packages
- [x] **STRIPE-04**: Stripe webhook endpoint with signature verification and raw body parsing
- [x] **STRIPE-05**: Webhook idempotency via `stripe_events` deduplication table (no duplicate processing)
- [x] **STRIPE-06**: `checkout.session.completed` webhook activates subscription or adds purchased credits
- [x] **STRIPE-07**: `invoice.paid` webhook handles monthly renewal and credit reset
- [x] **STRIPE-08**: `invoice.payment_failed` webhook marks subscription at risk and notifies user
- [x] **STRIPE-09**: `customer.subscription.updated` webhook handles plan changes (upgrade/downgrade)
- [x] **STRIPE-10**: `customer.subscription.deleted` webhook downgrades to On Demand at cycle end

### Subscription

- [ ] **SUB-01**: User can subscribe to Basic or Premium plan via Plan Selection page
- [ ] **SUB-02**: User can upgrade from Basic to Premium (immediate, remaining credits kept, new allocation prorated)
- [ ] **SUB-03**: User can downgrade from Premium to Basic (takes effect at end of billing cycle)
- [ ] **SUB-04**: User can cancel subscription (takes effect at end of billing cycle, purchased credits preserved)
- [ ] **SUB-05**: User can select On Demand plan (no subscription, pay per top-up only)
- [x] **SUB-06**: Subscription state stored locally with Stripe as source of truth (Subscription table)

### Credit Top-Up

- [ ] **TOPUP-01**: Predefined credit packages available for purchase (e.g., 50/200/500 credits)
- [ ] **TOPUP-02**: Credit packages configurable via CreditPackage table (amount, price, active status)
- [ ] **TOPUP-03**: User can purchase credits via Stripe Checkout from Manage Plan page
- [ ] **TOPUP-04**: Purchased credits added to balance on successful payment (via webhook)
- [ ] **TOPUP-05**: Free Trial users not eligible for credit top-up

### Billing UI

- [ ] **BILL-01**: Plan Selection page (`/settings/plan`) showing On Demand, Basic, Premium with pricing
- [ ] **BILL-02**: Plan Selection shows current plan/trial status
- [ ] **BILL-03**: Manage Plan page (`/settings/billing`) showing subscription details, credits, billing history
- [ ] **BILL-04**: Stripe Embedded Checkout integration for payment flow (not redirect)
- [ ] **BILL-05**: Billing history display on Manage Plan page (payments, renewals, top-ups, refunds)
- [ ] **BILL-06**: Settings navigation updated with "Plan & Billing" section
- [ ] **BILL-07**: `@stripe/stripe-js` and `@stripe/react-stripe-js` integrated in public frontend

### Data Model

- [x] **DATA-01**: Subscription table (user_id, stripe_subscription_id, stripe_customer_id, plan_tier, status, billing period, cancel_at_period_end)
- [x] **DATA-02**: PaymentHistory table (user_id, stripe_payment_intent_id, amount, type, credit_amount, status)
- [x] **DATA-03**: CreditPackage table (name, credit_amount, price_cents, stripe_price_id, is_active)
- [x] **DATA-04**: StripeEvent table (stripe_event_id, event_type, processed_at) for webhook idempotency
- [x] **DATA-05**: `trial_expires_at` field on User model (set to registration + configurable days)
- [x] **DATA-06**: `purchased_balance` field on UserCredit model for dual-balance tracking

### Admin Billing

- [ ] **ADMIN-01**: Admin can view user subscription status in user detail page
- [ ] **ADMIN-02**: Admin can view payment history per user
- [ ] **ADMIN-03**: Admin can force-set user tier regardless of Stripe state (manual override with reason logging)
- [ ] **ADMIN-04**: Admin can cancel user subscription on behalf of user
- [ ] **ADMIN-05**: Admin can view full Stripe billing event log per user
- [ ] **ADMIN-06**: Admin can issue full or partial refund for a specific payment (triggers Stripe refund API)
- [ ] **ADMIN-07**: Admin billing platform settings: trial_duration_days, credit prices, subscription prices, subscription credit amounts
- [ ] **ADMIN-08**: Admin can create and manage discount codes (percentage or fixed amount off subscriptions/top-ups)
- [ ] **ADMIN-09**: Admin can deactivate discount codes
- [ ] **ADMIN-10**: Discount codes integrate with Stripe Coupons/Promotion Codes API

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Stripe Customer Portal | Cannot display Spectra-specific credit balances, tier info, or top-up packages — build custom billing UI instead |
| Freemium tier (ongoing free access) | Business decision: no ongoing free access after trial. Users must choose a paid plan. |
| Real-time Stripe sync | Polling Stripe for state is fragile and rate-limited. Webhooks are the source of truth. |
| Custom credit amounts | Predefined packages only for v1. Custom amounts add UI and pricing complexity. |
| Annual billing | Defer to post-launch. Monthly only for v1. |
| Multi-currency | USD only for v1. Single currency simplifies Stripe setup. |
| Stripe Billing Credits | Manage credits entirely in Spectra DB. Stripe Billing Credits adds complexity without clear benefit at this stage. |
