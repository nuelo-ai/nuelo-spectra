**layout.tsx changes:**

1. Import ThemeProvider from next-themes:


2. Add `suppressHydrationWarning` to the `<html>` tag to prevent Next.js hydration errors with theme:
   ```tsx
   <html lang="en" suppressHydrationWarning>
   ```

3. Wrap `<Providers>` with `<ThemeProvider>` inside the `<body>` tag:
   ```tsx
   <body className={`${inter.variable} font-sans antialiased`}>
     <ThemeProvider
       attribute="class"
       defaultTheme="system"
       enableSystem
       disableTransitionOnChange
     >
       <Providers>{children}</Providers>
     </ThemeProvider>
   </body>
   ```

**Configuration rationale:**
- `attribute="class"` — adds `.dark` class to `<html>`, which matches the existing `.dark` CSS variables already defined in globals.css (lines 139-171)
- `defaultTheme="system"` — respects OS preference on first visit (per user decision)
- `enableSystem` — allows detecting OS theme changes in real-time
- `disableTransitionOnChange` — prevents jarring CSS transition flashes during toggle (per research recommendation)
- localStorage is used automatically by next-themes for persistence (THEME-02)

Note: The existing globals.css already has complete `:root` (light) and `.dark` (dark) CSS variable definitions including sidebar colors, chart colors, and skeleton loading states. The sonner.tsx Toaster component already imports `useTheme` from next-themes and is theme-aware. No CSS changes needed.

2. Inside the `UserSection` component function, add theme logic:
   ```typescript
   const { theme, setTheme } = useTheme();
   const [mounted, setMounted] = useState(false);

   useEffect(() => {
     setMounted(true);
   }, []);
   ```
   The `mounted` check prevents hydration mismatch — theme-dependent icons should only render after client mount.

3. Add a theme toggle DropdownMenuItem between the Settings item and the Logout separator. Place it right after the Settings `<DropdownMenuItem>` and before the `<DropdownMenuSeparator />` that precedes Logout:
   ```tsx
   {mounted && (
     <DropdownMenuItem
       onClick={() => setTheme(theme === "dark" ? "light" : "dark")}
       className="cursor-pointer"
     >
       {theme === "dark" ? (
         <Sun className="mr-2 h-4 w-4" />
       ) : (
         <Moon className="mr-2 h-4 w-4" />
       )}
       {theme === "dark" ? "Light mode" : "Dark mode"}
     </DropdownMenuItem>
   )}
   ```

Per user decision: toggle is inside user profile/menu dropdown (not in sidebar or navbar directly). Two options only — Light/Dark toggle. No explicit "System" option. System preference is used only for initial default (handled by ThemeProvider's `defaultTheme="system"`).

The `mounted` guard ensures the icon/text don't cause SSR hydration mismatch. Before mount, the menu item simply doesn't render (acceptable since the dropdown only opens on user interaction, which happens after mount).
