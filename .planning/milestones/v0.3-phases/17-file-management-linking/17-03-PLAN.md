  In FileLinkingDropdown, the upload dialog watches for newly created files and auto-links:
  - Before opening dialog, snapshot current file IDs: `const prevFileIds = new Set(files.map(f => f.id))`
  - After dialog closes (onUploadComplete), refetch files, find the new file ID(s)
  - Auto-link any new file(s) to the session via useLinkFile

  This works because uploads are sequential (single file at a time) and the files query is always fresh after invalidation.

  Implementation:
  ```typescript
  const { data: files } = useFiles();
  const prevFileIdsRef = useRef<Set<string>>(new Set());

  const handleOpenUploadDialog = () => {
    prevFileIdsRef.current = new Set(files?.map(f => f.id) || []);
    setUploadDialogOpen(true);
  };

  const handleUploadComplete = async () => {
    setUploadDialogOpen(false);
    // Wait for files query to refresh
    await queryClient.invalidateQueries({ queryKey: ["files"] });
    const updatedFiles = queryClient.getQueryData<FileListItem[]>(["files"]);
    if (updatedFiles) {
      const newFiles = updatedFiles.filter(f => !prevFileIdsRef.current.has(f.id));
      for (const newFile of newFiles) {
        linkFile({ sessionId, fileId: newFile.id });
      }
    }
  };
  ```
