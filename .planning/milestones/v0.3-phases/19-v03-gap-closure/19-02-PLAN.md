**GAP 7 fix — Bulk delete in MyFilesTable:**

In MyFilesTable.tsx, fix `handleBulkDeleteConfirm` (lines 125-142). The bug: `getRowId` uses `row.id` (UUID), so `rowSelection` keys ARE UUIDs, not numeric indices. The line `files[Number(idx)]?.id` converts UUID to NaN.

Fix: Replace the mapping logic with direct UUID usage:
```typescript
const handleBulkDeleteConfirm = async () => {
  const selectedIds = Object.keys(rowSelection).filter(
    (key) => rowSelection[key]
  );
  // selectedIds ARE already file UUIDs (via getRowId: row => row.id)
  if (selectedIds.length === 0) return;
  try {
    await bulkDeleteFiles.mutateAsync(selectedIds);
    toast.success(`${selectedIds.length} file(s) deleted`);
    setRowSelection({});
  } catch {
    toast.error("Failed to delete some files");
  }
  setBulkDeleteOpen(false);
};
```
Remove the `const fileIds = selectedIds.map(...)` intermediate step entirely. Use `selectedIds` directly as the argument to `bulkDeleteFiles.mutateAsync`.
