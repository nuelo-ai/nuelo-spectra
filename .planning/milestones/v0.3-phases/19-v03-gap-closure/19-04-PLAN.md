<task type="auto">
  <name>Task 1: Fix drag-drop analyzing hang with setTimeout(0) deferral</name>
  <files>frontend/src/components/file/FileUploadZone.tsx</files>
  <action>
Fix the React Strict Mode double-mount issue that disconnects TanStack Query's MutationObserver from the in-flight mutation, causing inline onSuccess to never fire.

Root cause: The `useEffect` at line 117-122 calls `onDrop(initialFiles)` during initial mount. React Strict Mode unmounts then remounts the component. During unmount, `useSyncExternalStore` unsubscribes the listener from the MutationObserver, which triggers `onUnsubscribe()` that removes the observer from the mutation's notification list. The `initialProcessed` ref guard (set to `true` on first mount) prevents retry on remount. Result: mutation completes but `onSuccess` never fires, so `uploadedFileId` is never set, summary polling never starts, UI stays stuck at "Uploading...".

Fix: In the `useEffect` that handles `initialFiles` (lines 117-122), wrap the `onDrop(initialFiles)` call in `setTimeout(() => { ... }, 0)`. This defers execution past the Strict Mode unmount-remount cycle. By the time setTimeout fires, the component is stably mounted with the observer properly subscribed.

Also move the `initialProcessed.current = true` inside the setTimeout callback so the guard is only set when the deferred call actually executes. Keep the outer condition check (`initialFiles && initialFiles.length > 0 && !initialProcessed.current && uploadStage === "idle"`) as-is for the guard.

Add a cleanup function to the useEffect that clears the timeout if the component unmounts before the setTimeout fires (prevents calling onDrop on an unmounted component).

The resulting code should look like:

```tsx
useEffect(() => {
  if (initialFiles && initialFiles.length > 0 && !initialProcessed.current && uploadStage === "idle") {
    const timer = setTimeout(() => {
      initialProcessed.current = true;
      onDrop(initialFiles);
    }, 0);
    return () => clearTimeout(timer);
  }
}, [initialFiles, onDrop, uploadStage]);
```

This matches the exact pattern already used in ChatInterface.tsx line 127-128 (setTimeout(0) to survive React Strict Mode double-mount).

Do NOT change any other code in FileUploadZone.tsx. Do NOT modify the onDrop function, the uploadFile mutation call, or the onSuccess callback.
