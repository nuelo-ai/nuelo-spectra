**backend/app/routers/collections.py** -- Single router file with all nested routes (~250-300 lines).

Router setup:
```python
router = APIRouter(prefix="/collections", tags=["Collections"])
```

All endpoints use `WorkspaceUser` (not `CurrentUser`) for tier-based access gating per locked decision.

**Collection CRUD endpoints:**

1. `POST /collections` (status 201) -> CollectionResponse
   - Body: CollectionCreate (name, description)
   - Check max_active_collections: call `get_class_config(current_user.user_class)`, get max_active_collections, if not -1 then count via `CollectionService.count_user_collections()` and raise 403 if at limit with message "Collection limit reached ({N} active collections)"
   - Call `CollectionService.create_collection()`
   - Return CollectionDetailResponse with file_count=0, signal_count=0, report_count=0

2. `GET /collections` -> list[CollectionListItem]
   - Call `CollectionService.list_user_collections()`
   - Build CollectionListItem from each dict (collection obj + file_count + signal_count)

3. `GET /collections/{collection_id}` -> CollectionDetailResponse
   - Call `CollectionService.get_collection_detail()`
   - 404 if not found
   - Build CollectionDetailResponse from dict

4. `PATCH /collections/{collection_id}` -> CollectionDetailResponse
   - Body: CollectionUpdate
   - Call `CollectionService.update_collection()`
   - 404 if not found
   - Return updated detail (re-fetch via get_collection_detail for counts)

**File endpoints (nested under /collections/{collection_id}/files):**

5. `POST /collections/{collection_id}/files` (status 201) -> CollectionFileResponse
   - Accept `UploadFile` (same as files router)
   - Verify collection ownership via `CollectionService.get_user_collection()`
   - Call `FileService.upload_file()` to handle disk write + File row creation
   - Call `CollectionService.add_file_to_collection()` to create junction row
   - Trigger background onboarding: `asyncio.create_task(_run_onboarding_background(file.id, user.id, context))` -- same pattern as files.py
   - Return CollectionFileResponse with file info

6. `POST /collections/{collection_id}/files/link` (status 201) -> CollectionFileResponse
   - Body: FileLinkRequest (file_id)
   - Verify collection ownership
   - Verify file ownership via `FileService.get_user_file()`
   - Check for duplicate via `CollectionService.get_collection_file()`, raise 409 if exists
   - Call `CollectionService.add_file_to_collection()`
   - Return CollectionFileResponse

7. `GET /collections/{collection_id}/files` -> list[CollectionFileResponse]
   - Call `CollectionService.list_collection_files()`
   - Build response with file.filename, file.file_size, file.data_summary from eagerly-loaded File relationship

8. `DELETE /collections/{collection_id}/files/{file_id}` (status 204)
   - Call `CollectionService.remove_file_from_collection()` -- deletes CollectionFile row only, NOT the File
   - 404 if not found

**Report endpoints (nested under /collections/{collection_id}/reports):**

9. `GET /collections/{collection_id}/reports` -> list[ReportListItem]
   - Call `CollectionService.list_collection_reports()`
   - Return metadata only (no markdown content in list per locked decision)

10. `GET /collections/{collection_id}/reports/{report_id}` -> ReportDetailResponse
    - Call `CollectionService.get_report()`
    - 404 if not found
    - Return full content + signal_count (count from same pulse_run_id)

11. `GET /collections/{collection_id}/reports/{report_id}/download` -> Response
    - Call `CollectionService.get_report_for_download()`
    - 404 if not found
    - Return Response with media_type="text/markdown", Content-Disposition: attachment header
    - Filename: `{report.title.replace(' ', '_')}.md`

Import the background onboarding pattern from files.py. Use `async_session_maker` from `app.database` for the background task session. Import `agent_service` from `app.services.agent`.
