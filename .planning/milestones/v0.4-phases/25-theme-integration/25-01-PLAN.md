  <name>Task 2: Integrate theme detection into ChartRenderer</name>
  <files>frontend/src/components/chart/ChartRenderer.tsx</files>
  <action>
    Modify ChartRenderer.tsx to detect the current theme and apply theme-specific Plotly layout config.

    **Add imports:**
    - `import { useTheme } from 'next-themes';`
    - `import { getChartThemeConfig, getPieChartOverrides } from '@/lib/chartTheme';`

    **Inside the ChartRenderer component (after existing refs/state):**
    - Call `const { resolvedTheme } = useTheme();`
    - Note: `resolvedTheme` from next-themes already resolves 'system' to actual 'light'/'dark'. No need to manually check systemTheme.
    - Derive: `const isDark = resolvedTheme === 'dark';`

    **In the useEffect (where Plotly.react is called):**

    After parsing chartData and extracting traces, before creating the layout object:

    1. Get theme config: `const themeConfig = getChartThemeConfig(isDark ? 'dark' : 'light');`

    2. Apply pie chart overrides to traces:
       ```
       const themedTraces = traces.map(trace => {
         const traceAny = trace as any;
         if (traceAny.type === 'pie') {
           const pieOverrides = getPieChartOverrides(isDark ? 'dark' : 'light');
           return { ...trace, ...pieOverrides };
         }
         return trace;
       });
       ```

    3. Merge layout with theme config. Theme config OVERRIDES backend layout for colors/fonts, but backend layout provides titles and axis labels. Use a deep merge for xaxis, yaxis, and title to preserve backend text content while applying theme font colors:
       ```
       const layout: Partial<Plotly.Layout> = {
         ...chartData.layout,
         ...themeConfig,
         xaxis: {
           ...chartData.layout?.xaxis,
           ...themeConfig.xaxis,
           title: {
             ...chartData.layout?.xaxis?.title,
             font: themeConfig.xaxis?.title?.font,
           },
         },
         yaxis: {
           ...chartData.layout?.yaxis,
           ...themeConfig.yaxis,
           title: {
             ...chartData.layout?.yaxis?.title,
             font: themeConfig.yaxis?.title?.font,
           },
         },
         title: {
           ...chartData.layout?.title,
           font: themeConfig.title?.font,
         },
         autosize: true,
         height: chartHeight,
         margin: { l: 50, r: 30, t: 40, b: 80 },
       };
       ```
       The increased bottom margin (50 -> 80) accommodates the horizontal legend positioned below the chart.

    4. Use `themedTraces` instead of `traces` in the Plotly.react call.

    **CRITICAL: Do NOT add `isDark` to the useEffect dependency array.**
    Keep the dependency array as `[data, height]` (unchanged from current code).

    Per user decision: "Only newly generated charts pick up the current theme -- already-rendered charts in the conversation keep their original theme styling."

    How this works:
    - When a chart is first mounted (new DataCard), useEffect fires with the CURRENT theme -> chart renders with current theme colors.
    - When user toggles theme, `isDark` changes, React re-renders the component, BUT useEffect does NOT re-fire (isDark is not in deps) -> chart keeps its original theme.
    - The next NEW chart generated after the toggle WILL use the new theme because it's a fresh component mount.

    This is intentional and correct. The ESLint exhaustive-deps rule may warn about `isDark` not being in deps. Suppress it with an inline comment:
    ```
    // eslint-disable-next-line react-hooks/exhaustive-deps
    }, [data, height]);
    ```
    Add a code comment explaining why: "Theme is captured on mount only. Per user decision, already-rendered charts keep their original theme when user toggles."

    **Do NOT change:**
    - The ResizeObserver logic
    - The cleanup logic
    - The error handling
    - The forwardRef/useImperativeHandle pattern
    - The calculateChartHeight function
    - The component's className or outer div structure
  </action>
