**3. Label Formatting Rules** (NEW section, add after code patterns):
```
**Label Formatting Rules:**
1. Chart title: Natural language summary of what the chart shows (max 8 words).
   - Good: "Revenue by Region", "Monthly Sales Trend"
   - Bad: "col_revenue vs col_region", "data_analysis_result"

2. Axis labels: Human-readable with units when inferable.
   - "revenue", "sales", "price", "cost", "amount" -> add " ($)"
   - "count", "quantity", "total" -> add " (Count)"
   - "temp", "temperature" -> add context-appropriate unit
   - "percent", "rate", "ratio" -> add " (%)"
   - For unknown units, use clean descriptive name without units.

3. Always use the Plotly Express `labels` parameter to rename raw column names:
   - Convert snake_case to Title Case: "total_revenue" -> "Total Revenue"
   - Remove prefixes: "col_revenue_2024" -> "Revenue (2024)"
   - Abbreviate long names: "average_monthly_customer_retention_rate" -> "Avg. Retention Rate (%)"

4. Pie/donut charts: Use clear category names. Plotly auto-generates legend.
```

**4. Common Pitfalls** (NEW section, add after label rules):
```
**Avoid These Mistakes:**
- Do NOT use bar chart for single numeric distributions (use histogram instead).
- Do NOT use pie chart for more than 8 categories (use horizontal bar instead).
- Do NOT generate a chart title longer than 8 words.
- For horizontal bar charts with many categories, use: fig.update_layout(yaxis={'categoryorder': 'total ascending'})
- For line charts, always sort data by date/time column ascending before plotting.
```
