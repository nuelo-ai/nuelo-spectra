**ChartExportButtons.tsx -- Create new component:**

Create `frontend/src/components/chart/ChartExportButtons.tsx` with:

```typescript
'use client';

import { useState } from 'react';
import { Download } from 'lucide-react';
import { Button } from '@/components/ui/button';
import Plotly from 'plotly.js-dist-min';

interface ChartExportButtonsProps {
  /** Function that returns the chart DOM element (from ChartRendererHandle.getElement) */
  getChartElement: () => HTMLDivElement | null;
  /** Base filename for downloads (without extension) */
  filename?: string;
}
```

The component renders two Button elements (matching the existing DownloadButtons.tsx style with `variant="outline"` and `size="sm"`):
- "PNG" button with Download icon (h-4 w-4) -- calls `handleExport('png')`
- "SVG" button with Download icon (h-4 w-4) -- calls `handleExport('svg')`

The `handleExport` function:
1. Gets the chart element via `getChartElement()`. If null, return early.
2. Sets a loading state for the clicked format (to disable button during export).
3. Calls `await Plotly.downloadImage(element, { format, width: 1200, height: 800, filename: sanitizedFilename })`.
   - `sanitizedFilename` = `filename` prop or `'spectra-chart'`, WITHOUT extension (Plotly adds it).
4. On error, console.error and reset loading state.
5. On success, reset loading state.

Wrap the two buttons in a `<div className="flex gap-2">`. Add a label span: `<span className="text-xs text-muted-foreground">Export:</span>`.

Use `useState<'png' | 'svg' | null>(null)` for tracking which button is currently exporting (show "..." or disable that button while exporting). Both buttons disabled while any export is in progress.
