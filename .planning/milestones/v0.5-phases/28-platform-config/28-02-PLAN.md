**1. Create `backend/app/services/admin/tiers.py`:**

Service function for tier change with atomic credit reset:

```python
async def change_user_tier(
    db: AsyncSession, user_id: UUID, new_class: str, admin_id: UUID
) -> dict:
```

- Import `get_class_config` from `app.services.user_class` and `CreditService` from `app.services.credit`
- Validate `new_class` exists via `get_class_config(new_class)` -- if None, raise ValueError("Unknown user class: {new_class}")
- Select user with `with_for_update()`: `select(User).where(User.id == user_id).with_for_update()`
- If not found, raise ValueError("User not found")
- Store `old_class = user_row.user_class`
- Set `user_row.user_class = new_class`
- Get new tier allocation: `class_config.get("credits", 0)`
- Select UserCredit with `with_for_update()`: `select(UserCredit).where(UserCredit.user_id == user_id).with_for_update()`
- If credit row exists, call `CreditService.execute_reset(db, credit, new_allocation, transaction_type="tier_change")`
- If no credit row exists, create one: `UserCredit(user_id=user_id, balance=Decimal(str(new_allocation)))`
- Handle unlimited tiers: if `class_config.get("reset_policy") == "unlimited"`, set balance to `Decimal("-1")` sentinel instead
- Return `{"old_class": old_class, "new_class": new_class, "new_balance": float(new_allocation)}`

**3. Add public signup-status endpoint to `backend/app/routers/auth.py`:**

Add BEFORE the authenticated endpoints (right after login endpoint is fine):

```python
@router.get("/signup-status")
async def get_signup_status(
    db: Annotated[AsyncSession, Depends(get_db)],
) -> dict:
    """Public endpoint: check if signup is allowed. No auth required."""
    from app.services import platform_settings
    allowed = await platform_settings.get(db, "allow_public_signup")
    return {"signup_allowed": allowed}
```

- Import `platform_settings` from `app.services`
- Check: `allow_signup = await platform_settings.get(db, "allow_public_signup")`
- If `not allow_signup`:
  - Check if `signup_data` has `invite_token` field (see schema change below)
  - If no `invite_token` or empty: raise `HTTPException(status_code=403, detail="Public registration is currently disabled. An invitation is required.")`
  - If `invite_token` provided: validate it against the Invitation model:
    - Hash the token: `token_hash = hashlib.sha256(invite_token.encode()).hexdigest()`

    - If not found: raise `HTTPException(status_code=403, detail="Invalid or expired invitation token.")`
    - If found: mark as accepted: `invitation.status = "accepted"`, `invitation.accepted_at = datetime.now(timezone.utc)`
