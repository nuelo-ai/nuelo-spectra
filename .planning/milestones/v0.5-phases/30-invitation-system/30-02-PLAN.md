    **2. Add endpoints to `backend/app/routers/auth.py`:**

    **GET `/invite-validate`** (public, no auth required):
    - Query parameter: `token: str`
    - Hash the token with SHA-256: `hashlib.sha256(token.encode()).hexdigest()`
    - Query Invitation where token_hash matches, status="pending", expires_at > now
    - If not found: raise HTTPException(400, "This invite has expired. Contact your administrator for a new one.") -- per locked decision
    - Return `InviteValidateResponse(email=invitation.email, valid=True)`
    - Do NOT use FOR UPDATE here (read-only check, no modification)

    **POST `/invite-register`** (public, no auth required):
    - Request body: InviteRegisterRequest (token, display_name, password)
    - Response: TokenResponse (same as login -- access_token, refresh_token, token_type)
    - Status code: 201

    Implementation:
    1. Hash the token: `token_hash = hashlib.sha256(body.token.encode()).hexdigest()`
    2. Query Invitation with `with_for_update()` (pessimistic lock to prevent race condition per research pitfall 1):
       ```python
       result = await db.execute(
           select(Invitation).where(
               Invitation.token_hash == token_hash,
               Invitation.status == "pending",
               Invitation.expires_at > datetime.now(timezone.utc),
           ).with_for_update()
       )
       invitation = result.scalar_one_or_none()
       ```
    3. If not found: raise HTTPException(403, "This invite has expired. Contact your administrator for a new one.")
    4. Mark invitation as accepted:
       ```python
       invitation.status = "accepted"
       invitation.accepted_at = datetime.now(timezone.utc)
       ```
    5. Read default_user_class from platform settings:
       ```python
       from app.services import platform_settings
       default_class = await platform_settings.get(db, "default_user_class")
       ```
    6. Create user using the existing auth_service.create_user, but construct a SignupRequest-compatible object:
       - email = invitation.email (from the token lookup, NOT from user input -- per research: prevents email enumeration)
       - password = body.password
       - first_name = body.display_name.strip() (map display_name to first_name per research pitfall 5)
       - last_name = None
       - Use auth_service.create_user(db, signup_data, default_class=default_class)
       - If email already registered (409), return appropriate error
    7. Generate tokens (auto-login per locked decision):
       - Use auth_service.create_tokens(user) or same token generation as login endpoint
       - Return TokenResponse with access_token and refresh_token
    8. Commit: `await db.commit()`

    **Important notes:**
    - No email verification required (locked decision: invite link proves email ownership)
    - Email comes from the invitation record, NOT from user input (security: prevents email swap)
    - Use `with_for_update()` on the invitation query to prevent concurrent registration race condition
    - The existing partial invite code in signup (lines 78-102) should be left as-is for backward compatibility
    - Import `hashlib` at top of auth.py (already imported for existing invite code)
