    **Create `frontend/src/app/(auth)/invite/[token]/page.tsx`:**

    Follow the same patterns as register/page.tsx and reset-password/page.tsx. Use the same (auth) layout which provides centered card styling.

    **State management:**
    - `token` from route params (useParams)
    - `email` from token validation response
    - `displayName` for the name input
    - `password` for the password input
    - `isLoading` for form submission
    - `isValidating` for initial token check
    - `error` for invalid/expired token state

    **On mount (useEffect):**
    1. Extract `token` from URL params
    2. Call `GET /auth/invite-validate?token={token}` (use `http://localhost:8000` or env var -- follow same pattern as register page which uses hardcoded localhost)
    3. On success: set `email` from response
    4. On error (400/any): set `error` = "This invite has expired. Contact your administrator for a new one."
    5. Set `isValidating = false`

    **Rendering states:**
    - `isValidating === true`: Show spinner (same as register page loading state)
    - `error !== null`: Show error card with message and link to login page
    - Valid token: Show registration form

    **Registration form (valid token state):**
    - Card component (same as register page: Card, CardHeader, CardContent from shadcn/ui)
    - CardTitle: "You've been invited to Spectra" (locked decision: welcome variant header)
    - CardDescription: "Complete your account setup"

    Form fields:
    1. **Email** (read-only, pre-filled):
       - Label: "Email"
       - Input type="email", value={email}, disabled=true, className with muted styling
       - This is the locked/pre-filled email per locked decision
    2. **Display name** (required):
       - Label: "Display name"
       - Input type="text", placeholder="Your name", min 1 char
       - Single field per locked decision (no first/last split)
    3. **Password** (required):
       - Label: "Password"
       - Input type="password", placeholder="........"
       - Helper text: "Must be at least 8 characters"
       - min 8 chars client-side validation

    Submit button: "Create account" / "Creating account..." when loading

    **Form submission (handleSubmit):**
    1. Client-side validation: display name not empty, password >= 8 chars
    2. POST to `/auth/invite-register` with body: `{ token, display_name: displayName.trim(), password }`
    3. On success (201): response contains access_token and refresh_token
       - Store tokens: use the same auth storage pattern as the login flow (check useAuth hook for how tokens are stored -- likely localStorage or cookie)
       - Redirect to "/" (main app) -- auto-login per locked decision
    4. On error: show toast with error message (use sonner toast, same as register page)

    **Important implementation notes:**
    - Use "use client" directive at top
    - Import from same UI components as register page (Button, Input, Label, Card, etc.)
    - Use `toast` from sonner for error messages
    - Use `useParams` from next/navigation to get token
    - Use `useRouter` from next/navigation for redirect after registration
    - For auth token storage after successful invite-register, inspect the useAuth hook to understand the storage mechanism (likely stores in localStorage and updates auth state). If useAuth has a method that accepts tokens directly, use it. Otherwise, manually store tokens and redirect.
    - Do NOT use the existing `signup` function from useAuth (it calls the regular signup endpoint, not invite-register)
    - The API base URL should follow the same pattern used in register/page.tsx (currently hardcoded as http://localhost:8000)
