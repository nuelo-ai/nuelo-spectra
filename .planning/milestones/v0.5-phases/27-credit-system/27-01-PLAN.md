- `CreditAdjustmentRequest(BaseModel)`: amount (Decimal) with Field description "Positive to add, negative to deduct", reason (str) with Field(min_length=1, max_length=500), password (str) with Field(min_length=1) -- admin password re-entry per locked decision.

Use `from decimal import Decimal` for all credit amounts.

Create `backend/app/services/credit.py` with a `CreditService` class using `@staticmethod` async methods (follows existing service pattern like ChatService):

**deduct_credit(db: AsyncSession, user_id: UUID, cost: Decimal) -> CreditDeductionResult:**
- Import and call `get_class_config` to check if user's class has `reset_policy == "unlimited"`. To get the user's class, query User.user_class with the provided user_id.
- For unlimited users: DO NOT deduct balance. Create a CreditTransaction with transaction_type="usage", amount=-cost, balance_after=-1 (sentinel for unlimited). Return success=True, balance=Decimal("-1").
- For normal users: `select(UserCredit).where(UserCredit.user_id == user_id).with_for_update()` to lock the row.
- If no credit row found, return success=False with error message.
- If `credit.balance < cost`, calculate next_reset using `_get_next_reset_date()` and return success=False with error_message="You're out of credits. Credits reset on {date}." (per locked decision).
- Otherwise: `credit.balance = credit.balance - cost`, create CreditTransaction(user_id, amount=-cost, balance_after=credit.balance, transaction_type="usage"), `await db.flush()`.
- Use `Decimal` for all arithmetic, not float.

**get_balance(db: AsyncSession, user_id: UUID, user_class: str) -> CreditBalanceResponse:**
- Query UserCredit for the user.
- Load class config via `get_class_config(user_class)`.
- Calculate `is_low`: True when balance < 20% of tier allocation OR balance < Decimal("3"), whichever triggers first (research recommendation for low-credit threshold).
- Calculate `next_reset_at` using `_get_next_reset_date()`.
- For unlimited users: is_unlimited=True, is_low=False, balance from DB (or -1 sentinel).

**admin_adjust(db: AsyncSession, user_id: UUID, amount: Decimal, reason: str, admin_id: UUID) -> CreditBalanceResponse:**
- Lock UserCredit row with `with_for_update()`.
- Update balance: `credit.balance = credit.balance + amount` (amount can be positive or negative).
- Prevent balance going below 0: if result < 0, set to 0.
- Create CreditTransaction with transaction_type="admin_adjustment", reason=reason, admin_id=admin_id.
- Flush and return updated balance info.

**execute_reset(db: AsyncSession, credit: UserCredit, allocation: int, transaction_type: str = "auto_reset") -> None:**
- Set `credit.balance = Decimal(str(allocation))` (always resets to tier allocation, ignoring admin bonuses -- per locked decision).
- Set `credit.last_reset_at = datetime.now(timezone.utc)`.
- Create CreditTransaction with the given transaction_type, amount=(allocation - old_balance), balance_after=allocation.
- `await db.flush()`.

**manual_reset(db: AsyncSession, user_id: UUID, admin_id: UUID) -> CreditBalanceResponse:**
- Lock UserCredit row. Query user's class. Load tier config.
- Call `execute_reset()` with transaction_type="manual_reset".
- Per locked decision: manual reset restarts user's cycle from today (last_reset_at is set to now by execute_reset).
- Create audit-friendly CreditTransaction with admin_id set.
- Return updated balance.

**get_transaction_history(db: AsyncSession, user_id: UUID, limit: int = 50, offset: int = 0) -> list[CreditTransaction]:**
- Query CreditTransaction for user_id, ordered by created_at DESC, with limit/offset.

**get_credit_distribution(db: AsyncSession) -> list[dict]:**
- Aggregate query: JOIN user_credits with users, GROUP BY user_class, return count and avg_balance per class. This powers CREDIT-09 (admin dashboard widget).

**get_low_credit_users(db: AsyncSession, threshold_pct: float = 0.1) -> list[dict]:**
- Query users whose balance < 10% of their tier allocation. JOIN user_credits with users, load class configs to get allocations. Return user_id, email, balance, tier_allocation, user_class. This powers CREDIT-10.

**_get_next_reset_date(signup_date: datetime, last_reset_at: datetime | None, reset_policy: str) -> datetime | None:** (private static helper)
- If reset_policy in ("none", "unlimited"): return None.
- anchor = last_reset_at or signup_date.
- For "weekly": return anchor + timedelta(weeks=1).
- For "monthly": return anchor + timedelta(days=30).

**is_reset_due(signup_date: datetime, last_reset_at: datetime | None, reset_policy: str) -> bool:** (static method, public -- used by scheduler)
- If reset_policy in ("none", "unlimited"): return False.
- now = datetime.now(timezone.utc).
- anchor = last_reset_at or signup_date.
- For "weekly": return (now - anchor) >= timedelta(weeks=1).
- For "monthly": return (now - anchor) >= timedelta(days=30).

All methods use `from decimal import Decimal` for credit math. Import UserCredit, CreditTransaction, User from models. Import get_class_config from user_class service.
