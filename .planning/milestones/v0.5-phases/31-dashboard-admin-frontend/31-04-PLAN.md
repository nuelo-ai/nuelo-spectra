       - `UserDetail`: extends UserSummary with credit_balance: number, total_sessions: number, total_files: number, total_messages: number

       - `useUserActivity(userId)`: user activity timeline query

       - All mutations invalidate ["admin", "users"] queries on success

    3. Create `src/components/shared/ConfirmModal.tsx`:

       - Props: open, onClose, onConfirm, title, description, confirmLabel, variant (default/destructive)

       - For destructive actions (delete user, bulk delete)
       - Generates random 6-character alphanumeric code displayed to user

       - Confirm button only enabled when typed code matches
       - Props: open, onClose, onConfirm, title, description

    5. Create `src/components/shared/StatusBadge.tsx`:
       - Color-coded badge for status (active=green, inactive=red) and tier (free=gray, standard=blue, premium=purple)
       - Uses shadcn Badge with variant

    6. Create `src/components/shared/DataTableShell.tsx`:
       - Reusable wrapper for TanStack Table: renders table header, body, pagination controls
       - Handles empty state message
       - Server-side pagination: page, pageSize, total, onPageChange
       - Medium density row spacing per locked decision

    7. Create `src/components/users/UserFilters.tsx`:
       - Search bar (text input for name/email search)
       - Filter dropdowns: Status (all/active/inactive), Tier (all/free/standard/premium), Date range
       - Active filters shown as removable chips per locked decision
       - Calls onFilterChange callback with updated filter params

    8. Create `src/components/users/BulkActionBar.tsx`:
       - Appears at top when rows are selected (sticky position)
       - Shows count of selected rows
       - Action buttons: Activate, Deactivate, Change Tier, Adjust Credits, Delete
       - Delete triggers ChallengeCodeDialog; Deactivate triggers ConfirmModal; others trigger appropriate dialogs

    9. Create `src/components/users/UserTable.tsx`:
       - TanStack Table with columns: checkbox (select), Avatar + Name (first column per locked decision), Email, Status (StatusBadge), Tier (StatusBadge), Created, Last Login, Actions (three-dot dropdown)
       - Row actions dropdown: View Detail, Activate/Deactivate, Reset Password, Change Tier, Adjust Credits, Delete
       - Delete uses ChallengeCodeDialog, Deactivate uses ConfirmModal
       - Supports row selection for bulk operations
       - Server-side pagination (not client-side, per anti-pattern guidance)
