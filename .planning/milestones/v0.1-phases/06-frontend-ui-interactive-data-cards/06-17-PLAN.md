  <action>
Install the two missing dependencies from the frontend directory:

```bash
cd /Users/marwazisiagian/Documents/ms-dev/spectra-project/spectra-dev/frontend && npm install @tailwindcss/typography remark-gfm
```

Then edit `frontend/src/app/globals.css` to add the typography plugin import. In Tailwind CSS v4, plugins are imported as CSS, not configured in a JS config file. Add this import AFTER the existing `@import "tailwindcss"` line (line 1) and BEFORE `@import "tw-animate-css"` (line 2):

```css
@import "tailwindcss";
@import "@tailwindcss/typography";
@import "tw-animate-css";
```

This activates the `prose` utility classes that are already used in FileUploadZone.tsx and FileInfoModal.tsx. Without this import, the prose classes are inert and all markdown HTML renders unstyled.

WHY @tailwindcss/typography: The prose classes (prose prose-sm dark:prose-invert) provide automatic styling for all HTML elements that ReactMarkdown generates (h1-h6, p, ul, ol, table, thead, tbody, tr, td, th, blockquote, code, pre, strong, em). Without it, ReactMarkdown converts markdown to raw HTML elements that have no visual formatting.

WHY remark-gfm: ReactMarkdown v10 does NOT include GitHub Flavored Markdown by default. Tables (pipe-separated syntax), strikethrough, task lists, and autolinks require remark-gfm as an explicit plugin. Without it, table markdown renders as literal pipe characters in paragraph text.
