  <action>
    1. Create `frontend/src/hooks/useSSEStream.ts` (Client-side hook):
       - Export a `useSSEStream` hook that manages SSE streaming state
       - Interface `StreamState`:
         - `events: StreamEvent[]` -- all received SSE events
         - `isStreaming: boolean` -- whether stream is active
         - `error: string | null` -- error message if stream failed
         - `currentStatus: string | null` -- latest status message ("Generating code...", etc.)
         - `streamedText: string` -- accumulated text from content_chunk events
         - `completedData: Record<string, any> | null` -- final completion payload
       - Export function `startStream(fileId: string, message: string): void`
         - Makes POST request to `/api/chat/${fileId}/stream` with `{ content: message }`
         - Sets `isStreaming = true`
         - Uses `fetch` with ReadableStream reader (NOT EventSource -- POST body needed)
         - Parses SSE format: lines starting with "data: " contain JSON payloads
         - Handles event types from StreamEvent:
           - `coding_started/validation_started/execution_started/analysis_started`: update currentStatus with user-friendly message
           - `content_chunk`: append `event.text` to streamedText
           - `progress`: update currentStatus
           - `retry`: update currentStatus with retry info
           - `node_complete`: store node data in events array
           - `completed`: set completedData, set isStreaming = false
           - `error`: set error message, set isStreaming = false
         - On stream end (reader done): set isStreaming = false
         - Cleanup: AbortController abort on component unmount
       - Export function `resetStream(): void` -- clears all state for next query
       - Use `useRef` for AbortController to handle cleanup
       - Use `useCallback` for stable function references
       - Map status event types to user-friendly messages:
         - coding_started -> "Generating code..."
         - validation_started -> "Validating code..."
         - execution_started -> "Running analysis..."
         - analysis_started -> "Interpreting results..."

    2. Create `frontend/src/hooks/useChatMessages.ts`:
       - `useChatMessages(fileId: string | null)`:
         - TanStack Query hook wrapping GET /api/chat/{fileId}/messages
         - queryKey: ['chat', 'messages', fileId]
         - enabled: only when fileId is truthy
         - Returns `ChatMessageList`
         - Refetch on window focus
       - `useAddLocalMessage()`: helper to optimistically add a user message to the chat
         - Uses queryClient.setQueryData to append message locally before server confirmation
         - Called when user sends a message (immediate visual feedback)
       - `useInvalidateChatMessages(fileId: string)`: helper to invalidate and refetch after stream completes
  </action>
