<task type="auto">
  <name>Task 2: Create file schemas and file service</name>
  <files>
    backend/app/schemas/file.py
    backend/app/services/file.py
  </files>
  <action>
    **Create `backend/app/schemas/file.py`:**

    Follow the existing pattern from schemas/auth.py (Pydantic BaseModel with ConfigDict).

    Define these schemas:
    - `FileUploadResponse`: id (UUID), original_filename (str), file_size (int), file_type (str), created_at (datetime). ConfigDict(from_attributes=True).
    - `FileListItem`: id (UUID), original_filename (str), file_size (int), file_type (str), created_at (datetime), updated_at (datetime). ConfigDict(from_attributes=True).
    - `FileDetailResponse`: Same as FileListItem plus stored_filename (str). ConfigDict(from_attributes=True).

    **Create `backend/app/services/file.py`:**

    Follow the existing pattern from services/auth.py (static methods on a service class, async SQLAlchemy operations).

    Define `FileService` class with these static methods:

    1. `upload_file(db: AsyncSession, user_id: UUID, upload_file: UploadFile, file_extension: str) -> File`:
       - Get upload_dir from settings via `get_settings()`
       - Create user directory: `{upload_dir}/{user_id}/` using pathlib, mkdir(parents=True, exist_ok=True)
       - Generate UUID stored filename: `{uuid4()}{file_extension}`
       - Save file to disk using aiofiles in 1MB chunks (CHUNK_SIZE = 1024 * 1024)
       - Track total_size during write; raise HTTPException 413 if exceeds max_file_size_mb * 1024 * 1024
       - Validate file with pandas using `asyncio.to_thread()`:
         - CSV: `pd.read_csv(file_path, nrows=5)`
         - xlsx/xls: `pd.read_excel(file_path, nrows=5)`
         - Raise ValueError with clear message on failure
       - Create File model instance with all fields, db.add(), await db.commit(), await db.refresh()
       - On ANY exception in the try block, clean up the saved file (unlink if exists) before re-raising
       - Return the File record

    2. `list_user_files(db: AsyncSession, user_id: UUID) -> list[File]`:
       - `select(File).where(File.user_id == user_id).order_by(File.created_at.desc())`
       - Return list of File objects

    3. `get_user_file(db: AsyncSession, file_id: UUID, user_id: UUID) -> File | None`:
       - `select(File).where(File.id == file_id, File.user_id == user_id)`
       - Return single File or None (enforces user isolation)

    4. `delete_file(db: AsyncSession, file_id: UUID, user_id: UUID) -> bool`:
       - First get the file record (with user_id check for isolation)
       - If not found, return False
       - Store file_path before deleting
       - Use `await db.execute(delete(File).where(File.id == file_id, File.user_id == user_id))` for reliable cascade (database-level ON DELETE CASCADE handles chat_messages)
       - Await db.commit()
       - Delete physical file from disk using pathlib Path.unlink(missing_ok=True)
       - Return True

    Define module-level helper `_validate_file(file_path: str, file_type: str) -> None` (synchronous, called via asyncio.to_thread):
    - Try read_csv or read_excel with nrows=5
    - On exception, raise ValueError with descriptive message

    Import patterns to follow:
    - `from fastapi import UploadFile, HTTPException, status`
    - `from sqlalchemy import select, delete`
    - `from sqlalchemy.ext.asyncio import AsyncSession`
    - `from app.models.file import File`
    - `from app.config import get_settings`
    - `import asyncio, aiofiles, pandas as pd`
    - `from pathlib import Path`
    - `from uuid import UUID, uuid4`

    IMPORTANT: Do NOT use `await file.read()` (reads entire file into memory). Use chunked reads with `while chunk := await upload_file.read(CHUNK_SIZE)`.
    IMPORTANT: The pandas validation call MUST be wrapped in `asyncio.to_thread()` to avoid blocking the async event loop.
    IMPORTANT: On delete, use SQL delete statement with WHERE clause (not ORM session.delete) for reliable cascade in async context (see research Pitfall 6).
  </action>
