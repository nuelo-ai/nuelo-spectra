  <action>
    1. **backend/app/models/base.py** - Base class:
       - `class Base(DeclarativeBase): pass`
       - This is the single source of truth for metadata

    2. **backend/app/models/user.py** - User model:
       - Table name: "users"
       - id: UUID primary key with default=uuid4
       - email: String(255), unique=True, index=True
       - hashed_password: String(255)
       - first_name: String(100), nullable=True
       - last_name: String(100), nullable=True
       - is_active: bool, default=True
       - created_at: DateTime(timezone=True), default=datetime.now(timezone.utc)
       - updated_at: DateTime(timezone=True), default + onupdate=datetime.now(timezone.utc)
       - Use SQLAlchemy 2.0 `Mapped[]` and `mapped_column()` syntax
       - Add relationships: files (one-to-many), chat_messages (one-to-many)

    3. **backend/app/models/file.py** - File model:
       - Table name: "files"
       - id: UUID primary key with default=uuid4
       - user_id: UUID, ForeignKey("users.id", ondelete="CASCADE"), index=True, NOT NULL
       - original_filename: String(255) - user-visible name
       - stored_filename: String(255) - UUID-based storage name
       - file_path: String(500) - full storage path
       - file_size: BigInteger - size in bytes
       - file_type: String(50) - "csv", "xlsx", "xls"
       - created_at: DateTime(timezone=True)
       - updated_at: DateTime(timezone=True)
       - Add relationship back to user, and one-to-many to chat_messages

    4. **backend/app/models/chat_message.py** - ChatMessage model:
       - Table name: "chat_messages"
       - id: UUID primary key with default=uuid4
       - user_id: UUID, ForeignKey("users.id", ondelete="CASCADE"), index=True, NOT NULL
       - file_id: UUID, ForeignKey("files.id", ondelete="CASCADE"), index=True, NOT NULL
       - role: String(20) - "user" or "assistant"
       - content: Text
       - message_type: String(50), nullable=True - for future use (e.g., "code", "analysis", "error")
       - metadata_json: JSON, nullable=True - for structured data (code blocks, execution results)
       - created_at: DateTime(timezone=True)
       - Relationships back to user and file

    5. **backend/app/models/__init__.py** - Import all models:
       ```python
       from app.models.base import Base
       from app.models.user import User
       from app.models.file import File
       from app.models.chat_message import ChatMessage
       ```

    6. **backend/alembic.ini** - Standard Alembic config:
       - script_location = alembic
       - sqlalchemy.url will be overridden in env.py from settings

    7. **backend/alembic/env.py** - Async Alembic setup:
       - Import Base from app.models.base
       - Import ALL models (user, file, chat_message) so Alembic sees them
       - Set target_metadata = Base.metadata
       - Override sqlalchemy.url from Settings
       - Use async_engine_from_config for online migrations
       - Follow the async cookbook from Alembic docs exactly

    8. **backend/alembic/script.py.mako** - Standard Alembic template

    After creating all files, run:
    ```bash
    cd backend
    alembic revision --autogenerate -m "create users files chat_messages tables"
    alembic upgrade head
    ```

    CRITICAL: Both user_id foreign keys in File and ChatMessage enforce data isolation at the database level. The ondelete="CASCADE" ensures deleting a user removes all their data. Index on user_id columns is required for query performance.
  </action>
