       - Use same secret_key and algorithm from settings

       - Decode JWT with explicit algorithm

       - Return email from "sub" claim

       - `ForgotPasswordRequest(BaseModel)`: email (EmailStr)

    3. **backend/app/services/email.py** - Email service:

       Create an abstract email sending interface that works with multiple providers:
       ```python
       async def send_password_reset_email(to_email: str, reset_link: str, settings: Settings) -> bool:
       ```

       Implementation strategy:
       - Use httpx async client to call email API
       - Support Mailgun API format (POST to https://api.mailgun.net/v3/{domain}/messages)
       - Include fallback: If EMAIL_SERVICE_API_KEY is not set or empty, log the reset link to console instead of sending email (development mode)
       - This allows development without email service configured

       Email content:
       - Subject: "Spectra - Password Reset Request"
       - Body: Simple HTML with reset link, 10-minute expiry notice
       - From: settings.email_from

       IMPORTANT: The development fallback (console logging) is critical for Plan 02 testing. Do NOT require email service for local development.

    4. **Add to backend/app/routers/auth.py** (from Plan 02):

       `POST /auth/forgot-password`:
       - Accepts ForgotPasswordRequest body
       - Query user by email
       - If user exists: create reset token, build reset link (settings.frontend_url + "/reset-password?token=" + token), send email
       - ALWAYS return 202 with {"message": "If the email exists, a reset link has been sent"} regardless of whether user exists
       - This prevents user enumeration attacks

       `POST /auth/reset-password`:
       - Accepts ResetPasswordRequest body
       - Verify reset token to get email
       - Query user by email
       - If not found, raise HTTPException(400, "Invalid token")
       - Hash new password, update user.hashed_password
       - Commit to database
       - Return 200 with {"message": "Password reset successful"}

    IMPORTANT: The forgot-password endpoint must ALWAYS return 202 to prevent email enumeration. Even if the email doesn't exist in the database, return the same response.

       ```python
       )
       ```

       ```python
       from contextlib import asynccontextmanager

           yield
           # Shutdown: dispose engine
           await engine.dispose()

       app = FastAPI(title="Spectra API", version="0.1.0", lifespan=lifespan)
       ```
