    1. **backend/app/utils/security.py** - Password and JWT utilities:

       Password hashing (use pwdlib, NOT passlib):
       ```python
       from pwdlib import PasswordHash
       password_hash = PasswordHash.recommended()  # Uses Argon2

       def hash_password(password: str) -> str:
           return password_hash.hash(password)

       def verify_password(plain_password: str, hashed_password: str) -> bool:
           return password_hash.verify(plain_password, hashed_password)
       ```

       JWT token creation:
       - `create_access_token(user_id: str, settings: Settings) -> str` - Creates JWT with {"sub": user_id, "exp": now+30min, "type": "access"}
       - `create_refresh_token(user_id: str, settings: Settings) -> str` - Creates JWT with {"sub": user_id, "exp": now+30days, "type": "refresh"}
       - `create_tokens(user_id: str, settings: Settings) -> dict` - Returns {"access_token": ..., "refresh_token": ..., "token_type": "bearer"}
       - `verify_token(token: str, token_type: str, settings: Settings) -> str` - Decodes JWT, validates type claim, returns user_id. Raises HTTPException on invalid/expired token.
       - ALWAYS specify algorithms=[settings.algorithm] in jwt.decode() to prevent "none" algorithm attack
       - Only store user_id (as str(uuid)) in "sub" claim - NEVER store email or PII in JWT

    2. **backend/app/schemas/auth.py** - Auth request/response schemas:

       - `LoginRequest(BaseModel)`: email (EmailStr), password (str)
       - `TokenResponse(BaseModel)`: access_token (str), refresh_token (str), token_type (str = "bearer")
       - `RefreshRequest(BaseModel)`: refresh_token (str)

    1. **backend/app/services/auth.py** - Auth business logic:

       `async def create_user(db: AsyncSession, signup: SignupRequest) -> User`:
       - Check if email already exists (SELECT where email = signup.email)
       - If exists, raise HTTPException(409, "Email already registered")
       - Hash password with hash_password()
       - Create User instance with hashed_password
       - db.add(user), await db.commit(), await db.refresh(user)
       - Return user

       `async def authenticate_user(db: AsyncSession, email: str, password: str) -> User | None`:
       - Query User by email
       - If not found or password invalid, return None
       - Return user
       - IMPORTANT: Use constant-time comparison via pwdlib verify (already built-in)

       `async def get_user_by_id(db: AsyncSession, user_id: UUID) -> User | None`:
       - Query User by id
       - Return user or None

    2. **backend/app/dependencies.py** - Reusable dependencies:

       `get_db()` - Import from database.py (re-export for cleaner imports)

       `get_current_user(token, db, settings) -> User`:
       - Use OAuth2PasswordBearer(tokenUrl="auth/login") for token extraction
       - Call verify_token(token, "access", settings) to get user_id
       - Query User by id from database
       - If user not found or not active, raise 401
       - Return User object
       - This dependency is the KEY to data isolation - every protected endpoint uses it

       Export as typed dependency:
       ```python
       CurrentUser = Annotated[User, Depends(get_current_user)]
       DbSession = Annotated[AsyncSession, Depends(get_db)]
       ```

    3. **backend/app/routers/auth.py** - Auth endpoints:

       `POST /auth/signup`:
       - Accepts SignupRequest body
       - Calls auth_service.create_user()
       - Returns TokenResponse (auto-login after signup)
       - Status code 201

       `POST /auth/login`:
       - Accepts LoginRequest body (NOT OAuth2PasswordRequestForm - we use JSON, not form data)
       - Calls auth_service.authenticate_user()
       - If None, raise HTTPException(401, "Incorrect email or password")
       - Create tokens with create_tokens(str(user.id), settings)
       - Return TokenResponse

       `POST /auth/refresh`:
       - Accepts RefreshRequest body
       - Verify refresh token with verify_token(token, "refresh", settings)
       - Get user by id from database
       - If not found/inactive, raise 401
       - Create new token pair
       - Return TokenResponse

       `GET /auth/me`:
       - Protected endpoint (requires get_current_user)
       - Returns UserResponse of current user
       - This endpoint validates the entire auth chain works

       Router prefix: "/auth", tags: ["Authentication"]
