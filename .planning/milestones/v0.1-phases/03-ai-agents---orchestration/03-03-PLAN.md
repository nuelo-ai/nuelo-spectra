<feature>
  <name>AST-Based Code Validation</name>
  <files>backend/app/agents/code_checker.py, backend/tests/test_code_checker.py</files>
  <behavior>
    The CodeValidator uses ast.NodeVisitor to walk the AST of generated Python code and check:

    1. Syntax: Code must parse without SyntaxError
       - Input: "def foo(" -> Output: ValidationResult(is_valid=False, errors=["Syntax error: ..."])
       - Input: "import pandas as pd" -> Output: ValidationResult(is_valid=True, errors=[])

    2. Import allowlist: Only allowed libraries may be imported
       - Input: "import pandas" -> Output: valid (pandas in allowlist)
       - Input: "import os" -> Output: invalid, error "Disallowed module: os"
       - Input: "from subprocess import run" -> Output: invalid, error "Disallowed module: subprocess"
       - Input: "import pandas; import numpy" -> Output: valid (both allowed)
       - Input: "import pandas; import os" -> Output: invalid (os not allowed)

    3. Unsafe builtins: eval, exec, __import__, compile must be rejected
       - Input: "eval('1+1')" -> Output: invalid, error "Unsafe builtin: eval"
       - Input: "exec('print(1)')" -> Output: invalid, error "Unsafe builtin: exec"
       - Input: "__import__('os')" -> Output: invalid, error "Unsafe builtin: __import__"
       - Input: "compile('pass', '', 'exec')" -> Output: invalid, error "Unsafe builtin: compile"

    4. Unsafe operations: open, file I/O functions
       - Input: "open('file.txt')" -> Output: invalid, error "Unsafe operation: open"
       - Input: "f = open('test', 'w')" -> Output: invalid

    5. Attribute-based unsafe calls: os.system, subprocess.run via method chains
       - Input: "import os\nos.system('ls')" -> After import check catches 'os', this is already blocked
       - But also check for: getattr calls that could bypass checks

    6. Valid code passes all checks:
       - Input: "import pandas as pd\ndf = pd.DataFrame({'a': [1,2,3]})\nresult = df.describe()" -> valid
       - Input: "import numpy as np\nresult = np.mean([1,2,3])" -> valid

    ValidationResult dataclass:
      - is_valid: bool
      - errors: list[str]

    validate_code(code: str) function:
      - Takes code string
      - Loads allowlist from config (or accepts as parameter for testing)
      - Returns ValidationResult
  </behavior>
  <implementation>
    1. Create ValidationResult dataclass with is_valid and errors fields
    2. Create CodeValidator(ast.NodeVisitor) class:
       - __init__(allowed_modules, unsafe_builtins, unsafe_modules, unsafe_operations)
       - visit_Import: check each alias.name against allowed_modules and unsafe_modules
       - visit_ImportFrom: check node.module against allowed_modules and unsafe_modules
       - visit_Call: check for unsafe builtins (ast.Name nodes with id in unsafe set)
       - Handle nested imports: "from os.path import join" -> check "os.path" and "os"
    3. Create validate_code(code: str, allowlist: dict | None = None) -> ValidationResult:
       - Try ast.parse(code), return syntax error if fails
       - Instantiate CodeValidator with allowlist from config (or provided)
       - Call validator.visit(tree)
       - Return ValidationResult
    4. All allowlist data loaded from YAML config via get_allowed_libraries(), get_unsafe_builtins(), get_unsafe_modules()
  </implementation>
</feature>
