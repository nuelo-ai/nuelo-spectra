  <behavior>
    Test the Manager Agent's routing classification logic, fallback behavior, and route-specific agent adaptations.

    **Test categories and expected behaviors:**

    **1. Routing Classification (manager_node)**
    - Memory queries route to MEMORY_SUFFICIENT:
      - "What were the columns?" -> MEMORY_SUFFICIENT
      - "Can you explain that result?" -> MEMORY_SUFFICIENT
      - "What was the average we calculated?" -> MEMORY_SUFFICIENT
      - "Show me the result again" -> MEMORY_SUFFICIENT
    - Modification queries route to CODE_MODIFICATION:
      - "Add a trend column" (with previous_code) -> CODE_MODIFICATION
      - "Filter by region = North" (with previous_code) -> CODE_MODIFICATION
      - "Sort by date descending" (with previous_code) -> CODE_MODIFICATION
      - "Show top 10 instead of 5" (with previous_code) -> CODE_MODIFICATION
    - New analysis queries route to NEW_ANALYSIS:
      - First query (no messages): "Show me sales by region" -> NEW_ANALYSIS
      - Different analysis: "Now analyze retention instead" -> NEW_ANALYSIS
      - Explicit new: "Start fresh with monthly trends" -> NEW_ANALYSIS
    - Edge cases:
      - Modification query WITHOUT previous_code -> NEW_ANALYSIS (can't modify nothing)
      - Empty query -> NEW_ANALYSIS (fallback)

    **2. Fallback Behavior (ROUTING-04)**
    - LLM timeout -> NEW_ANALYSIS with fallback reasoning
    - LLM returns malformed response -> NEW_ANALYSIS with fallback reasoning
    - LLM raises exception -> NEW_ANALYSIS with fallback reasoning
    - Routing decision is logged even on fallback

    **3. Route-Specific Agent Behavior**
    - MEMORY_SUFFICIENT route: data_analysis_agent returns analysis without generated_code/execution_result
    - CODE_MODIFICATION route: coding_agent receives previous_code in prompt
    - NEW_ANALYSIS route: coding_agent behaves identically to pre-Phase-9 behavior

    **4. Graph Topology**
    - Manager is entry point
    - Manager routes to data_analysis (MEMORY_SUFFICIENT) or coding_agent (CODE_MODIFICATION/NEW_ANALYSIS)
    - Existing edges unchanged (coding_agent -> code_checker, execute -> data_analysis)

    **5. Configuration**
    - Manager agent config loads from prompts.yaml
    - routing_context_messages defaults to 10
    - Manager uses specified provider/model

    **6. Structured Logging (ROUTING-08)**
    - Routing decision logged with event, route, reasoning, message_count, has_previous_code

    All tests must be fully mocked — zero live API calls. Use the same mocking patterns as Phase 7 Plan 4 tests (mock LLM responses, mock get_llm factory, mock settings).
