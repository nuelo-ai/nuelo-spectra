    **Create a helper function for API key resolution.** All 4 agents need to resolve the API key based on the per-agent provider (not global `settings.llm_provider`). Create a shared helper in either a new utility or inline in each agent.

    Recommended approach: Add a `get_api_key_for_provider(provider: str, settings) -> str` function to `backend/app/agents/config.py` (the config module is already imported by all agents). This function maps provider names to settings fields:
    - `"anthropic"` -> `settings.anthropic_api_key`
    - `"openai"` -> `settings.openai_api_key`
    - `"google"` -> `settings.google_api_key`
    - `"ollama"` -> `""` (Ollama needs no API key)
    - `"openrouter"` -> `settings.openrouter_api_key`
    - Unknown -> raise `ValueError` with clear message

    NOTE: This helper goes into `backend/app/agents/config.py` (not `backend/app/config.py`). It accepts a settings instance as argument to avoid circular imports.

    **Migrate `backend/app/agents/onboarding.py`:**

    ```python

    temperature = get_agent_temperature("onboarding")
    api_key = get_api_key_for_provider(provider, self.settings)
    max_tokens = get_agent_max_tokens("onboarding")

    # Build kwargs for provider-specific options
    kwargs = {"max_tokens": max_tokens, "temperature": temperature}
    if provider == "ollama":
        kwargs["base_url"] = self.settings.ollama_base_url

    llm = get_llm(provider=provider, model=model, api_key=api_key, **kwargs)
    ```
