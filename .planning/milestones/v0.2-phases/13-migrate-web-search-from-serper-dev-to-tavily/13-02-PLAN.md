**DEFECT 2 fix: ChatInterface memory streaming branch must extract and render follow-up suggestion chips.**

In `frontend/src/components/chat/ChatInterface.tsx`, in the `isMemoryRoute` streaming branch (lines 369-394), extract `follow_up_suggestions` from the `da_response` node_complete event and render suggestion chips below the ChatMessage.

Replace the current memory route block (lines 369-394) with logic that:
1. Extracts `follow_up_suggestions` from `analysisEvent?.data?.follow_up_suggestions` (same pattern as `getStreamingDataCard()` at lines 213-216)
2. After the ChatMessage, renders follow-up chips using the same chip UI pattern as DataCard (lines 175-191 of DataCard.tsx)

Updated code for the `if (isMemoryRoute)` block (lines 369-394):
```tsx
if (isMemoryRoute) {
  const analysisEvent = events.find(
    (e) => e.type === "node_complete" && e.node === "da_response"
  );
  const analysisText =
    analysisEvent?.data?.analysis ||
    analysisEvent?.data?.final_response ||
    streamedText;

  // Extract follow-up suggestions from da_response event
  const followUpSuggestions: string[] | undefined =
    analysisEvent?.data?.follow_up_suggestions;

  if (analysisText) {
    return (
      <div>
        <ChatMessage
          message={{
            id: "streaming",
            file_id: fileId,
            role: "assistant",
            content: analysisText,
            message_type: null,
            metadata_json: null,
            created_at: new Date().toISOString(),
          }}
          isStreaming={!analysisEvent}
          streamedText={analysisText}
        />
        {/* Follow-up suggestions for memory route */}
        {followUpSuggestions && followUpSuggestions.length > 0 && analysisEvent && (
          <div className="max-w-[70%] ml-11 mt-2 space-y-2">
            <h4 className="text-xs font-medium text-muted-foreground">Continue exploring</h4>
            <div className="flex flex-wrap gap-2">
              {followUpSuggestions.map((suggestion) => (
                <button
                  key={suggestion}
                  onClick={() => handleSend(suggestion)}
                  className="px-3 py-1.5 rounded-full border text-xs hover:bg-accent hover:border-primary/30 transition-all duration-200 bg-background cursor-pointer"
                >
                  {suggestion}
                </button>
              ))}
            </div>
          </div>
        )}
      </div>
    );
  }
  return <TypingIndicator />;
}
```

Key details:
- `isStreaming={!analysisEvent}` makes the cursor disappear when the event arrives (stream complete)
- Chips only render when `analysisEvent` exists (stream complete, not while still typing)
- `ml-11` aligns chips with the message content (past the avatar)
- Chip styling matches DataCard's follow-up chip classes exactly
