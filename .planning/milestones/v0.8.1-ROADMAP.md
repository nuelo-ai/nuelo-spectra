### Phase 54: Pulse Analysis Fixes

**Goal:** Pulse Analysis shows accurate data and provides a complete user experience — credits reflect actual usage, Signal View works on mobile, users can bridge to Chat from a Signal, and timestamps show both date and time throughout
**Depends on:** Phase 53
**Requirements:** PULSE-01, PULSE-02, PULSE-03, PULSE-04, PULSE-05
**Plans:** 4 plans

Plans:

- [x] 54-01-PLAN.md — Credits Used accuracy: backend aggregate subquery + schema + router + frontend type wiring (PULSE-01)
- [x] 54-02-PLAN.md — Timestamp date+time fix: activity-feed, file-table, and collections page (PULSE-04, PULSE-05)
- [x] 54-03-PLAN.md — Mobile Signal View toggle + Chat bridge button in SignalDetailPanel (PULSE-02, PULSE-03)
- [x] 54-04-PLAN.md — Human visual verification of all 5 PULSE requirements

**Details:**

- `COALESCE(SUM(credit_cost), 0.0)` correlated subquery in `CollectionService.get_collection_detail` returns 0.0 when no completed runs — no null from DB; `credits_used: float = 0.0` schema default ensures create response valid without field
- `toLocaleString` used instead of `toLocaleDateString` for Pulse timestamps — `toLocaleDateString` silently ignores hour/minute options in all browsers
- Mobile toggle uses `showDetail` boolean in `signals/page.tsx` — list panel and detail panel each wrapped in div with `sm:flex` breakpoint classes; selecting a signal sets `showDetail` true
- Chat bridge button in `SignalDetailPanel` disabled when `collectionFiles.length === 0` to prevent empty session creation; opens new chat session in a new tab (not same-tab navigation)
- PULSE-03 post-verification fix: Chat bridge button styled green

**Completed:** 2026-03-10

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- WorkspacePage SidebarTrigger uses shrink-0 header strip above flex-1 content — not sticky/fixed
- Nav icon alignment requires SidebarGroup wrapper (not just pl-1 removal) — shadcn p-2 padding context is provided by SidebarGroup, not SidebarMenu
- toLocaleString vs toLocaleDateString — toLocaleDateString ignores time options in all browsers; use toLocaleString when hour/minute display is required
- Chat bridge button opens new tab — UX decision to avoid losing current Pulse context

**Issues Resolved:**
- LBAR-01: SidebarTrigger missing from Collection Detail, Signal View, and Report pages (fixed in 53-05)
- LBAR-02: Nav icon misalignment due to missing SidebarGroup wrapper (fixed in 53-06)
- Credits Used card hardcoded "5" replaced with actual aggregate from completed PulseRun records
- Timestamps showing date only — fixed to show date + time
