<task type="auto">
  <name>Task 2: Create backend docker-entrypoint.sh</name>
  <files>backend/docker-entrypoint.sh</files>
  <action>
Create `backend/docker-entrypoint.sh` with the following behavior:

1. **Shebang and strict mode:** `#!/usr/bin/env bash` with `set -euo pipefail`

2. **Parse DB connection from DATABASE_URL:**
   - Support both `postgresql+asyncpg://user:pass@host:port/db` and `postgresql://user:pass@host:port/db` formats
   - Extract host with: `DB_HOST="${DB_HOST:-$(echo "$DATABASE_URL" | sed -E 's|.*@([^:/]+).*|\1|')}"` — allows override via explicit `DB_HOST` env var
   - Extract port with: `DB_PORT="${DB_PORT:-$(echo "$DATABASE_URL" | sed -E 's|.*@[^:]+:([0-9]+)/.*|\1|')}"` — allows override via explicit `DB_PORT` env var
   - Default port: `DB_PORT="${DB_PORT:-5432}"` (handles URLs without explicit port)

3. **Wait for PostgreSQL with bounded retries:**
   - Use `pg_isready -h "$DB_HOST" -p "$DB_PORT" -q` in an `until` loop
   - Max 30 retries with 2-second sleep between attempts (60s total timeout)
   - Log each retry attempt with count: `[entrypoint] PostgreSQL not ready (attempt N/30), retrying in 2s...`
   - Exit 1 with clear error if max retries exceeded

4. **Run Alembic migrations:**
   - Run from /app (WORKDIR): `/app/.venv/bin/python -m alembic upgrade head`
   - Use the venv python explicitly (not system python) since uv installs to .venv
   - alembic.ini's `prepend_sys_path = .` and `script_location = %(here)s/alembic` work from /app

5. **Start uvicorn via exec:**
   - `exec /app/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers "${UVICORN_WORKERS:-1}"`
   - `exec` replaces bash with uvicorn as PID 1 for proper SIGTERM handling
   - `UVICORN_WORKERS` defaults to 1 if not set

Log prefix all messages with `[entrypoint]` for easy identification in container logs.

Make the file executable: `chmod +x backend/docker-entrypoint.sh`
  </action>
