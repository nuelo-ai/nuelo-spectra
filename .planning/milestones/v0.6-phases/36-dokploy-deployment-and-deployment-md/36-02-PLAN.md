<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Deploy admin backend (Tailscale-only, no domain)</name>
  <action>
    The admin backend uses the SAME Dockerfile.backend as the public backend but with SPECTRA_MODE=admin.
    It must NOT have a domain assigned in Dokploy. Access is via Tailscale only.

    **Step 1 — Create Dokploy Application for admin backend:**
    In Dokploy UI → Applications → New Application:

    General tab:
    - Name: `spectra-admin-backend`
    - Source: Git (same repo as public backend)
    - Repository: `your-org/spectra-dev`
    - Branch: `main` (same branch)
    - Build Type: `Dockerfile`
    - Dockerfile Path: `Dockerfile.backend`
    - Docker Context Path: `.`

    **Step 2 — Set environment variables (Environment tab):**
    ```
    DATABASE_URL=postgresql+asyncpg://spectra:PASSWORD@spectra-db-HASH:5432/spectra
    SPECTRA_MODE=admin
    SECRET_KEY=<SAME-secret-key-as-public-backend>
    ALGORITHM=HS256
    ACCESS_TOKEN_EXPIRE_MINUTES=30
    REFRESH_TOKEN_EXPIRE_DAYS=30
    FRONTEND_URL=
    CORS_ORIGINS=["http://<TAILSCALE_IP>:3001"]
    ADMIN_CORS_ORIGIN=http://<TAILSCALE_IP>:3001
    APP_VERSION=v0.6.0
    ANTHROPIC_API_KEY=sk-ant-...
    OPENAI_API_KEY=
    GOOGLE_API_KEY=
    OLLAMA_BASE_URL=
    OPENROUTER_API_KEY=
    LANGSMITH_API_KEY=
    LANGSMITH_TRACING=
    LANGSMITH_PROJECT=
    E2B_API_KEY=
    TAVILY_API_KEY=
    SEARCH_DEPTH=basic
    SEARCH_DAILY_QUOTA=
    SMTP_HOST=
    SMTP_PORT=587
    SMTP_USER=
    SMTP_PASS=
    SMTP_FROM_EMAIL=
    SMTP_FROM_NAME=
    ENABLE_SCHEDULER=false
    ADMIN_EMAIL=
    ADMIN_PASSWORD=
    ```
    Critical notes:
    - DATABASE_URL: SAME value as public backend (shared PostgreSQL instance)
    - SECRET_KEY: SAME value as public backend (both must match — they share the same DB)
    - SPECTRA_MODE=admin: activates admin-only routes and disables public routes
    - CORS_ORIGINS and ADMIN_CORS_ORIGIN: use the Tailscale IP from Plan 36-01 Task 2
    - ENABLE_SCHEDULER=false: only one service should run the scheduler (public backend handles it)
    - FRONTEND_URL: leave empty (not used in admin mode)

    **Step 3 — Configure published port for Tailscale access (Advanced tab → Ports):**
    Click Advanced → Ports → Add Port:
    - Published Port: `8001` (host port — what Tailscale clients connect to)
    - Target Port: `8000` (container port — uvicorn listens here)
    - Protocol: TCP
    Click Save.

    **Step 4 — DO NOT add any domain (Domains tab):**
    Leave the Domains tab empty. Adding a domain here would route it through Traefik publicly.

    **Step 5 — Deploy.**
    Monitor logs. Expect the same startup sequence as public backend:
    pg_isready wait → alembic upgrade head → uvicorn start.
    Alembic will apply no new migrations if public backend already ran them — "No migrations to apply" is expected and correct.
  </action>
  <verify>
    On the Dokploy host server:
    ```bash
    # Check admin backend is running
    docker ps | grep admin-backend

    # Get admin backend container ID
    ADMIN_BE=$(docker ps --filter "name=admin-backend" --format "{{.ID}}" | head -1)

    # Check SPECTRA_MODE env var is set to admin
    docker exec $ADMIN_BE env | grep SPECTRA_MODE
    # Expected: SPECTRA_MODE=admin

    # Check port 8001 is published on host
    docker ps --filter "name=admin-backend" --format "{{.Ports}}"
    # Expected: 0.0.0.0:8001->8000/tcp

    # Health check from Tailscale client (tailscale up --accept-routes first):
    curl http://<TAILSCALE_IP>:8001/health
    # Expected: {"status": "ok"}

    # Verify NOT reachable from public internet:
    curl --max-time 5 http://<SERVER_PUBLIC_IP>:8001/health
    # Expected: timeout or connection refused (firewall blocking)
    ```

    Confirm:
    - [ ] Dokploy shows spectra-admin-backend as "Running"
    - [ ] SPECTRA_MODE=admin is set
    - [ ] Port 8001 published on host
    - [ ] No domain assigned in Dokploy
    - [ ] Health endpoint accessible via Tailscale, blocked from public internet
  </verify>
  <done>spectra-admin-backend is running with SPECTRA_MODE=admin, port 8001 published for Tailscale access, no Dokploy domain assigned, and health endpoint confirmed reachable only via Tailscale.</done>
  <resume-signal>Type "admin-backend-deployed" once the admin backend is running and verified reachable via Tailscale.</resume-signal>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Deploy public frontend and admin frontend</name>
  <action>
    Deploy both frontends. The public frontend needs the public backend URL; the admin frontend needs the admin backend via Tailscale.

    **PART A — Public Frontend:**

    In Dokploy UI → Applications → New Application:

    General tab:
    - Name: `spectra-public-frontend`
    - Source: Git (same repo)
    - Branch: `main`
    - Build Type: `Dockerfile`
    - Dockerfile Path: `Dockerfile.frontend`
    - Docker Context Path: `.`

    Environment tab:
    ```
    BACKEND_URL=http://<PUBLIC-BACKEND-CONTAINER-IP>:8000
    ```
    To get the public backend container IP, run on the Dokploy host:
    ```bash
    docker inspect $(docker ps --filter "name=public-backend" --format "{{.ID}}" | head -1) \
      | python3 -c "import sys,json; d=json.load(sys.stdin); print(d[0]['NetworkSettings']['Networks']['dokploy-network']['IPAddress'])"
    ```
    Example: `BACKEND_URL=http://10.254.0.3:8000`

    NOTE: In Plan 36-03, once the public backend gets a domain (api.yourdomain.com), update this to `BACKEND_URL=https://api.yourdomain.com`. For now, use the container IP for initial validation.

    Domains tab: Leave empty (will add HTTPS domain in Plan 36-03).
    Click Deploy.

    ---

    **PART B — Admin Frontend:**

    In Dokploy UI → Applications → New Application:

    General tab:
    - Name: `spectra-admin-frontend`
    - Source: Git (same repo)
    - Branch: `main`
    - Build Type: `Dockerfile`
    - Dockerfile Path: `Dockerfile.admin`
    - Docker Context Path: `.`

    Environment tab:
    ```
    BACKEND_URL=http://<TAILSCALE_IP>:8001
    ```
    Use the Tailscale IP from Plan 36-01 Task 2.
    Example: `BACKEND_URL=http://100.64.0.5:8001`

    Advanced tab → Ports → Add Port:
    - Published Port: `3001` (host port for Tailscale access)
    - Target Port: `3000` (container port — Next.js server listens here)
    - Protocol: TCP

    Domains tab: Leave empty. Admin frontend is Tailscale-only.
    Click Deploy.
  </action>
  <verify>
    After both services are deployed:

    ```bash

    docker exec $PUB_FE curl -s http://localhost:3000/api/health

    # Verify admin frontend container is running

    # Check admin frontend port 3001 is published

    # From a Tailscale client device (with --accept-routes):
    curl http://<TAILSCALE_IP>:3001/api/health
    # Expected: {"status":"ok"} (proxied to admin backend via BACKEND_URL)
    ```

    Confirm:
    - [ ] spectra-public-frontend Running in Dokploy, no domain yet
    - [ ] spectra-admin-frontend Running in Dokploy, port 3001 published, no domain
    - [ ] Public frontend /api/health returns 200 (proxies to public backend)
    - [ ] Admin frontend /api/health accessible via Tailscale at http://<TAILSCALE_IP>:3001
  </verify>
