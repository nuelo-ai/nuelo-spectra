However, add a dedicated `api` mode CORS configuration. In the CORS section (around line 297), add:
```python
if mode == "api":
    # API mode accepts requests from any origin (API clients, not browsers)
    cors_origins = ["*"]
```

Wait — actually, with `allow_credentials=True`, wildcard origins are not allowed by CORS spec. Since API keys use Bearer tokens (not cookies), API mode does NOT need `allow_credentials=True`. Update the CORS block so that when `mode == "api"`, use `allow_origins=["*"]` and `allow_credentials=False`. This is correct because API clients authenticate via Bearer header, not cookies.

Add this before the existing `app.add_middleware(CORSMiddleware, ...)` block:

```python
if mode == "api":
    # API mode: Bearer token auth, no cookies needed, allow all origins
    app.add_middleware(
        CORSMiddleware,
        allow_origins=["*"],
        allow_credentials=False,
        allow_methods=["GET", "POST", "PUT", "DELETE"],
        allow_headers=["Authorization", "Content-Type"],
    )
else:
    app.add_middleware(
        CORSMiddleware,
        allow_origins=cors_origins,
        allow_credentials=True,
        allow_methods=["*"],
        allow_headers=["*"],
        expose_headers=["X-Admin-Token"],
    )
```
