**1. Add in-memory deactivation set to dependencies** (`backend/app/dependencies.py`):

Add module-level revocation mechanism (following `_reset_cooldowns` pattern in auth.py and `_login_attempts` pattern in admin/auth.py):

```python
import threading
import time

# In-memory set of recently deactivated user IDs for immediate token invalidation.
# TTL matches access token expiry. After TTL, the DB is_active=False check suffices.
_deactivated_users: dict[UUID, float] = {}
_deactivation_lock = threading.Lock()

def mark_user_deactivated(user_id: UUID, ttl_seconds: int = 1800) -> None:
    """Add user to revocation set. TTL defaults to 30 min (token expiry)."""
    with _deactivation_lock:
        _deactivated_users[user_id] = time.time()
        # Cleanup expired entries
        cutoff = time.time() - ttl_seconds
        expired = [uid for uid, ts in _deactivated_users.items() if ts < cutoff]
        for uid in expired:
            del _deactivated_users[uid]

def clear_user_deactivation(user_id: UUID) -> None:
    """Remove user from revocation set (on reactivation)."""
    with _deactivation_lock:
        _deactivated_users.pop(user_id, None)

def is_user_deactivated(user_id: UUID) -> bool:
    """Check if user was recently deactivated."""
    return user_id in _deactivated_users
```

**3. Add service functions** (`backend/app/services/admin/users.py`):

- `deactivate_user(db, user_id)`:
  - Fetch user with `with_for_update()`. Raise ValueError if not found.
  - If already inactive, raise ValueError("User is already deactivated").
  - Set `user.is_active = False`. Flush.
  - Call `mark_user_deactivated(user_id)` from dependencies module for immediate token invalidation.
  - Return user.

- `activate_user(db, user_id)`:
  - Fetch user with `with_for_update()`. Raise ValueError if not found.

  - Call `clear_user_deactivation(user_id)` from dependencies module.
  - Return user.

  - Build reset link: `{frontend_base_url}/reset-password?token={raw_token}`.
  - Call `send_password_reset_email(user.email, user.first_name, reset_link)`.
  - Flush.
  - Return user email for confirmation.
