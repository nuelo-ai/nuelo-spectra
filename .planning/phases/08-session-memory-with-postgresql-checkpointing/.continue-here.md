<context>

**Mental state:**
We discovered a critical architectural issue during UAT testing. The current LangGraph implementation enforces a rigid pipeline (Coding → Checker → Execute → Analyst) for EVERY single query, which means:
- Simple questions like "What were the columns?" still trigger full code generation
- Follow-up queries like "Explain that result" regenerate code unnecessarily
- Memory exists but isn't being leveraged to skip unnecessary work

This is a **fundamental design flaw** that makes the system slow and expensive for conversational use cases. The user immediately recognized this and requested a Manager Agent approach - a routing layer that decides whether a query needs:
1. Just memory (fast, cheap)
2. Code modification (medium)
3. Fresh code generation (current behavior)

- Problem statement with current vs ideal behavior
