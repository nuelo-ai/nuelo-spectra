  <action>
Create `Dockerfile.backend` at the project root. The build context is the project root (`.`), so all paths are relative to root.

Add `# syntax=docker/dockerfile:1` at the top for BuildKit features.

**Stage 1: Builder (python:3.12-slim)**
- `FROM python:3.12-slim AS builder`
- `COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/` to get the uv binary
- Set env vars: `UV_LINK_MODE=copy` (no symlinks), `UV_COMPILE_BYTECODE=1` (faster startup)
- `WORKDIR /app` — MUST match runtime stage WORKDIR to avoid venv path mismatch
- Copy dependency files first for layer caching: `COPY backend/pyproject.toml backend/uv.lock ./`
- Install dependencies without project: `RUN --mount=type=cache,target=/root/.cache/uv uv sync --locked --no-install-project --no-editable`
- Copy backend source: `COPY backend/ .`
- Install project: `RUN --mount=type=cache,target=/root/.cache/uv uv sync --locked --no-editable`

**Stage 2: Runtime (python:3.12-slim)**
- `FROM python:3.12-slim`
- Install runtime system dependencies:
  ```
  RUN apt-get update && apt-get install -y --no-install-recommends \
      postgresql-client curl \
      && rm -rf /var/lib/apt/lists/*
  ```
  - `postgresql-client` provides `pg_isready` for the entrypoint script
  - `curl` is needed for the HEALTHCHECK command
- Create non-root user:
  ```
  RUN groupadd --gid 1001 appuser && \
      useradd --uid 1001 --gid appuser --shell /bin/bash --create-home appuser
  ```
- `WORKDIR /app` — MUST be `/app` to match builder stage
- Copy the built venv and source from builder: `COPY --from=builder --chown=appuser:appuser /app /app`
- Copy the entrypoint script: `COPY --chown=appuser:appuser backend/docker-entrypoint.sh /app/docker-entrypoint.sh`
- Make entrypoint executable: `RUN chmod +x /app/docker-entrypoint.sh`
- Declare uploads volume: `VOLUME ["/app/uploads"]`
- Switch to non-root: `USER appuser`
- Expose port: `EXPOSE 8000`
- Health check:
  ```
  HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
      CMD curl -f http://localhost:8000/health || exit 1
  ```
- Entrypoint: `ENTRYPOINT ["/app/docker-entrypoint.sh"]`

**Important notes per research:**
- Do NOT use `CMD` — the entrypoint script handles everything (pg_isready, alembic, exec uvicorn)
- The builder COPY copies `backend/` contents into `/app` (not `backend/app` into `/app/app`), so the source layout at `/app` is: `app/`, `alembic/`, `alembic.ini`, `pyproject.toml`, `uv.lock`, `.venv/`
- The second `COPY backend/docker-entrypoint.sh` is needed because the builder stage's `COPY backend/ .` already includes it, but we explicitly copy with correct ownership in runtime stage for clarity
