<task type="auto">
  <name>Task 1: Write Dockerfile.frontend for public Next.js app</name>
  <files>Dockerfile.frontend</files>
  <action>
Create `Dockerfile.frontend` at the project root. Build context is project root (`.`).

Add `# syntax=docker/dockerfile:1` at the top.

**Stage 1: deps (node:22-alpine)**
- `FROM node:22-alpine AS base` — shared base
- `FROM base AS deps`
- `RUN apk add --no-cache libc6-compat` — needed by some native Node modules
- `WORKDIR /app`
- Copy only dependency files for layer caching: `COPY frontend/package.json frontend/package-lock.json ./`
- `RUN npm ci` — clean install, production + dev deps (dev needed for build)

**Stage 2: builder (from base)**
- `FROM base AS builder`
- `WORKDIR /app`
- Copy node_modules from deps: `COPY --from=deps /app/node_modules ./node_modules`
- Copy frontend source: `COPY frontend/ .`
- Set BACKEND_URL as build arg with default:
  ```
  ARG BACKEND_URL=http://localhost:8000
  ENV BACKEND_URL=${BACKEND_URL}
  ```
  This is CRITICAL: next.config.ts rewrites are evaluated at build time, so BACKEND_URL must be available during `npm run build`. The ARG allows override via `--build-arg BACKEND_URL=http://backend:8000`.
- `RUN npm run build`

**Stage 3: runner (from base)**
- `FROM base AS runner`
- `WORKDIR /app`
- `ENV NODE_ENV=production`
- Create non-root user:
  ```
  RUN addgroup --system --gid 1001 nodejs && \
      adduser --system --uid 1001 nextjs
  ```
- Copy public assets: `COPY --from=builder /app/public ./public`
- Copy standalone output with ownership:
  ```
  COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
  COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static
  ```
  Both copies are required — standalone doesn't include static assets.
- Switch to non-root: `USER nextjs`
- Expose and configure:
  ```
  EXPOSE 3000
  ENV PORT=3000
  ENV HOSTNAME="0.0.0.0"
  ```
  HOSTNAME=0.0.0.0 is mandatory — without it, Next.js only listens on 127.0.0.1 and health checks from Docker network fail.
- Health check using wget (available in Alpine, no curl needed):
  ```
  HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
      CMD wget --no-verbose --tries=1 --spider http://localhost:3000/api/health || exit 1
  ```
- Start command: `CMD ["node", "server.js"]`
