Modify the `execute_in_sandbox` function in `backend/app/agents/graph.py` to extract an optional `chart` key from the JSON output alongside the existing `result` key. This is the output parser modification (INFRA-04).

**Location:** The success handler block starting at line 448 (`if result.success:`).

**Current behavior (lines 448-477):**
The parser iterates stdout lines in reverse, finds a line that looks like JSON, extracts `parsed["result"]`, and returns `{"execution_result": execution_result}`.

**Modified behavior:**
1. Initialize `chart_specs = ""` before the parsing loop
2. After extracting `parsed["result"]`, check for optional `parsed["chart"]`:
   ```python
   if "chart" in parsed:
       chart_specs = json.dumps(parsed["chart"])
       logger.info(f"Chart JSON extracted ({len(chart_specs)} bytes)")
   ```
3. Handle large chart JSON: Join all stdout lines before parsing as a fallback for charts that span multiple stdout list items. Try single-line parsing first (fast path), fall back to joined parsing if single-line fails and output looks like it contains chart data.
4. Add chart JSON size validation: If `len(chart_specs) > 2_000_000` (2MB), log a warning and set `chart_error` instead:
   ```python
   if len(chart_specs) > 2_000_000:
       logger.warning(f"Chart JSON too large ({len(chart_specs)} bytes), discarding")
       chart_specs = ""
       chart_error = "Chart data too large. Try aggregating data before charting."
   ```
5. Return the extended dict:
   ```python
   return {
       "execution_result": execution_result,
       "chart_specs": chart_specs,
       "chart_error": chart_error,
   }
   ```

**Backward compatibility:** When stdout contains `{"result": ...}` without a `chart` key (all existing code), `chart_specs` remains `""` and `chart_error` remains `""`. Existing behavior is 100% preserved.

**Error path extension:** In the execution failure branch (the `else` block after line 478), also reset chart fields to prevent stale data:
```python
# In the retry Command update dict:
"chart_specs": "",
"chart_error": "",
```

And in the halt Command update dict:
```python
"chart_specs": "",
"chart_error": "",
```

**Important:** Do NOT modify the file loading code, the multi-file/single-file branching, or the E2B runtime call. Only modify the result handling after `if result.success:` and the error handling after `else:`.
