       - Add module-level cooldown dict: `_reset_cooldowns: dict[str, float] = {}` (maps email to last request timestamp)
       - Rewrite `forgot_password()`:
         a. **Cooldown check first** (before DB query): Look up `request.email` in `_reset_cooldowns`. If found and `time.time() - last_request < 120`, return the standard 202 response immediately (do NOT raise an error -- anti-enumeration). Clean up expired cooldown entries opportunistically (entries older than 120 seconds).
         b. **Query user** by email (existing code)
         c. If user exists:
            - **Invalidate all previous tokens**: `UPDATE password_reset_tokens SET is_active = False WHERE email = :email AND is_active = True` (use SQLAlchemy: `await db.execute(update(PasswordResetToken).where(PasswordResetToken.email == request.email, PasswordResetToken.is_active == True).values(is_active=False))`)
            - **Generate new token**: Call `create_reset_token()` from email service to get `(raw_token, token_hash)`
            - **Store token in DB**: Create `PasswordResetToken(email=request.email, token_hash=token_hash, expires_at=datetime.now(timezone.utc) + timedelta(minutes=10))`
            - **Build reset link**: `f"{settings.frontend_url}/reset-password?token={raw_token}"`
            - **Send email**: `await send_password_reset_email(user.email, user.first_name, reset_link, settings)` -- note passing `user.first_name` for personalized greeting
            - **Record cooldown**: `_reset_cooldowns[request.email] = time.time()`
         d. ALWAYS return same 202 response (anti-enumeration per locked decision): `MessageResponse(message="If the email exists, a reset link has been sent")`

       - Rewrite `reset_password()`:
         a. Remove JWT token verification (`verify_password_reset_token` call)
         b. **Hash incoming token**: `token_hash = hashlib.sha256(request.token.encode()).hexdigest()` (add `import hashlib`)
         c. **Look up token in DB**: Query `PasswordResetToken` where `token_hash` matches AND `is_active == True` AND `is_used == False` AND `expires_at > datetime.now(timezone.utc)`
         d. If no valid token found: raise `HTTPException(400, "Invalid or expired reset token")`
         e. **Look up user by token's email**: Query User where email matches token.email
         f. If no user: raise `HTTPException(400, "Invalid or expired reset token")` (generic message)
         g. **Mark token as used**: `token_record.is_used = True`
         h. **Update password**: `user.hashed_password = hash_password(request.new_password)`
         i. **Commit**: `await db.commit()`
         j. Return `MessageResponse(message="Password reset successful")`

    3. **Important anti-patterns to avoid:**
       - Do NOT use `from sqlalchemy import select` inside function body (put in top-level imports)
       - Do NOT log the reset link or token in production logs (metadata only)
       - Do NOT return different error messages for "token expired" vs "token invalid" vs "token used" -- always generic "Invalid or expired reset token" to prevent information leakage
