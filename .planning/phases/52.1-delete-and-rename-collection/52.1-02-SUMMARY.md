## Decisions Made

- useUpdateCollection invalidates both list and detail caches since the collection name/description changes affect both views
- useDeleteCollection invalidates only the list cache since the detail no longer exists
- useEffect on `[open, collection]` syncs form state on every open to correctly handle the case where a user renames one collection then immediately opens the dialog for another
- Edit mode after PATCH calls `onOpenChange(false)` without navigation; create mode still navigates to the new collection detail page
- `onSuccess` callback in DeleteCollectionDialog is optional — detail page uses it for navigation, list page omits it since cache invalidation auto-removes the card
