  <action>
    Create `backend/app/services/chat_session.py` with a `ChatSessionService` class following the same static method pattern as existing services (ChatService, FileService):

    1. **create_session(db, user_id, title)** -> ChatSession
       - Create ChatSession with user_id and title
       - db.add(), await db.commit(), await db.refresh()
       - Return session

    2. **list_user_sessions(db, user_id, limit=50, offset=0)** -> tuple[list[ChatSession], int]
       - SELECT from chat_sessions WHERE user_id, ordered by updated_at DESC (most recent first)
       - Include eager loading of files relationship using `selectinload(ChatSession.files)` for file_count
       - Paginate with limit/offset
       - Return (sessions, total_count)

    3. **get_user_session(db, session_id, user_id)** -> ChatSession | None
       - SELECT single session by id AND user_id (ownership check)
       - Eager load files relationship using `selectinload(ChatSession.files)`
       - Return session or None

    4. **update_session(db, session_id, user_id, title)** -> ChatSession | None
       - Get session (with ownership check)
       - Update title if provided
       - await db.commit(), await db.refresh()
       - Return updated session or None if not found

    5. **delete_session(db, session_id, user_id)** -> bool
       - Get session (with ownership check)
       - await db.delete(session) - cascade will delete messages (per model config)
       - await db.commit()
       - Return True if deleted, False if not found

    6. **link_file_to_session(db, session_id, file_id, user_id)** -> ChatSession
       - Get session (ownership check)
       - Get file (ownership check - file must belong to same user)
       - Check file not already linked (avoid duplicate association rows)
       - Check file count limit: if len(session.files) >= 10, raise ValueError("Maximum 10 files per session")
       - Append file to session.files (SQLAlchemy auto-manages association table)
       - Update session.updated_at
       - await db.commit(), await db.refresh()
       - Return session

    7. **unlink_file_from_session(db, session_id, file_id, user_id)** -> ChatSession
       - Get session (ownership check)
       - Remove file from session.files list
       - Update session.updated_at
       - await db.commit(), await db.refresh()
       - Return session
       - NOTE: This does NOT delete messages (DATA-06) - messages stay with session

    8. **get_session_files(db, session_id, user_id)** -> list[File]
       - Get session with files eagerly loaded
       - Return session.files

    Import requirements: `from sqlalchemy import func, select`, `from sqlalchemy.orm import selectinload`, `from sqlalchemy.ext.asyncio import AsyncSession`, model imports.

  <action>
    Create `backend/app/routers/chat_sessions.py` with APIRouter:

    ```python
    router = APIRouter(prefix="/sessions", tags=["Chat Sessions"])
    ```

    Endpoints:

    1. **POST /** -> ChatSessionResponse (201)
       - Body: ChatSessionCreate
       - Creates new session for current_user
       - Returns created session

    2. **GET /** -> ChatSessionList
       - Query params: limit (default 50), offset (default 0)
       - Lists current_user's sessions ordered by updated_at DESC
       - Returns list with total count

    3. **GET /{session_id}** -> ChatSessionDetail
       - Returns session with linked files (detail view)
       - 404 if not found or not owned by user

    4. **PATCH /{session_id}** -> ChatSessionResponse
       - Body: ChatSessionUpdate
       - Updates session title
       - 404 if not found

    5. **DELETE /{session_id}** -> {"message": "Session deleted"}
       - Deletes session and its messages (cascade)
       - 404 if not found
       - Returns 200 with confirmation message

    6. **POST /{session_id}/files** -> ChatSessionDetail (201)
       - Body: ChatSessionFileLink (contains file_id)
       - Links file to session (many-to-many)
       - 404 if session or file not found
       - 400 if file already linked
       - 400 if file limit (10) exceeded
       - Returns updated session detail with files

    7. **DELETE /{session_id}/files/{file_id}** -> ChatSessionDetail
       - Unlinks file from session
       - 404 if session not found or file not linked
       - Returns updated session detail

    8. **GET /{session_id}/messages** -> ChatMessageList
       - Query params: limit (default 50), offset (default 0)
       - Lists messages for session (uses ChatService.list_session_messages)
       - 404 if session not found

    For each endpoint:
    - Use `CurrentUser` and `DbSession` dependencies (from app.dependencies)
    - Follow existing router patterns in chat.py and files.py
    - Return proper HTTP status codes
    - Include docstrings
    - Handle errors with HTTPException

    For the ChatSessionDetail response, manually construct it to include file_count:
    ```python
    session = await ChatSessionService.get_user_session(db, session_id, current_user.id)
    return ChatSessionDetail(
        **{k: v for k, v in session.__dict__.items() if not k.startswith('_')},
        files=[FileBasicInfo.model_validate(f) for f in session.files],
        file_count=len(session.files)
    )
    ```
