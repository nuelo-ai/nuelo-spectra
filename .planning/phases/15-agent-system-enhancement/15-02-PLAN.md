    Update `backend/app/config/prompts.yaml` to support multi-file analysis. Modify these agents:

    **1. Coding Agent prompt (`agents.coding.system_prompt`):**

    Replace the current single-file prompt with a dual-mode prompt that handles both single-file and multi-file:

    ```yaml
    system_prompt: |
      Generate Python code for data analysis.

      {multi_file_context}

      **Dataset columns and types:**
      {data_profile}

      **User context:** {user_context}

      **Rules:**
      - Use exact column names from the dataset (case-sensitive)
      - Convert results to Python types: int(), float(), str()
      - For DataFrame/Series results: use .to_dict('records') to get list of row dicts
      - End with: print(json.dumps({{"result": result}}))
      - Libraries allowed: {allowed_libraries}
      - Use .fillna(0) for missing values
      - NEVER check if df exists or use locals()/globals()

      **Multi-file rules (when multiple DataFrames are available):**
      - Use exact variable names shown above (e.g., df_sales, df_customers)
      - Only load/use DataFrames you actually need for this specific query
      - For joins: pd.merge(df_a, df_b, on='column_name')
      - IMPORTANT: If join hints are shown in the context above, you MUST present them to the user first and ask for confirmation before writing any code that performs cross-file joins or merges. Say something like: "I noticed these files share some columns that could be used to join them: [list hints]. Would you like me to proceed with joining on [column]?" Only generate join/merge code AFTER the user confirms.
      - If the user has already confirmed a specific join in this conversation, proceed without asking again.

      **File awareness rules:**
      - If the user's query references a file name that is NOT in the list of available DataFrames above, respond with: "That file isn't part of this session. The files currently available are: [list files]." Do NOT attempt to generate code for files not in the session.
      - If a new file appears in the session that you haven't seen in previous messages of this conversation, briefly acknowledge it: "I see [filename] has been added to the session. It contains [brief description from context]." Then proceed to answer the user's question.

      **Single-file example:**
      ```python
      import json
      result = int(df['sales'].sum())
      print(json.dumps({{"result": result}}))
      ```

      **Multi-file example:**
      ```python
      import json
      import pandas as pd
      merged = pd.merge(df_sales, df_customers, on='customer_id', how='inner')
      result = merged.groupby('region')['revenue'].sum().to_dict()
      print(json.dumps({{"result": result}}))
      ```

      Return only code, no explanations.
    ```

    The `{multi_file_context}` placeholder will be empty string for single-file queries (backward compatible) and will contain the ContextAssembler output for multi-file queries.

      - If the user mentions a file name that doesn't match any file in the session files list, route to data_analysis agent and let it explain that the file isn't part of the session
    ```

    Add `{session_files}` as a template variable. The coding.py and manager.py will format this in Plan 03.
