  <action>
    **1. Create TypeScript types (frontend/src/types/session.ts):**
    Mirror backend schemas from `backend/app/schemas/chat_session.py`. Create:
    - `ChatSessionResponse` — id (string), user_id (string), title (string), file_count (number), created_at (string), updated_at (string)
    - `ChatSessionDetail` — extends ChatSessionResponse with files: FileBasicInfo[]
    - `ChatSessionList` — sessions: ChatSessionResponse[], total: number
    - `FileBasicInfo` — id (string), original_filename (string), file_type (string), row_count (number | null), column_count (number | null)
    - `ChatSessionCreate` — title: string
    - `ChatSessionUpdate` — title: string

    **2. Create session store (frontend/src/stores/sessionStore.ts):**
    Zustand store replacing tabStore with session-centric state:
    - `currentSessionId: string | null` — currently active session
    - `leftSidebarOpen: boolean` — default true
    - `rightPanelOpen: boolean` — default false (per user decision: default closed)
    - `setCurrentSession(id: string | null)` — set active session
    - `toggleLeftSidebar()` — toggle left sidebar
    - `toggleRightPanel()` — toggle right panel
    - `setRightPanelOpen(open: boolean)` — explicit setter
    Follow same Zustand `create` pattern as tabStore.ts.

    **3. Create session query hooks (frontend/src/hooks/useChatSessions.ts):**
    - `useChatSessions()` — useQuery fetching `GET /sessions?limit=50&offset=0`, returns sessions array. queryKey: ['sessions']. staleTime: 5 minutes.
    - `useSessionDetail(sessionId: string | null)` — useQuery fetching `GET /sessions/${sessionId}`, returns ChatSessionDetail. queryKey: ['sessions', sessionId]. enabled: !!sessionId.
    Both use apiClient.get() for auth-aware fetching.

    **4. Create session mutation hooks (frontend/src/hooks/useSessionMutations.ts):**
    - `useCreateSession()` — useMutation calling `POST /sessions` with { title }. onSuccess: invalidate ['sessions']. Returns created session.
    - `useUpdateSession()` — useMutation calling `PATCH /sessions/${sessionId}` with { title }. Include optimistic update: update ['sessions'] cache immediately, rollback on error. onSettled: invalidate ['sessions'].
    - `useDeleteSession()` — useMutation calling `DELETE /sessions/${sessionId}`. onSuccess: invalidate ['sessions'].
    - `useLinkFile()` — useMutation calling `POST /sessions/${sessionId}/files` with { file_id }. onSuccess: invalidate both ['sessions'] and ['sessions', sessionId].
    - `useUnlinkFile()` — useMutation calling `DELETE /sessions/${sessionId}/files/${fileId}`. onSuccess: invalidate both ['sessions'] and ['sessions', sessionId].
    All mutations use apiClient methods and useQueryClient for cache invalidation.
  </action>
