    **Create `backend/app/routers/files.py`:**

    Follow the exact pattern from routers/auth.py (APIRouter with prefix and tags, typed dependencies).

    ```python
    router = APIRouter(prefix="/files", tags=["Files"])
    ```

    Endpoints:

    1. `POST /files/upload` (status 201, response_model=FileUploadResponse):
       - Parameters: `file: UploadFile`, `current_user: CurrentUser`, `db: DbSession`
       - Validate file extension from original filename: must be in {".csv", ".xlsx", ".xls"}
       - If invalid extension, raise HTTPException 400 with message listing allowed formats
       - Call `FileService.upload_file(db, current_user.id, file, ext)`
       - Catch `ValueError` from pandas validation and re-raise as HTTPException 400 with the error message
       - Return FileUploadResponse.model_validate(file_record)

    2. `GET /files/` (response_model=list[FileListItem]):
       - Parameters: `current_user: CurrentUser`, `db: DbSession`
       - Call `FileService.list_user_files(db, current_user.id)`
       - Return list of FileListItem.model_validate(f) for each file

    3. `GET /files/{file_id}` (response_model=FileDetailResponse):
       - Parameters: `file_id: UUID`, `current_user: CurrentUser`, `db: DbSession`
       - Call `FileService.get_user_file(db, file_id, current_user.id)`
       - If None, raise HTTPException 404 "File not found"
       - Return FileDetailResponse.model_validate(file)

    4. `DELETE /files/{file_id}` (status 204, no response body):
       - Parameters: `file_id: UUID`, `current_user: CurrentUser`, `db: DbSession`
       - Call `FileService.delete_file(db, file_id, current_user.id)`
       - If returns False, raise HTTPException 404 "File not found"
       - Return None (204 No Content)

    Imports: `from fastapi import APIRouter, UploadFile, HTTPException, status`, `from uuid import UUID`, `from pathlib import Path`, `from app.dependencies import CurrentUser, DbSession`, `from app.schemas.file import FileUploadResponse, FileListItem, FileDetailResponse`, `from app.services.file import FileService`

    Define ALLOWED_EXTENSIONS = {".csv", ".xlsx", ".xls"} at module level.

    **Create `backend/app/schemas/chat.py`:**

    Follow existing pattern from schemas/auth.py.

    - `ChatMessageCreate(BaseModel)`: content (str), role (str) with default "user". Validate role is in {"user", "assistant"}.
    - `ChatMessageResponse(BaseModel)`: id (UUID), file_id (UUID), role (str), content (str), message_type (str | None), metadata_json (dict | None), created_at (datetime). ConfigDict(from_attributes=True).
    - `ChatMessageList(BaseModel)`: messages (list[ChatMessageResponse]), total (int).

    **Create `backend/app/services/chat.py`:**

    Follow existing pattern from services/auth.py.

    `ChatService` class with static methods:

    1. `list_file_messages(db: AsyncSession, file_id: UUID, user_id: UUID, limit: int = 50, offset: int = 0) -> tuple[list[ChatMessage], int]`:
       - Query ChatMessage where file_id matches AND user_id matches (double isolation)
       - Order by created_at ascending (oldest first, chat history order)
       - Apply limit and offset for pagination
       - Also query count for total
       - Return (messages, total_count)

    2. `create_message(db: AsyncSession, file_id: UUID, user_id: UUID, content: str, role: str = "user", message_type: str | None = None, metadata_json: dict | None = None) -> ChatMessage`:
       - Create ChatMessage instance with all fields
       - db.add(), await db.commit(), await db.refresh()
       - Return the ChatMessage record

    Imports: `from sqlalchemy import select, func`, `from sqlalchemy.ext.asyncio import AsyncSession`, `from app.models.chat_message import ChatMessage`, `from uuid import UUID`

    **Create `backend/app/routers/chat.py`:**

    ```python
    router = APIRouter(prefix="/chat", tags=["Chat"])
    ```

    Endpoints:

    1. `GET /chat/{file_id}/messages` (response_model=ChatMessageList):
       - Parameters: `file_id: UUID`, `current_user: CurrentUser`, `db: DbSession`, `limit: int = 50`, `offset: int = 0`
       - First verify file ownership: call `FileService.get_user_file(db, file_id, current_user.id)`. If None, raise 404 "File not found"
       - Call `ChatService.list_file_messages(db, file_id, current_user.id, limit, offset)`
       - Return ChatMessageList with messages and total

    2. `POST /chat/{file_id}/messages` (status 201, response_model=ChatMessageResponse):
       - Parameters: `file_id: UUID`, `body: ChatMessageCreate`, `current_user: CurrentUser`, `db: DbSession`
       - First verify file ownership: call `FileService.get_user_file(db, file_id, current_user.id)`. If None, raise 404 "File not found"
       - Call `ChatService.create_message(db, file_id, current_user.id, body.content, body.role)`
       - Return ChatMessageResponse.model_validate(message)

    IMPORTANT: Both chat endpoints MUST verify file ownership via FileService.get_user_file before operating. This prevents user A from reading/writing chat messages on user B's files even if they guess the file_id.
