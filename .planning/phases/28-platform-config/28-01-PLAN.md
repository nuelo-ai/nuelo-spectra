**1. Create `backend/app/services/platform_settings.py`:**

Module-level TTL cache service (same pattern as `user_class.py`). Must contain:

- `_settings_cache: dict[str, str] | None = None` and `_cache_loaded_at: float = 0.0` and `_CACHE_TTL_SECONDS: float = 30.0`
- `DEFAULTS` dict with JSON-encoded defaults for all known keys:
  - `"allow_public_signup"`: `json.dumps(True)`
  - `"default_user_class"`: `json.dumps("free")`
  - `"invite_expiry_days"`: `json.dumps(7)`
  - `"credit_reset_policy"`: `json.dumps("weekly")`
  - `"default_credit_cost"`: `json.dumps("1.0")` (string representation of Decimal to avoid float precision issues)
- `VALID_KEYS = set(DEFAULTS.keys())`
- `async def get_all(db: AsyncSession) -> dict[str, str]` -- load all settings with 30s TTL. Query `select(PlatformSetting)`, merge with DEFAULTS for missing keys.
- `async def get(db: AsyncSession, key: str) -> Any` -- get single parsed setting. Calls `get_all()`, returns `json.loads(raw)`.
- `async def upsert(db: AsyncSession, key: str, value: Any, admin_id: UUID) -> None` -- upsert single key. Use `select(...).where(PlatformSetting.key == key)` then update or create. Store `json.dumps(value)`.
- `def invalidate_cache() -> None` -- reset `_settings_cache` to None and `_cache_loaded_at` to 0.0.
- `def validate_setting(key: str, value: Any) -> str | None` -- return error message string if invalid, None if valid. Validation rules:
  - `allow_public_signup`: must be `bool`
  - `default_user_class`: must be `str` and must exist in `get_user_classes().keys()` (import from user_class)
  - `invite_expiry_days`: must be `int`, 1 <= value <= 365
  - `credit_reset_policy`: must be `str` in `{"manual", "weekly", "monthly"}`
  - `default_credit_cost`: must be `(int | float)`, value > 0
  - Unknown key: return "Unknown setting key: {key}"

**2. Create `backend/app/schemas/platform_settings.py`:**

- `class SettingsResponse(BaseModel)`: fields matching each setting key with proper types:
  - `allow_public_signup: bool`
  - `default_user_class: str`
  - `invite_expiry_days: int`
  - `credit_reset_policy: str`
  - `default_credit_cost: float`
- `class SettingsUpdateRequest(BaseModel)`: all fields Optional, with `model_config = ConfigDict(extra="forbid")`:
  - `allow_public_signup: bool | None = None`
  - `default_user_class: str | None = None`
  - `invite_expiry_days: int | None = None`
  - `credit_reset_policy: str | None = None`
  - `default_credit_cost: float | None = None`
  - Add `model_post_init` validator: at least one field must be non-None (raise ValueError otherwise)

**3. Create `backend/app/routers/admin/settings.py`:**

Router with `prefix="/settings"`, `tags=["admin-settings"]`.

- `GET /settings` -> `SettingsResponse`:
  - Use `CurrentAdmin` and `DbSession` dependencies
  - Call `platform_settings.get_all(db)`, parse each value with `json.loads()`, return as `SettingsResponse`

- `PATCH /settings` -> `SettingsResponse`:
  - Accept `SettingsUpdateRequest` body, `CurrentAdmin`, `DbSession`, `Request` (for audit IP)
  - Iterate over `body.model_dump(exclude_none=True).items()`
  - For each key/value: call `platform_settings.validate_setting(key, value)` -- if error, raise `HTTPException(422, detail=error_msg)`
  - For each key/value: call `platform_settings.upsert(db, key, value, current_admin.id)`
  - Call `platform_settings.invalidate_cache()` AFTER all upserts
  - Create audit log entry via `log_admin_action(db, admin_id=current_admin.id, action="update_settings", target_type="platform_settings", target_id="global", details={"changed_keys": list(changed.keys())}, ip_address=client_ip)`
  - `await db.commit()`
  - Re-read and return all settings as `SettingsResponse`

NOTE on SETTINGS-06 (credit overrides) and TIER-02 (admin edit tier credits): These are DEFERRED per user decision. Do NOT implement `tier_credit_overrides` key. Tier credit allocations are read-only from `user_classes.yaml`.

NOTE on SETTINGS-05 (credit_reset_policy): Store it in platform_settings as a global value. The scheduler continues reading per-tier reset_policy from yaml. This setting is informational for now -- it does NOT override per-tier yaml values.
