**Step 2: Create `frontend/src/components/chart/ChartRenderer.tsx`**

This is a 'use client' component that renders Plotly charts. Key requirements:
- Import Plotly from 'plotly.js-dist-min' (this component will be dynamically imported with ssr:false in Plan 02, so direct import is correct HERE)
- Accept props: `data: string` (JSON string from backend fig.to_json()), optional `height?: number`
- Use `useRef<HTMLDivElement>` for the chart container
- Use `useEffect` with `[data]` dependency:
  - Parse `data` with JSON.parse in try/catch
  - Calculate dynamic height using a `calculateChartHeight` helper function:
    - Base height: 400px
    - Scale: +10px per 100 data points across all traces (sum trace.x.length or trace.y.length or trace.values.length)
    - Cap at 700px max
    - Use backend layout.height if explicitly set (override dynamic calc)
  - Call `Plotly.react(chartRef.current, chartData.data || [], layout, config)` where:
    - layout merges chartData.layout with: `autosize: true`, calculated height, `margin: { l: 50, r: 30, t: 40, b: 50 }`
    - config: `{ responsive: true, displayModeBar: true, displaylogo: false, modeBarButtonsToRemove: ['sendDataToCloud'] }`
  - On parse/render error: set error state, log to console
- Set up ResizeObserver in the SAME useEffect:
  - Observe chartRef.current
  - On resize: call `Plotly.Plots.resize(chartRef.current)` (no manual debouncing needed per research)
- Cleanup function: `resizeObserver.disconnect()` AND `Plotly.purge(chartRef.current)` (CRITICAL for memory leak prevention)
- Render: `<div ref={chartRef} className="w-full" />` (no hardcoded height on container -- Plotly manages it)
- If error state: render a centered muted error message inside a min-height container
- Export as default export (required for next/dynamic)
