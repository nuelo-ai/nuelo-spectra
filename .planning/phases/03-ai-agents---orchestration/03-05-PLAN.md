    3. Add run_chat_query to backend/app/services/agent_service.py:

       **async def run_chat_query(db: AsyncSession, file_id: UUID, user_id: UUID, user_query: str) -> dict:**
       - Load file record (FileService.get_user_file)
       - If file not found, raise HTTPException 404
       - If file.data_summary is None, raise HTTPException 400 with message "File has not been analyzed yet. Please wait for onboarding to complete."
       - Get or create the compiled graph via get_or_create_graph()
       - Build thread_id: f"file_{file_id}_user_{user_id}"
       - Build config: {"configurable": {"thread_id": thread_id}}
       - Build initial state:
         {
           "file_id": str(file_id),
           "user_id": str(user_id),
           "user_query": user_query,
           "data_summary": file.data_summary,
           "user_context": file.user_context or "",
           "generated_code": "",
           "validation_result": "",
           "validation_errors": [],
           "error_count": 0,
           "max_steps": settings.agent_max_retries,
           "execution_result": "",
           "analysis": "",
           "messages": [],
           "final_response": "",
           "error": ""
         }
       - Invoke graph: result = await graph.ainvoke(initial_state, config)
       - Save user message to chat_messages via ChatService.create_message(role="user", content=user_query)
       - Save assistant response to chat_messages via ChatService.create_message(
           role="assistant",
           content=result.get("final_response", result.get("analysis", "")),
           message_type="agent_response",
           metadata_json={
             "generated_code": result.get("generated_code"),
             "execution_result": result.get("execution_result"),
             "error_count": result.get("error_count", 0),
             "error": result.get("error")
           }
         )
       - Return dict with: user_query, generated_code, execution_result, analysis (or final_response), error, retry_count

       This function bridges the HTTP layer and the LangGraph workflow.
       Thread ID format ensures per-file per-user memory isolation (CONTEXT.md: memory at chat tab level).
       data_summary from Onboarding is passed into the graph state so Coding Agent has context.
  </action>
