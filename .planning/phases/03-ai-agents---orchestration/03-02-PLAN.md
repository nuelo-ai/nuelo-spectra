<task type="auto">
  <name>Task 2: Build Onboarding Agent and agent service layer</name>
  <files>
    backend/app/agents/onboarding.py

  <action>
    1. Create backend/app/agents/onboarding.py with class OnboardingAgent:

       The Onboarding Agent has two phases:
       a) Data profiling (pure Python/pandas, no LLM needed)
       b) LLM summarization (takes profile + user context, generates natural language summary)

       **profile_data(file_path: str, file_type: str) -> dict:**
       - Read the file with pandas (pd.read_csv or pd.read_excel)
       - Generate profile dict containing:
         - shape: {rows: int, columns: int}
         - columns: {col_name: {dtype, missing_count, missing_pct, unique_count, ...numeric stats}}
         - sample_data: first 5 rows as list of dicts (df.head(5).to_dict(orient='records'))
         - quality_issues: list of strings (duplicates, high missing %, etc.)
       - Run in thread pool (asyncio.to_thread) since pandas is blocking
       - Pattern from research: use df.describe(), df.dtypes, df.isna().sum()
       - Handle errors gracefully: if profiling fails, return minimal profile with error

       **generate_summary(profile: dict, user_context: str) -> str:**
       - Initialize ChatAnthropic with model from settings
       - Load system prompt from YAML config (get_agent_prompt("onboarding"))
       - Build user message with: profile JSON + user_context (if provided)
       - Invoke LLM with system prompt + user message
       - Return the content string from LLM response
       - Use max_tokens from prompts.yaml config (1500)

       **run(file_path: str, file_type: str, user_context: str = "") -> OnboardingResult:**
       - Call profile_data to get dict
       - Call generate_summary with profile and user_context
       - Return OnboardingResult dataclass with: data_summary (str), data_profile (dict), sample_data (str)

       Create OnboardingResult as a simple dataclass in the same file.

       IMPORTANT: Use langchain_anthropic.ChatAnthropic (not langchain_openai).
       Initialize with: ChatAnthropic(model=settings.agent_model, api_key=settings.anthropic_api_key, max_tokens=max_tokens)

    2. Create backend/app/services/agent_service.py:

       This is the service layer that bridges FastAPI endpoints with agent logic.

       **async run_onboarding(db: AsyncSession, file_id: UUID, user_id: UUID, user_context: str = "") -> str:**
       - Load file record from database (using FileService.get_user_file)
       - If file not found, raise HTTPException 404
       - Instantiate OnboardingAgent
       - Call agent.run(file_path=file.file_path, file_type=file.file_type, user_context=user_context)
       - Update file record: file.data_summary = result.data_summary, file.user_context = user_context
       - Commit to database
       - Return the data_summary string

       **async update_user_context(db: AsyncSession, file_id: UUID, user_id: UUID, additional_context: str) -> str:**
       - Load file record
       - Append additional_context to existing file.user_context (with newline separator)
       - Commit to database
       - Return updated user_context
       - Note: v1 always appends (per CONTEXT.md deferred decision)

       Import settings with get_settings(). Use asyncio.to_thread for pandas blocking calls within the OnboardingAgent.

       Set LangSmith env vars at module level (only if settings.langsmith_tracing is True):
       if settings.langsmith_tracing and settings.langsmith_api_key:
           os.environ["LANGSMITH_TRACING"] = "true"
           os.environ["LANGSMITH_API_KEY"] = settings.langsmith_api_key
           os.environ["LANGSMITH_PROJECT"] = settings.langsmith_project
