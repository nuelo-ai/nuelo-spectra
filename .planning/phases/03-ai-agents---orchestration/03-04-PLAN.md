    1. Create backend/app/agents/coding.py:

       **async def coding_agent(state: ChatAgentState) -> dict:**
       - Load system prompt from YAML config via get_agent_prompt("coding")
       - Format prompt with state data:
         - {data_summary} from state["data_summary"]
         - {user_context} from state["user_context"]
         - {allowed_libraries} from get_allowed_libraries() joined as comma-separated string
       - If state has validation_errors (retry scenario), append to user message:
         "Previous code had issues:\n" + "\n".join(validation_errors) + "\nPlease regenerate the code fixing these issues."
       - Initialize ChatAnthropic(model=settings.agent_model, api_key=settings.anthropic_api_key, max_tokens=max_tokens_from_config)
       - Build messages list: [SystemMessage(content=formatted_prompt), HumanMessage(content=state["user_query"])]
       - If retry: append the error context to the HumanMessage
       - Invoke LLM: response = await llm.ainvoke(messages)
       - Extract code from response: parse code block from response.content
         - Try to extract content between ```python and ``` markers
         - If no markers found, use entire response as code
       - Return {"generated_code": extracted_code}

       Helper function: extract_code_block(text: str) -> str
       - Use regex to find ```python\n...\n``` block
       - Fallback: return text stripped of any markdown formatting

    2. Create backend/app/agents/data_analysis.py:

       **async def data_analysis_agent(state: ChatAgentState) -> dict:**
       - Load system prompt from YAML config via get_agent_prompt("data_analysis")
       - Format prompt with:
         - {user_query} from state["user_query"]
         - {executed_code} from state["generated_code"]
         - {execution_result} from state["execution_result"]
       - Initialize ChatAnthropic with settings
       - Build messages: [SystemMessage(content=formatted_prompt), HumanMessage(content="Please analyze these results.")]
       - Invoke LLM
       - Return {"analysis": response.content, "final_response": response.content}

       Both agents follow the same pattern:
       - Import get_settings, get_agent_prompt from config
       - Import ChatAnthropic from langchain_anthropic
       - Import SystemMessage, HumanMessage from langchain_core.messages
       - All LLM calls use await llm.ainvoke() (async)
       - All prompts loaded from YAML (not hardcoded)

    Create backend/app/agents/graph.py that assembles the complete chat analysis pipeline:

    1. **Code Checker node function (wraps AST + LLM validation):**

       async def code_checker_node(state: ChatAgentState) -> Command[Literal["execute", "coding_agent", "halt"]]:
       - First: Run AST validation using validate_code() from code_checker.py
       - If AST validation fails: increment error_count, route to "coding_agent" (retry)
       - If AST validation passes: Run LLM logical check using code_checker system prompt
         - Load system prompt from YAML, format with {user_query}, {generated_code}
         - Invoke ChatAnthropic
         - Parse response: if starts with "VALID", route to "execute"
         - If starts with "INVALID", extract issues, increment error_count, route to "coding_agent"
       - Circuit breaker: if error_count >= max_steps, route to "halt" instead of retry
       - Return Command(goto=target, update={...state updates...})

       Use langgraph.types.Command for routing (per research Pattern 1).

    2. **Execute stub node:**

       async def execute_code_stub(state: ChatAgentState) -> dict:
       - This is a STUB for Phase 3. Real sandbox execution comes in Phase 5.
       - For now: execute code using exec() in a restricted local namespace
       - Create namespace with: {"pd": pd, "np": np, "result": None}
       - exec(state["generated_code"], {"__builtins__": {}}, namespace)
       - Return {"execution_result": str(namespace.get("result", "No result variable found"))}
       - Wrap in try/except: on error, return {"execution_result": f"Error: {str(e)}", "error": str(e)}
       - Add a comment: # TODO Phase 5: Replace with E2B sandbox execution
       - IMPORTANT: This stub is for development only. The AST validation from Code Checker
         provides the safety layer. Phase 5 adds OS-level isolation.

    3. **Halt node:**

       async def halt_node(state: ChatAgentState) -> dict:
       - Return {"final_response": f"I was unable to generate valid code after {state['max_steps']} attempts. Errors: {', '.join(state.get('validation_errors', []))}. Please try rephrasing your question.", "error": "max_retries_exceeded"}

    4. **build_chat_graph(postgres_url: str) -> CompiledGraph:**
       - Initialize PostgresSaver checkpointer from postgres_url
       - Create StateGraph(ChatAgentState)
       - Add nodes:
         - "coding_agent" -> coding_agent function
         - "code_checker" -> code_checker_node function
         - "execute" -> execute_code_stub function
         - "data_analysis" -> data_analysis_agent function
         - "halt" -> halt_node function
       - Add edges:
         - set_entry_point("coding_agent")
         - add_edge("coding_agent", "code_checker")
         - code_checker uses Command return for conditional routing (no add_conditional_edges needed)
         - add_edge("execute", "data_analysis")
         - set_finish_point("data_analysis") AND set_finish_point("halt")
       - Compile with checkpointer
       - Return compiled graph

       Note on Command routing: When a node returns Command[Literal[...]], LangGraph
       automatically handles the routing. No need for add_conditional_edges.
       The code_checker_node returns Command(goto="execute"|"coding_agent"|"halt").

    5. **get_or_create_graph() -> CompiledGraph:**
       - Module-level cached graph instance
       - Lazy initialization: build on first call, return cached thereafter
       - Gets postgres_url from settings.database_url
       - Calls build_chat_graph(postgres_url)
