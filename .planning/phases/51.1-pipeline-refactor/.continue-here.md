<context>
The core issue this session uncovered: the original 51.1 refactor had the Coder agent generating both statistical analysis AND Plotly charts in a single sandbox script. Any chart error (wrong column names, deprecated Plotly API) killed the entire signal — even though the analysis was valid. This caused most hypotheses to be rejected (1/7, 4/8 validated in test runs).

The fix was architectural: separate concerns so Coder does analysis-only, Viz agent does charts-only. The viz agent also needed the deep_profile (column schema) to avoid hallucinating column names.

After the structural fix, signal validation improved dramatically (6/8 validated). Remaining viz failures (2/6 no chart) should be addressed by the retry mechanism that was added but not yet tested.

The owner wants all prompts in prompts.yaml (not hardcoded), and emphasized that `extract_code_block()` already strips markdown fences but added the no-markdown directive as a safeguard.

Database was cleared at start of session for clean testing. Alembic migration was manually applied.
</context>
