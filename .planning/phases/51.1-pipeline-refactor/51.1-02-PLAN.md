  <name>Task 1: Rewrite pulse.py — orchestrator pipeline with structured output</name>
  <files>backend/app/agents/pulse.py</files>
  <action>
Complete rewrite of `backend/app/agents/pulse.py`. Keep the module-level docstring pattern but replace everything.

**New PulseAgentState** (TypedDict):
- Keep existing fields: collection_id, user_id, pulse_run_id, user_context, file_data, file_profiles, error
- Replace signal_candidates/validated_signals/signals_output/report_content with:
  - `hypotheses: list[dict]` — from hypothesis node
  - `signal_results: list[dict]` — merged results per signal (coder output + business finding + chart data + generated code)
  - `report: dict` — from report writer node (executive_summary, content)

**Helper: _get_llm_for_agent(agent_name: str)** — generalized version of `_get_pulse_llm()` that takes agent name. Uses `get_agent_provider`, `get_agent_model`, `get_agent_temperature`, `get_agent_max_tokens`, `get_api_key_for_provider`, `get_llm`.

**Node 1: profile_data_node** — Keep as-is (already works). Reuse `profile_files_in_sandbox`.

**Node 2: generate_hypotheses_node** — Pulse brain generates signal candidates:
- Use `_get_llm_for_agent("pulse_agent")` (the brain uses the main pulse_agent config)
- `llm.with_structured_output(HypothesisOutput)` — NO manual JSON parsing
- Build prompt with deep_profile, data_summary, user context. Only the Pulse Agent (the brain) receives `data_summary` — downstream agents do not.
- System prompt enforces business language and prescriptive code_instructions
- Return `{"hypotheses": [h.model_dump() for h in result.hypotheses]}`

**Node 3: validate_signals_node** — Fan-out per-signal validation loop:
- For each hypothesis, run the full sub-pipeline: Coder -> Validator -> Sandbox -> Interpret -> Viz
- Use `asyncio.gather()` for parallelism across signals
- Inner function `_process_single_signal(hypothesis, file_data, settings)` handles one signal:

  a. **Coder**: `_get_llm_for_agent("pulse_coder")` with system prompt from prompts.yaml. User message contains the hypothesis's `code_instructions`, file paths, and `deep_profile` (column names, types, stats — so the Coder knows the data schema). Do NOT pass `data_summary` — the prescriptive `code_instructions` from the Pulse Agent already encode the business intent. Use `extract_code_block()` on response. Do NOT use `with_structured_output` for Coder — it generates code, not structured data.

  b. **Validator**: `validate_code(code)` — reuse directly. Skip signal if invalid.

  c. **Sandbox**: `E2BSandboxRuntime` with `settings.pulse_sandbox_timeout_seconds`. Parse last stdout line as JSON. Skip signal if execution fails.

  d. **Interpreter**: `_get_llm_for_agent("pulse_interpreter").with_structured_output(BusinessFinding)`. Pass the hypothesis title, sandbox results (compact aggregated data), and original code_instructions. Returns BusinessFinding with business-language title, finding, severity, evidence.

  e. **Visualization**: `_get_llm_for_agent("pulse_viz")`. Generate Plotly chart code per the `chart_hint`. Use `extract_code_block()` on response, `validate_code()`, execute in sandbox. Parse `fig.to_json()` from stdout. If viz fails, signal still valid (chart_data = None).

  Return merged dict per signal: title, finding, severity, category, evidence, chart_data, chart_type, generated_code (the coder's code), analysis_text.

- Return `{"signal_results": [list of merged dicts]}`

**Node 4: write_report_node** — Report Writer agent:
- Use `_get_llm_for_agent("pulse_report_writer").with_structured_output(ReportOutput)`
- Pass all signal results as context
- Return `{"report": result.model_dump()}`

**build_pulse_graph()**: 4 nodes in sequence:
`profile_data -> generate_hypotheses -> validate_signals -> write_report`

Key constraints:
- Zero imports from Chat agent modules (no ChatAgentState, no get_stream_writer)
- Reuse ONLY: `validate_code`, `extract_code_block`, `E2BSandboxRuntime`, `profile_files_in_sandbox`
- All LLM calls except Coder and Viz (which produce code) use `with_structured_output()`
- Business language enforcement in all prompts
- **Token efficiency**: Each agent receives only the minimum context it needs. Only the Pulse Agent gets `data_summary`. Coder gets `deep_profile` + `code_instructions` + file paths. Interpreter gets compact sandbox output + hypothesis context. Viz gets `chart_hint` + sandbox results. Report Writer gets signal results only.
