Add a new top-level key `search` (NOT inside agents) with:
```yaml
search:
  query_formulation_prompt: |
    Based on the user's data analysis question, formulate 1-{max_queries} web search queries
    to find relevant external benchmarks or industry data.

    User's question: {user_query}
    Data context: {data_context}

    RULES:
    - Create GENERIC search queries about industry standards, benchmarks, averages, and trends
    - NEVER include specific company names, personal names, dollar amounts, or identifiable data values from the dataset
    - Each query should be 3-8 words, focused on publicly available benchmarks
    - If the question is about the user's own data and doesn't need external comparison, return an empty array []
    - Return ONLY a JSON array of strings, nothing else

    Good examples: ["average SaaS churn rate 2025", "ecommerce revenue benchmarks by industry"]
    Bad examples: ["Acme Corp Q4 revenue", "john doe salary comparison", "$45000 average salary"]

    Return your queries:
  analysis_with_search_context: |
    **Web Search Results:**
    {search_context}

    Incorporate relevant web search findings into your analysis where they provide useful external benchmarks or context. Cite specific sources when referencing external data.
  search_unavailable_notice: "Note: Web search was requested but unavailable. Answering from available data only."
```

Add a helper function `async def formulate_search_queries(user_query, data_summary, llm, max_queries=3) -> list[str]`:
- Load search query formulation prompt from prompts.yaml via `load_prompts()["search"]["query_formulation_prompt"]`
- Format with `user_query`, `data_context=data_summary[:500]`, `max_queries`
- Call `await llm.ainvoke([HumanMessage(content=prompt)])`
- Parse response content as JSON array, strip markdown code fences if present (same pattern as existing JSON parsing in this file)
- Return list of queries (max max_queries items), return `[]` on parse failure
- Log formulated queries: `logger.info({"event": "search_queries_formulated", "count": len(queries), "queries": queries})`

In `data_analysis_agent()` STANDARD MODE section (after LLM initialization, before building messages):

```python
# Web search integration (SEARCH-01, SEARCH-02)
search_sources = []
search_context = ""
web_search_enabled = state.get("web_search_enabled", False)

if web_search_enabled:
    search_service = get_search_service()
    if search_service:
        # Formulate generic search queries (SEARCH-02 — agent decides)
        search_queries = await formulate_search_queries(
            user_query=state["user_query"],
            data_summary=state.get("data_summary", ""),
            llm=llm,
            max_queries=settings.search_max_queries,
        )

        if search_queries:
            # Execute searches with real-time status events
            for query in search_queries:
                writer({
                    "type": "search_started",
                    "message": f"Searching: '{query}'...",
                })

            # Parallel search execution (RESEARCH Pitfall 6)
            results = await search_service.search_multiple(search_queries)

            for result in results:
                if result.success and result.results:
                    for r in result.results:
                        search_sources.append({"title": r.title, "url": r.link})
                    search_context += f"\nSearch results for '{result.query}':\n"
                    for r in result.results:
                        search_context += f"- {r.title}: {r.link}\n"

            if search_sources:
                writer({
                    "type": "search_completed",
                    "message": f"Found {len(search_sources)} sources",
                    "sources_count": len(search_sources),
                })
            else:
                # All searches failed or returned no results
                writer({
                    "type": "search_failed",
                    "message": "Web search unavailable — answering from available data",
                })
        # else: agent decided search not needed (silent skip per CONTEXT.md)
```
