**2. Create `backend/app/agents/tools/web_search.py`** — @tool definition:
- Import `tool` from `langchain_core.tools` and `get_stream_writer` from `langgraph.config`.
- Define `search_web` as an async @tool function with signature `async def search_web(query: str) -> str`.
- Docstring MUST include: "Search the web for information using Google Search. Use this tool when you need external data like industry benchmarks, market statistics, or comparative data that isn't in the uploaded dataset." and Args description: "query: A generic search query (3-8 words). NEVER include specific company names, personal data, or raw values from the dataset."
- Inside the tool:
  - Call `writer = get_stream_writer()` and emit `{"type": "search_started", "message": f"Searching: '{query}'..."}`.
  - Get SearchService via `SearchService.from_settings()`. If None, return "Web search is not configured. Answer from available data only."
  - Call `result = await service.search(query)`.
  - If `not result.success`: emit `{"type": "search_failed", "message": "Web search unavailable"}` and return error message.
  - If no results: return "No relevant search results found."
  - Format results: `"Search results for '{query}':\n"` then for each result: `"- {r.title}: {r.link}\n"`.
  - Emit `{"type": "search_completed", "message": f"Found {len(result.results)} results", "sources_count": len(result.results)}`.
  - Return formatted string.

**3. Rewrite `backend/app/agents/data_analysis.py`** — split into two exported functions:

**`da_with_tools_node(state: ChatAgentState) -> dict`:**
- Emit `analysis_started` SSE event.
- Build LLM using existing per-agent config pattern (get_agent_provider/model/temperature/api_key/max_tokens for "data_analysis").
- Check `state.get("web_search_enabled", False)`. If True, call `llm_with_tools = llm.bind_tools([search_web])`. Wrap bind_tools in try/except — if it fails (provider doesn't support tools), log warning and fall back to bare LLM.
- If False, use bare LLM (no tools bound).
- Check routing_decision: if MEMORY_SUFFICIENT, build memory-mode prompt (same logic as current memory mode). If standard, build analysis prompt from YAML with execution context.
- IMPORTANT: For the tool-calling loop to work, construct messages properly. On first call (no tool messages in state yet), use `[SystemMessage(content=system_prompt), HumanMessage(content="Please analyze these results.")]`. On subsequent calls (after ToolNode returns results), the state.messages will contain AIMessage with tool_calls + ToolMessage with results. Use ALL messages from state that are part of the tool-calling conversation.
- Invoke `await llm_with_tools.ainvoke(messages)`.
- Return `{"messages": [response]}` — this is what tools_condition reads to decide routing.

**`da_response_node(state: ChatAgentState) -> dict`:**
- The last AIMessage in state.messages is the final response (no tool_calls).
- Extract content from the last AIMessage.
- Extract search sources from all ToolMessages in state.messages: parse each ToolMessage content for lines matching `"- {title}: {url}"` pattern. Build list of `{"title": title, "url": url}` dicts.
- Deduplicate sources by URL.
- Parse JSON response with fallback (strip markdown fences, try json.loads, extract "analysis" and "follow_up_suggestions"). This is the EXISTING parsing logic from the current data_analysis_agent — preserve it exactly.
- Return `{"analysis": analysis_text, "final_response": analysis_text, "follow_up_suggestions": follow_ups, "search_sources": unique_sources, "messages": [AIMessage(content=analysis_text)]}`.
- NOTE: The clean AIMessage at the end ensures the Manager Agent sees a readable message on subsequent queries, not a tool-calling message.

**Remove the old `data_analysis_agent` function** entirely. It is replaced by `da_with_tools_node` and `da_response_node`.
