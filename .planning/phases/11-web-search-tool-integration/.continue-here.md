<context>
User raised a valid concern about search quality: Serper returns snippets but search_web tool discards them, only passing titles+URLs to the LLM. Research was done on improving this with two options:

**Option A: Tavily (replace Serper)**
- Single API call returns search + full page content via `include_raw_content: "markdown"`
- Built-in LangChain tool (TavilySearchResults)
- PII/prompt injection protection
- Free tier: 1,000 credits/month, paid: $0.008/credit
- Simplest architecture: one tool, one API call

**Option B: Keep Serper + add read_urls tool (Trafilatura)**
- Two-tool pattern: search_web + read_urls
- Trafilatura for HTML -> clean text (F1: 0.958, open source)
- asyncio.gather() for parallel fetching (3 pages ~2-5s)
- 10,000 chars/page truncation
- No extra API cost

User was reviewing Tavily pricing page when they paused. No decision made yet on which approach to take. This enhancement could be:
1. A quick fix to search_web (add Serper snippets - minimal change)
2. A Phase 11.1 insert for full content reading (Tavily or Trafilatura)
3. Deferred to v0.3

The user seemed interested in Tavily's all-in-one approach.
</context>
