Create `backend/app/services/sandbox/models.py`:
- `ExecutionResult` dataclass with fields:
  - `stdout: list[str]` (lines printed to stdout)
  - `stderr: list[str]` (lines printed to stderr)
  - `results: list[dict]` (cell results -- last line value, plots, etc.)
  - `error: dict | None` (structured error: {"name": str, "value": str, "traceback": str} or None)
  - `execution_time_ms: int` (duration for monitoring)
  - `success: bool` property computed from `error is None`

Create `backend/app/services/sandbox/runtime.py`:
- `SandboxRuntime` Protocol with two methods:
  - `execute(code: str, timeout: float = 60.0, data_file: bytes | None = None, data_filename: str | None = None) -> ExecutionResult`
  - `cleanup() -> None`
- Use `typing.Protocol` (NOT abc.ABC -- Protocol is lighter weight, duck typing).

Create `backend/app/services/sandbox/e2b_runtime.py`:
- `E2BSandboxRuntime` class implementing SandboxRuntime.
- Constructor accepts `timeout_seconds: float = 60.0`.
- `execute()` method:
  1. Uses `Sandbox.create(timeout=int(timeout))` context manager for guaranteed cleanup.
  2. If `data_file` and `data_filename` provided, uploads to sandbox filesystem via `sandbox.filesystem.write(f"/home/user/{data_filename}", data_file)`.
  3. Calls `sandbox.run_code(code, timeout=timeout)`.
  4. Converts E2B execution object to `ExecutionResult`:
     - `stdout = execution.logs.stdout if execution.logs else []`
     - `stderr = execution.logs.stderr if execution.logs else []`
     - `results = [r.__dict__ if hasattr(r, '__dict__') else r for r in (execution.results or [])]`
     - `error = {"name": execution.error.name, "value": execution.error.value, "traceback": execution.error.traceback} if execution.error else None`
     - `execution_time_ms = int((time.time() - start_time) * 1000)`
  5. CRITICAL: Wraps entire sandbox execution in a synchronous function because E2B SDK is synchronous. This function will be called via `asyncio.to_thread()` by the caller (graph.py in Plan 02). Do NOT use async inside execute() -- the E2B SDK is synchronous.
- `cleanup()` method: no-op (context manager handles cleanup).
- Handle exceptions:
  - `TimeoutError` from E2B -> return ExecutionResult with error dict {"name": "TimeoutError", "value": "Execution exceeded {timeout}s limit", "traceback": ""}
  - `E2BException` or generic Exception -> return ExecutionResult with error dict capturing exception type and message
  - NEVER let exceptions propagate -- always return an ExecutionResult (the retry system in Plan 02 needs structured errors, not exceptions)

Create `backend/app/services/sandbox/__init__.py`:
- Export: `SandboxRuntime`, `E2BSandboxRuntime`, `ExecutionResult`
