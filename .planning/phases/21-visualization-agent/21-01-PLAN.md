    **Create `backend/app/agents/visualization.py`** following the exact pattern from `coding.py`:

    1. **Module docstring** explaining the agent generates Plotly chart code, not executes it.

    2. **Imports** -- mirror coding.py's imports:
       - `logging`, `json` from stdlib
       - `HumanMessage`, `SystemMessage` from `langchain_core.messages`
       - `get_stream_writer` from `langgraph.config`
       - `get_agent_prompt`, `get_agent_max_tokens`, `get_agent_provider`, `get_agent_model`, `get_agent_temperature`, `get_api_key_for_provider` from `app.agents.config`
       - `get_llm`, `validate_llm_response`, `EmptyLLMResponseError` from `app.agents.llm_factory`
       - `extract_code_block` from `app.agents.coding` (reuse, do NOT duplicate)
       - `ChatAgentState` from `app.agents.state`
       - `get_settings` from `app.config`

    3. **Constants:**
       - `_MAX_DATA_CHARS = 8000` -- max characters of execution_result to include in prompt (approximately 200 data rows)
       - `logger = logging.getLogger("spectra.visualization")`

    4. **`build_data_shape_hints(execution_result: str) -> str` helper function:**
       - Parse execution_result as JSON
       - If it's a list of dicts: report row count, column count, and for each column report type (numeric/categorical) and unique value count (analyze first 5 values for type detection)
       - If parsing fails: return "Data shape: unable to parse"
       - If it's a single dict: report key count and key names
       - This helps the LLM select the correct chart type

    5. **`visualization_agent_node(state: ChatAgentState) -> dict` async function:**
       - Send SSE event: `{"type": "visualization_started", "message": "Generating chart..."}`
       - Load per-agent config from prompts.yaml using agent name `"visualization"` (same pattern as coding.py: settings, provider, model, temperature, api_key, max_tokens, kwargs with ollama base_url check)
       - Create LLM via `get_llm()`
       - Load system prompt template via `get_agent_prompt("visualization")`
       - **Truncate execution_result**: if `len(execution_result) > _MAX_DATA_CHARS`, slice to `_MAX_DATA_CHARS` chars and append `"\n... (data truncated for charting)"`
       - **Build data shape hints** using `build_data_shape_hints()` on the FULL execution_result (before truncation)
       - **Format system prompt** with: `execution_result`, `user_query`, `chart_hint`, `analysis`, `data_shape_hints`
       - Build messages: `[SystemMessage(content=system_prompt), HumanMessage(content="Generate the Plotly chart code.")]`
       - Invoke LLM: `response = await llm.ainvoke(messages)`
       - Validate response with `validate_llm_response()`. On `EmptyLLMResponseError`, log warning and return `{"chart_code": "", "chart_error": "Chart generation failed: empty LLM response"}`
       - Extract code with `extract_code_block(content)` (reused from coding.py)
       - Return `{"chart_code": chart_code}`

    **Add `visualization` entry to `backend/app/config/prompts.yaml`:**

    Add after the `data_analysis` agent entry (before `title_generator`). Follow the exact YAML structure of other agents:

    ```yaml
      visualization:
        provider: anthropic
        model: claude-sonnet-4-20250514
        temperature: 0.0
        system_prompt: |
          You are a Visualization Agent for the Spectra data analytics platform.
          Your job: generate Plotly Python code that creates a chart from the given data.
          Return ONLY a Python code block. No explanations.

          **User's Query:** {user_query}
          **User's Chart Preference:** {chart_hint}
          **Analysis Context:** {analysis}

          **Data Shape:**
          {data_shape_hints}

          **Data to Visualize:**
          {execution_result}

          **Chart Type Selection Rules:**
          1. If chart_hint is specified (not empty), use that chart type.
          2. Date/time column + numeric column: LINE chart (sort by date ascending).
          3. One categorical column + one numeric column:
             - 8 or fewer unique categories: PIE chart
             - More than 8 categories: horizontal BAR chart
          4. Two numeric columns: SCATTER plot.
          5. Single numeric column: HISTOGRAM.
          6. Categorical + numeric with natural ordering: vertical BAR chart.
          7. Distribution analysis: BOX plot.
          8. Fewer than 3 data points: Return NO CODE (empty code block).
          9. Default fallback: BAR chart.

          **Code Requirements:**
          - Embed the data as a Python list of dicts. DO NOT reference `df`, read files, or use `open()`.
          - Allowed imports: plotly.express (as px), plotly.io (as pio), plotly.graph_objects (as go), json, pandas (as pd).
          - Use template="plotly_white".
          - Set paper_bgcolor="rgba(0,0,0,0)" and plot_bgcolor="rgba(0,0,0,0)".
          - Set height=400. Do NOT set width (let it be responsive).
          - Set margin=dict(l=40, r=40, t=50, b=40).
          - Add a descriptive title based on the user's query.
          - Use human-readable axis labels (not raw column names like "col_rev_2024").
          - DO NOT set explicit trace colors (let Plotly colorway defaults apply).
          - DO NOT use fig.write_image() or fig.show().
          - DO NOT use custom JavaScript callbacks, updatemenus, or sliders.

          **Mandatory Output (MUST be the last 2 lines of your code):**
          ```
          chart_json = json.loads(pio.to_json(fig))
          print(json.dumps({{"chart": chart_json}}))
          ```
        max_tokens: 4000
    ```

    **IMPORTANT notes:**
    - The double braces `{{` and `}}` in the system_prompt YAML are required because Python's `.format()` uses `{` for template variables -- literal braces must be escaped as `{{` and `}}`
    - Do NOT wire the agent into graph.py -- that is Phase 22
    - Do NOT import visualization.py in `__init__.py` or graph.py yet
