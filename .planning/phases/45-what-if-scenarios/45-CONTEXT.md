# Phase 45: What-If Scenarios - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Add the complete What-If Scenarios flow to the existing `pulse-mockup/` app — two new routes (objective selection page + session page), triggered from the signal detail panel and the investigation report page. Covers objective input, scenario generation loading, 4-panel session view with per-scenario refinement chat, and a generated What-If Report in the existing report viewer. No comparison view (deferred). No Add Scenario action (dropped). Static mockup only — all data hardcoded.

</domain>

<decisions>
## Implementation Decisions

### Entry Points
- "What-If" button appears in **two places**: the signal detail panel (alongside "Investigate") AND at the bottom of a completed investigation report page ("Explore What-If Scenarios" CTA)
- Entry from signal detail requires a completed investigation on that signal (button disabled otherwise)
- Both entry points navigate to the same objective selection route

### Objective Selection Page
- Route: `/workspace/collections/[id]/signals/[signalId]/whatif`
- Standalone full-page route (consistent with `/investigate` pattern from Phase 44)
- **Root cause context**: compact read-only card at the top — shows root cause title, confidence badge, and 1–2 sentence summary. Non-interactive, just an anchor for context.
- **Objective input**: command-palette-style Action Search Bar
  - Clicking/focusing the bar opens a dropdown with 3–4 AI-suggested objectives based on the root cause (e.g., "Reduce lead time by 20%", "Identify alternative suppliers", "Mitigate stockout risk")
  - Selecting a suggestion **populates the input field but does NOT auto-submit** — user can edit the text
  - User can also ignore suggestions and type their own objective freely
  - A "Generate Scenarios" button submits the objective manually
- After submit: full-page loading state (animated labeled steps — same pattern as `detection-loading.tsx`) with steps like "Analyzing root cause", "Generating scenarios", "Scoring confidence" — then auto-navigates to the session page

### What-If Session Page
- Route: `/workspace/collections/[id]/signals/[signalId]/whatif/[sessionId]`
- **4-panel layout** — mirrors the signal detail page structure, extended with a 4th panel:
  1. App shell sidebar (existing)
  2. Scenario list panel (~280px) — compact scenario cards
  3. Scenario detail panel — full scenario content, updates on AI refinement
  4. Refinement chat panel (rightmost) — natural language chat scoped to selected scenario

### Scenario List Panel
- Default: **3 scenarios** generated by Spectra
- First scenario (highest confidence) is **auto-selected on load** (same pattern as signal auto-selection)
- Compact scenario cards in the list: **name + confidence badge + impact range** (e.g., "Diversify Suppliers · [High] · +12–18%")
- Switching selected scenario switches both the detail panel and the chat panel context
- "Generate What-If Report" button pinned at the bottom of the list panel (or in the panel header)

### Scenario Detail Panel
- Full scenario content displayed:
  - Name
  - Narrative (2–3 sentences)
  - Estimated impact range (e.g., "+12–18% reduction in lead time")
  - Assumptions list (3–4 bullet points)
  - Confidence badge + rationale (why the AI assigned this confidence level)
  - Data backing summary (what data supports this scenario)
- When AI updates via refinement chat: **full panel refresh** — all fields re-render with new values. Chat history in the right panel is preserved across the refresh.

### Refinement Chat Panel
- Rightmost panel — always visible alongside the detail panel
- **Natural language free-form chat** (not structured Q&A choices like Investigation)
- User types instructions in plain language: "Change the timeline from 6 months to 3 months", "Remove the third assumption", "Make this more conservative"
- AI responds in the chat confirming what changed, and the scenario detail panel refreshes with updated values
- Each scenario maintains its **own independent chat history** — switching scenarios shows that scenario's chat thread
- Chat input is a standard message input at the bottom of the panel

### What-If Report (WHAT-07)
- Triggered by "Generate What-If Report" button in the session page
- Generates report and **auto-navigates to `/reports/[reportId]`** — same document-style viewer from Phase 43
- Report type: `"whatif"` (new type alongside `"pulse"` and `"investigation"`)
- Report content:
  - Stated objective
  - Root cause context (the investigation finding that prompted the What-If)
  - All 3 scenarios with their full details (narrative, impact, assumptions, confidence, data backing)
  - AI-recommended approach with rationale (which scenario to pursue and why)

### Claude's Discretion
- Exact visual design of the Action Search Bar dropdown (styling of suggestion items, hover states, icons)
- How the 4th chat panel is sized relative to the detail panel (suggested: ~300px fixed width)
- Mock content for objectives, scenarios, assumptions, AI chat responses
- Exact wording of loading step labels for scenario generation
- How the chat panel header identifies which scenario is currently in context
- "Generate What-If Report" button placement (list panel bottom vs. panel header)

</decisions>

<specifics>
## Specific Ideas

- Action Search Bar: command-palette pattern (like the attached reference image) — search/input field that reveals a dropdown list of suggestions on focus. Select populates field, does not submit. User edits then manually submits.
- 4-panel layout is the key UX innovation of this phase — it mirrors modern AI-native SaaS tools (Cursor, Linear AI) where a chat panel drives live updates to a content panel
- The refinement chat is intentionally free-form (not guided Q&A) because scenario refinement is open-ended, unlike investigation which needs structured narrowing
- Full panel refresh on AI update is intentional simplicity — no field-level highlight animations needed
- Chat history must survive panel refreshes (the user's conversation thread is the audit trail of their refinement decisions)

</specifics>
