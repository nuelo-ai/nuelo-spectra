<objective>
Add two enforcement layers for admin credentials: (1) a Pydantic startup validator that refuses to boot
when SPECTRA_MODE is dev or admin and ADMIN_EMAIL/ADMIN_PASSWORD are missing, and (2) an automatic
admin seed in docker-entrypoint.sh that runs after migrations before uvicorn starts.

Purpose: Prevent operators from accidentally deploying dev/admin-mode instances that cannot be
logged into — fail fast with a clear error rather than booting silently with an inaccessible admin panel.
Output: Updated config.py with model_validator, updated docker-entrypoint.sh with conditional seed
block, updated .env.docker.example with accurate documentation.
</objective>

  <name>Task 1: Add model_validator to Settings for fail-fast startup validation</name>
  <files>backend/app/config.py</files>
  <action>
Add a Pydantic v2 `model_validator(mode="after")` to the `Settings` class that enforces admin
credentials are set when the mode requires them.

Steps:
1. Add `model_validator` to the import: change `from pydantic_settings import BaseSettings, SettingsConfigDict`
   to also import `from pydantic import model_validator` (add as a new import line since `pydantic` is
   already a transitive dependency of `pydantic-settings`).

2. Add the following method to the `Settings` class, placed AFTER the `admin_last_name` field and
   BEFORE the `admin_session_timeout_minutes` field (keep field grouping intact):

```python
@model_validator(mode="after")
def validate_admin_credentials_for_admin_modes(self) -> "Settings":
    """Require ADMIN_EMAIL and ADMIN_PASSWORD when SPECTRA_MODE is dev or admin.

    This validator fires at Settings() construction (once, via lru_cache) so
    the process fails immediately at startup rather than at first admin request.
    """
    if self.spectra_mode in ("dev", "admin"):
        missing = []
        if not self.admin_email:
            missing.append("ADMIN_EMAIL")
        if not self.admin_password:
            missing.append("ADMIN_PASSWORD")
        if missing:
            raise ValueError(
                f"{', '.join(missing)} must be set when SPECTRA_MODE is "
                f"'{self.spectra_mode}'. Add "
                f"{'them' if len(missing) > 1 else 'it'} to your .env file and restart."
            )
    return self
```

CRITICAL constraints:
- Do NOT change `admin_email: str = ""` or `admin_password: str = ""` field defaults — the empty
  string default is intentional so Pydantic does not error on missing env vars in public mode;
  the validator provides the explicit message instead.
- Do NOT move the mode validity check from main.py (`if mode not in ("public", "admin", "dev")`).
  Leave it as-is. The new validator is additive only.
- Do NOT call `seed_admin` from within this validator or anywhere in config.py — validation only.
- Raise `ValueError` (not `SystemExit`) — Pydantic v2 validators must raise ValueError; Pydantic
  wraps it in ValidationError which propagates during lru_cache construction at module import.
  </action>

  <name>Task 2: Add conditional admin seed block to docker-entrypoint.sh</name>
  <files>backend/docker-entrypoint.sh</files>
  <action>
Insert a conditional seed block into `backend/docker-entrypoint.sh` AFTER the migrations echo line
and BEFORE the uvicorn exec line.

Current file structure (abbreviated):
```bash
echo "[entrypoint] Running database migrations..."
/app/.venv/bin/python -m alembic upgrade head
echo "[entrypoint] Migrations complete."

# --- Start uvicorn via exec (replaces bash as PID 1) ---
echo "[entrypoint] Starting uvicorn..."
exec /app/.venv/bin/uvicorn app.main:app \
  --host 0.0.0.0 \
  --port 8000 \
  --workers "${UVICORN_WORKERS:-1}"
```

Insert the following block BETWEEN the "Migrations complete." echo and the "# --- Start uvicorn ---" comment:

```bash
# --- Seed admin user (if ADMIN_EMAIL is set) ---
if [ -n "${ADMIN_EMAIL:-}" ]; then
    echo "[entrypoint] Seeding admin user (${ADMIN_EMAIL})..."
    /app/.venv/bin/python -m app.cli seed-admin
    echo "[entrypoint] Admin seed complete."
else
    echo "[entrypoint] ADMIN_EMAIL not set — skipping admin seed (public mode)."
fi
```

CRITICAL constraints:
- Use `${ADMIN_EMAIL:-}` (with dash-empty fallback) so the check works correctly whether the var
  is unset OR set to empty string — both cases skip the seed.
- Use `[ -n "${ADMIN_EMAIL:-}" ]` (length-nonzero test) NOT `[ -z ... ]` (which inverts the logic).
- The seed block MUST come AFTER `alembic upgrade head` (schema must exist before seed runs) and
  BEFORE `exec /app/.venv/bin/uvicorn` (admin user must exist before any request can be served).
- Use `/app/.venv/bin/python -m app.cli seed-admin` — the CLI reads ADMIN_EMAIL/ADMIN_PASSWORD from
  env automatically via get_settings(); no args needed.
- `set -euo pipefail` is already at the top of the script — if seed exits non-zero (e.g., DB error),
  the entrypoint will abort, which is the correct behavior (fail fast).
- Do NOT run seed unconditionally (would break public mode deployments where ADMIN_EMAIL is not set).
  </action>
