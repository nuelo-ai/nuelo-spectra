**chartTypeCompatibility.ts -- Create new file at `frontend/src/lib/chartTypeCompatibility.ts`:**

This utility analyzes Plotly chart JSON to determine which chart types are compatible for switching and what the current chart type is.

Export a type:
```typescript
export type SwitchableChartType = 'bar' | 'line' | 'scatter';
```

Export function `detectCurrentChartType(chartDataJson: string): SwitchableChartType | null`:
1. Parse the JSON string to get the chart data object.
2. Get the first trace from `chartData.data[0]`.
3. If no trace, return null.
4. Determine type from trace properties:
   - If `trace.type === 'bar'` -> return `'bar'`
   - If `trace.type === 'scatter'` and `trace.mode` includes `'lines'` -> return `'line'`
   - If `trace.type === 'scatter'` and `trace.mode` includes `'markers'` (or mode is undefined/null) -> return `'scatter'`
   - If `trace.type === 'scatter'` with no mode -> return `'scatter'` (Plotly default)
   - For any other type (pie, histogram, box, etc.) -> return null (not switchable)
5. Wrap in try/catch, return null on parse errors.

Export function `getCompatibleChartTypes(chartDataJson: string): SwitchableChartType[]`:
1. Call `detectCurrentChartType(chartDataJson)`.
2. If null (not a switchable type), return empty array `[]`.
3. Parse the JSON to get the first trace.
4. Check data shape:
   - Both `trace.x` and `trace.y` exist and are arrays:
     - If x values are numeric (check first non-null value with `typeof ... === 'number'`): return `['bar', 'line', 'scatter']` (all 3 compatible).
     - If x values are strings (categorical): return `['bar', 'line']` (scatter doesn't make sense with categorical x).
   - Otherwise: return array containing only the detected current type.
5. Wrap in try/catch, return empty array on errors.

Important: Access trace properties using type assertions since Plotly's PlotData type is complex. Use `as any` or `Record<string, unknown>` pattern for trace property access (same pattern as ChartRenderer.tsx).

**ChartTypeSwitcher.tsx -- Create new file at `frontend/src/components/chart/ChartTypeSwitcher.tsx`:**

```typescript
'use client';
```

Import: `useState` from React, `BarChart3`, `TrendingUp`, `Circle` from lucide-react (icons for bar/line/scatter), `Plotly` from plotly.js-dist-min, `SwitchableChartType` and `getCompatibleChartTypes` and `detectCurrentChartType` from the compatibility utility.

Props interface:
```typescript
interface ChartTypeSwitcherProps {
  /** Function to get the Plotly chart DOM element */
  getChartElement: () => HTMLDivElement | null;
  /** JSON string of original chart data (for compatibility analysis) */
  chartDataJson: string;
}
```

Component logic:
1. Call `detectCurrentChartType(chartDataJson)` to get the initial type. If null, return null (hide switcher entirely).
2. Call `getCompatibleChartTypes(chartDataJson)` to get compatible types. If length <= 1, return null (no switching possible).
3. Use `useState<SwitchableChartType>(initialType)` for tracking current selection.
4. `handleTypeChange(newType: SwitchableChartType)`:
   - If newType === currentType, return (no-op).
   - Get chart element via `getChartElement()`. If null, return.
   - Use Plotly.restyle() to change the trace type:
     - If `newType === 'line'`: `Plotly.restyle(element, { type: 'scatter', mode: 'lines' }, [0])`
     - If `newType === 'scatter'`: `Plotly.restyle(element, { type: 'scatter', mode: 'markers' }, [0])`
     - If `newType === 'bar'`: `Plotly.restyle(element, { type: 'bar', mode: undefined }, [0])` -- set mode to undefined to clear line/marker mode.
   - Set currentType to newType.
   - Wrap in try/catch, console.error on failure.

Render: A row of small toggle buttons (not a select dropdown -- more visual and matches Spectra's UI style):
```tsx
<div className="flex items-center gap-1">
  <span className="text-xs text-muted-foreground mr-1">Chart type:</span>
  {compatibleTypes.map((type) => (
    <button
      key={type}
      onClick={() => handleTypeChange(type)}
      className={`flex items-center gap-1 px-2 py-1 text-xs rounded border transition-colors ${
        currentType === type
          ? 'bg-primary text-primary-foreground border-primary'
          : 'bg-background hover:bg-accent border-border'
      }`}
      title={`Switch to ${type} chart`}
    >
      {/* Icon per type */}
      {type === 'bar' && <BarChart3 className="h-3 w-3" />}
      {type === 'line' && <TrendingUp className="h-3 w-3" />}
      {type === 'scatter' && <Circle className="h-3 w-3" />}
      {type.charAt(0).toUpperCase() + type.slice(1)}
    </button>
  ))}
</div>
```

The active button has `bg-primary text-primary-foreground` (filled/highlighted), inactive buttons have `bg-background` (outlined). This matches shadcn/ui toggle patterns used elsewhere in the app.
