<task type="auto">
  <name>Task 2: Fix PostgresSaver context manager usage in graph.py</name>
  <files>backend/app/agents/graph.py</files>
  <action>
Fix the `build_chat_graph()` function in backend/app/agents/graph.py.

The current code at lines 436-437 is broken:
```python
checkpointer = PostgresSaver.from_conn_string(postgres_url)
checkpointer.setup()
```

`PostgresSaver.from_conn_string()` returns an `Iterator[PostgresSaver]` (context manager), not a PostgresSaver object directly. Calling `.setup()` on the context manager raises `AttributeError: '_GeneratorContextManager' object has no attribute 'setup'`.

IMPORTANT: The graph is cached globally via `get_or_create_graph()` and reused across requests. A `with` block would close the connection when the block exits. Instead, consume the context manager manually so the checkpointer persists for the lifetime of the cached graph.

Replace lines 436-437 with:

```python
    # Initialize PostgreSQL checkpointer
    # from_conn_string() returns a context manager; we enter it manually
    # since the graph is cached globally and the checkpointer must persist
    checkpointer_ctx = PostgresSaver.from_conn_string(postgres_url)
    checkpointer = checkpointer_ctx.__enter__()
    checkpointer.setup()
```

This manually enters the context manager to get the actual PostgresSaver object, then calls `.setup()` on it. Since the graph is cached globally in `_cached_graph`, the checkpointer will live for the application lifetime, which is the intended behavior.

Do NOT change any other code in graph.py. The graph node definitions, edges, and compilation remain exactly as-is.
