<task type="auto">
  <name>Task 1: Harden FileUploadZone state machine with defensive rendering and error timeout</name>
  <files>frontend/src/components/file/FileUploadZone.tsx</files>
  <action>
The upload flow has failed in UAT twice despite code-level fixes looking correct. The issue is a race condition between React state transitions and React Query data availability across renders. The useEffect sets uploadStage to "ready" when summary.data_summary is truthy, but the NEXT render may not have the same summary reference because React Query can return a new object.

Rewrite the FileUploadZone upload flow with these specific changes:

1. STORE analysis text in local state instead of relying on React Query data across renders:
   - Add a new useState: `const [analysisText, setAnalysisText] = useState<string | null>(null);`
   - In the useEffect that transitions to "ready", ALSO set the analysis text: `setAnalysisText(summary.data_summary);`
   - Change the render condition from `summary?.data_summary` to `analysisText` for the analysis display block
   - This decouples the display from React Query's data lifecycle

2. ADD an error timeout for onboarding:
   - Add a new useEffect that starts a 60-second timeout when uploadStage enters "analyzing"
   - If 60 seconds pass without transitioning to "ready", set a new state `uploadError` and show an error message
   - Clear the timeout when transitioning to "ready" or when component unmounts
   - Display: "Analysis is taking longer than expected. Your file was uploaded successfully. Close this dialog and check the sidebar -- your file should appear there."
   - Include a "Close" button that calls onUploadComplete and resets state

3. ADD console.log statements at critical state transitions for debugging:
   - Log when uploadStage changes (in each setter call)
   - Log when useFileSummary returns data (in the useEffect)
   - Log when "Continue to Chat" button is clicked
   - Log when refetchQueries completes
   - Prefix all logs with "[FileUploadZone]" for easy filtering
   - These are development-mode aids; keep them concise (one line each)

4. ENSURE the "Continue to Chat" button handler is robust:
   - Wrap the refetchQueries + openTab + onUploadComplete sequence in a try/catch
   - If refetchQueries fails, still close the dialog and open the tab (the sidebar will eventually update on window focus)
   - Add a brief loading state on the button while awaiting refetch ("Preparing..." text)

5. RESET analysisText and hasTransitioned ref when state returns to "idle" (important for re-uploads in same dialog session).

Do NOT change the useFileSummary hook call (the OR condition fix from 06-14 is correct). Do NOT change the polling interval. Do NOT change the useEffect dependency array beyond what is described above.
