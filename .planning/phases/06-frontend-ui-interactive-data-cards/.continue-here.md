## Decisions Made

### Architecture Decisions:
1. **Disabled PostgreSQL checkpointing temporarily**
   - PostgresSaver.aget_tuple() raises NotImplementedError with async streaming
   - Impact: Each query starts fresh (no LangGraph state persistence between queries)
   - Chat history still saved to DB via ChatService (frontend shows full history)
   - Acceptable for v1.0 MVP
   - TODO: Investigate AsyncPostgresSaver or upgrade langgraph-checkpoint-postgres

2. **Bypassed Next.js proxy entirely**
   - Next.js rewrites doing 307 redirect in dev mode
   - Browser drops Authorization headers on cross-origin redirects (security policy)
   - Changed all API calls from `/api/*` to `http://localhost:8000/*`
   - CORS already configured on backend for localhost:3000
   - Faster than debugging Next.js config

3. **Increased token limits to prevent truncation**
   - Coding agent was hitting 800 token limit
   - Code got cut off mid-generation → syntax errors
   - Increased to 2000 tokens with guidance to keep code concise

## Root Causes Diagnosed

### Issue 1: Chat Returns Empty Error (UAT Test 5)
**Symptom:** SSE returns `data: {"type": "error", "message": ""}`

**Root Cause Chain:**
1. PostgresSaver initialized with sync `__enter__()`
2. LangGraph tried to use async `aget_tuple()` → NotImplementedError
3. Exception caught but `str(e)` was empty
4. Added logging to see full traceback
5. Disabled checkpointing as temporary fix

### Issue 2: Code Truncation After Multiple Retries
**Symptom:** Generated code ends abruptly mid-line, causes syntax error

**Root Cause:**
- Coding agent max_tokens: 800 too low
- On retry attempts, AI generates more comprehensive code with error handling
- Exceeds 800 tokens → gets cut off mid-generation
- Increased to 2000 tokens

### Issue 3: E2B Authentication Failure
**Symptom:** `AuthenticationException: API key is required`

**Root Cause:**
- E2B_API_KEY in .env but not passed to Sandbox.create()
- E2B SDK expects either env var OR explicit api_key parameter
- Fixed by passing `api_key=settings.e2b_api_key`
