Rewrite the analysis completion logic in FileUploadZone.tsx. There are two bugs to fix and one feature to add:

**Bug 1: Side-effect code in component body (lines 34-53)**

The current code runs directly in the component body:
```typescript
if (uploadStage === "analyzing" && summary?.data_summary) {
  setUploadStage("ready");
  // ... setTimeout with state updates
}
```

This executes on EVERY render, creating multiple setTimeout callbacks and race conditions.

**Fix:** Replace lines 34-53 with a `useEffect` hook. Add a `hasTransitioned` ref to prevent duplicate transitions:

```typescript
const hasTransitioned = useRef(false);

useEffect(() => {
  if (uploadStage === "analyzing" && summary?.data_summary && !hasTransitioned.current) {
    hasTransitioned.current = true;
    setUploadStage("ready");
    setProgress(100);
  }
}, [uploadStage, summary?.data_summary]);
```

Add `useRef` to the React imports at the top of the file:
```typescript
import { useCallback, useState, useEffect, useRef } from "react";
```

**Bug 2: Dialog closes before query refetch completes**

The old code used a fixed 1500ms setTimeout to close the dialog, but query refetch might not complete in time.

**Fix:** Remove the auto-close from the useEffect. Instead, add a "Continue" button in the ready state that the user clicks after seeing the analysis. This also solves the analysis visibility problem.

When `uploadStage === "ready"` and `summary?.data_summary` exists, display the analysis text in the upload dialog before the user dismisses it. This addresses the UAT finding that users never see the analysis result.
