  <name>Task 1: Create settings hooks and form components</name>
  <files>
    frontend/src/hooks/useSettings.ts

    frontend/src/components/settings/AccountInfo.tsx
  </files>
  <action>
    1. Create `frontend/src/hooks/useSettings.ts`:
       - `useUpdateProfile()`: TanStack Query useMutation
         - mutationFn: PATCH /api/auth/me with `{ first_name, last_name }`
         - Uses apiClient.patch()
         - On success: invalidate user query, show success toast via sonner
         - Returns UserResponse
       - `useChangePassword()`: TanStack Query useMutation
         - mutationFn: POST /api/auth/change-password with `{ current_password, new_password }`
         - Uses apiClient.post()
         - On success: show success toast "Password changed successfully"
         - On 401 error: show error toast "Current password is incorrect"
         - On other errors: show error toast with message

    2. Create `frontend/src/components/settings/ProfileForm.tsx` (Client Component):
       - Uses useAuth() to get current user data for initial values
       - Two input fields:
         - First Name: shadcn Input, pre-filled with user.first_name
         - Last Name: shadcn Input, pre-filled with user.last_name
       - "Save Changes" button:
         - Uses useUpdateProfile() mutation
         - Loading state while request is in-flight (button shows spinner)
         - Disabled if no changes from initial values
       - Form validation: no special validation beyond non-empty (names are optional in backend)
       - On success: sonner toast "Profile updated successfully", update auth context user
       - Wrapped in shadcn Card with title "Profile Information" and description "Update your personal details"

    3. Create `frontend/src/components/settings/PasswordForm.tsx` (Client Component):
       - Three input fields:
         - Current Password: type="password", required
         - New Password: type="password", required, min 8 characters
         - Confirm New Password: type="password", must match New Password
       - Client-side validation:
         - New password minimum 8 characters
         - Confirm password must match new password
         - Show inline validation errors below fields
       - "Change Password" button:
         - Uses useChangePassword() mutation
         - Loading state while request is in-flight
         - Disabled if validation fails
       - On success: clear all fields, sonner toast "Password changed successfully"
       - On 401: sonner toast "Current password is incorrect" (error variant)
       - Wrapped in shadcn Card with title "Change Password" and description "Update your account password"

    4. Create `frontend/src/components/settings/AccountInfo.tsx` (Client Component):
       - Uses useAuth() to get current user data
       - Read-only display of:
         - Email address (not editable)
         - Account creation date (formatted as readable date: "February 3, 2026")
         - Account status: "Active" badge (green)
       - Wrapped in shadcn Card with title "Account Information" and description "Your account details"
       - Email and date displayed with Label + static text (not input fields)
