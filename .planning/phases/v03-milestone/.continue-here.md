## Session 3 (this session — Bug 7 found and fixed)

### Bug 7: React Strict Mode stream abort — messages never reach backend
- **File:** `frontend/src/components/chat/ChatInterface.tsx`
- **Root cause:** Next.js 16 enables React Strict Mode by default. In dev, React simulates unmount/remount. The pending stream useEffect called `startStream()` on first mount, but `useSSEStream`'s cleanup aborted the in-flight fetch during simulated unmount. The `pendingStreamHandled` ref blocked retry on second mount.
- **Fix:** Replaced ref guard with `setTimeout(0)` to defer `startStream()` past the Strict Mode cleanup cycle. SessionStorage serves as the guard — checked both before and inside the timer.
- **User confirmed:** Chat works from frontend after fix.

### Committed as: `10f25ae fix: resolve React Strict Mode stream abort in ChatInterface pending stream handoff`
