<context>
The v0.3 milestone restructured Spectra from file-tab-centric to session-centric UX. All 5 phases (14-18) were code-complete. During UAT, we found and fixed 4 critical bugs (session 404, sidebar collapse, redirect loop, detail 500) in a previous session. This session tackled a 5th issue: sessions auto-creating on page load, spamming the sidebar.

The lazy session creation refactor required a careful approach because:
1. FileLinkingDropdown needs a sessionId to link files via API
2. ChatInput has its own file enforcement check (linkedFileIds.length === 0)
3. router.replace() unmounts the component, killing any in-flight async operations

Solution: two-phase approach where files are selected locally (pendingFileIds state), then on first message send: create session → link files → store message in sessionStorage → navigate → auto-send from useEffect.

The user's latest error (500 on /files/) is NOT a code bug — curl confirms all endpoints return 200 with fresh tokens. The backend hot-reloaded which may have invalidated the user's JWT session.
