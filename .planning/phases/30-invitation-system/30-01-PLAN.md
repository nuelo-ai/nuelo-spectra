    **1. Create `backend/app/schemas/invitation.py`:**
    - `CreateInviteRequest`: email (EmailStr), required
    - `InviteResponse`: id (UUID), email, status, created_at, expires_at, accepted_at (optional)
    - `InviteListResponse`: items (list[InviteResponse]), total (int), page (int), page_size (int)
    - `InviteDetailResponse`: same as InviteResponse but with invited_by info
    - `DuplicateInviteWarning`: message (str), existing_invite_id (UUID) -- for the duplicate pending invite warning

    **2. Create `backend/app/services/admin/invitations.py`:**
    - Import `create_reset_token` from `app.services.email` (reuse for invite token generation)
    - Import `Invitation` model, `User` model, `platform_settings`
    - Module-level `_resend_cooldowns: dict[str, float] = {}` for 10-minute resend cooldown tracking (same pattern as `_reset_cooldowns` in auth.py)

    Functions:
    - `async def create_invitation(db, email, admin_id, settings) -> dict`:
      - Check if email already registered in users table -> raise ValueError("This email is already registered")
      - Check for existing pending invite (status="pending" AND expires_at > now) -> raise `DuplicateInviteError(email, invite_id)`
      - Check max_pending_invites platform setting -> raise ValueError if exceeded
      - Generate token via `create_reset_token()`
      - Read `invite_expiry_days` from platform_settings
      - Create Invitation row (email, token_hash, invited_by=admin_id, status="pending", expires_at=now+expiry_days)
      - db.add() but do NOT commit (caller commits after audit log)
      - Return `{"invitation": invitation, "raw_token": raw_token}`

    - `async def list_invitations(db, status_filter=None, page=1, page_size=20) -> dict`:
      - Query Invitation table with optional status filter
      - For "pending" filter: also filter expires_at > now (truly pending, not silently expired)
      - For "expired" filter: include both status="expired" AND (status="pending" AND expires_at <= now)
      - Order by created_at DESC
      - Return paginated results with total count

    - `async def revoke_invitation(db, invite_id) -> Invitation`:
      - Fetch invitation by id
      - Raise ValueError if not found or not status="pending" or already expired
      - Set status = "expired" (per locked decision: revoked becomes Expired)
      - Return updated invitation

    - `async def resend_invitation(db, invite_id, admin_id, settings) -> dict`:
      - Fetch invitation by id
      - Raise ValueError if not found or not status="pending"
      - Check resend cooldown: if email in `_resend_cooldowns` and (now - last_resend) < 600 seconds, raise ValueError("Please wait before resending. Minimum 10 minutes between resends.")
      - Set old invitation status = "expired" (invalidate previous token per locked decision)
      - Create new Invitation with fresh token and full expiry window
      - Update `_resend_cooldowns[email] = now`
      - Return `{"invitation": new_invitation, "raw_token": raw_token}`

    - Define `DuplicateInviteError(Exception)` with email and invite_id attributes

    **3. Add `send_invite_email()` to `backend/app/services/email.py`:**
    - Follow exact same pattern as `send_password_reset_email()`
    - Parameters: to_email, invite_link, expiry_date (str, human-readable), settings
    - Dev mode: log invite link to console (same format as password reset dev mode)
    - Production: render `invite.html` and `invite.txt` templates
    - Template variables: invite_link, expiry_date
    - Subject: "You've been invited to join Spectra"
    - From: same as password reset (settings.smtp_from_name/email)
    - No personal message field (locked decision)
    - No inviter name in email (locked decision: minimal content)

    **4. Create `backend/app/templates/email/invite.html`:**
    - Match EXACT same structure as password_reset.html (same table layout, same CSS inline styles)
    - Same branded header: "Spectra" in #2563eb blue
    - Body text: "You've been invited to join Spectra" (professional, formal tone per locked decision)
    - Brief text: "Click the button below to create your account."
    - CTA button: "Accept Invitation" (same blue #2563eb, same border-radius: 6px, same padding)
    - Link text fallback: "Or copy and paste this link into your browser:" + {{ invite_link }}
    - Expiry notice: "This invitation expires on {{ expiry_date }}."
    - NO "if you didn't request this" text (unlike password reset)
    - NO inviter name, NO platform description (locked decision: minimal content)

    **5. Create `backend/app/templates/email/invite.txt`:**

      - Add `max_pending_invites: int` to SettingsResponse
      - Add `max_pending_invites: int | None = None` to SettingsUpdateRequest
      - Update the at_least_one_field validator to include max_pending_invites
