  <action>
    DNS records MUST be propagated before adding the domain in Dokploy. If you add the domain before DNS resolves to the correct IP, Let's Encrypt will fail to provision the certificate and you must delete and re-add the domain entry.

    **Step 1 — Add DNS A records (in your DNS provider — Cloudflare, Route53, etc.):**

    Add these A records pointing to your Dokploy server's public IP:
    - `api.yourdomain.com` → `<SERVER_PUBLIC_IP>`
    - `app.yourdomain.com` → `<SERVER_PUBLIC_IP>`

    If using Cloudflare: set Proxy status to "DNS only" (grey cloud), not "Proxied". Traefik's Let's Encrypt HTTP challenge requires direct access to the server.

    **Step 2 — Verify DNS propagation before proceeding:**
    ```bash
    # Run from your local machine or any external server
    dig +short api.yourdomain.com
    # Expected: <SERVER_PUBLIC_IP>

    dig +short app.yourdomain.com
    # Expected: <SERVER_PUBLIC_IP>
    ```
    Wait until BOTH records return your server IP. DNS propagation typically takes 1-15 minutes; can take up to 48 hours if using high TTLs. Do not proceed to Step 3 until both return the correct IP.

    **Step 3 — Add domain to public backend in Dokploy (AFTER DNS propagates):**
    In Dokploy UI → spectra-public-backend → Domains tab → Add Domain:
    - Host: `api.yourdomain.com`
    - Container Port: `8000`
    - HTTPS: Enabled
    - Certificate: `letsencrypt`
    Click Save and wait 30-60 seconds for Traefik to provision the certificate.

    **Step 4 — Verify public backend HTTPS:**
    ```bash
    curl -s https://api.yourdomain.com/health
    # Expected: {"status": "ok"}
    ```
    If you get a certificate error, wait another minute and retry. If it persists after 5 minutes, check Dokploy's Traefik logs for ACME errors. If DNS was not fully propagated, delete the domain entry and re-add it.

    **Step 5 — Add domain to public frontend in Dokploy:**
    In Dokploy UI → spectra-public-frontend → Domains tab → Add Domain:
    - Host: `app.yourdomain.com`
    - Container Port: `3000`
    - HTTPS: Enabled
    - Certificate: `letsencrypt`
    Click Save.

    **Step 6 — Update public backend CORS for the production domain:**
    In Dokploy UI → spectra-public-backend → Environment tab, update:
    ```
    FRONTEND_URL=https://app.yourdomain.com
    CORS_ORIGINS=["https://app.yourdomain.com"]
    ```
    Click Save → Redeploy.

    **Step 7 — Update public frontend BACKEND_URL to use HTTPS domain:**
    In Dokploy UI → spectra-public-frontend → Environment tab, update:
    ```
    BACKEND_URL=https://api.yourdomain.com
    ```
    Click Save → Redeploy.

    This is the critical link: the public frontend must now proxy through the HTTPS domain so that SSE streaming and all API calls use the production URL.

    # Spectra Production Deployment Guide

    Spectra deploys as four Dokploy Application services with a split-horizon architecture:
    - **Public services** (API + frontend): publicly accessible via HTTPS
    - **Admin services** (admin API + admin frontend): accessible only via Tailscale VPN

    ## Architecture Overview

    ```
    Internet → Traefik (Dokploy) → spectra-public-backend  (port 8000 internal)
                                 → spectra-public-frontend  (port 3000 internal)

    Tailscale VPN → host:8001 → spectra-admin-backend
                 → host:3001 → spectra-admin-frontend

    Both backends  → Dokploy-managed PostgreSQL (shared DB, same SECRET_KEY)
    Public backend → spectra-uploads named volume at /app/uploads
    ```

    ## Prerequisites

    - Dokploy installed on a VPS (see https://dokploy.com)
    - Git repository connected to Dokploy
    - A domain with DNS management access
    - Tailscale account (free tier is sufficient — https://login.tailscale.com)
    - VPS provider with cloud firewall support (DigitalOcean, Hetzner, AWS, etc.)

    ## Step 1: Create PostgreSQL Database

    In Dokploy UI → Databases → New → PostgreSQL:
    - Name: `spectra-db`
    - Set a strong password

    After creation, copy the **Internal Connection URL** from the Connection tab:
    ```
    postgresql://spectra:PASSWORD@spectra-db-HASH:5432/spectra
    ```

    **Important:** Change `postgresql://` to `postgresql+asyncpg://` when setting DATABASE_URL in service environment variables. FastAPI's async SQLAlchemy requires the asyncpg driver prefix.

    ## Step 2: Generate SECRET_KEY

    Run once. Use the **same value** for both backend services:
    ```bash
    python3 -c "import secrets; print(secrets.token_hex(32))"
    ```
    Save this value. It must be identical in both spectra-public-backend and spectra-admin-backend.

    ## Step 3: Install Tailscale on Dokploy Host

    SSH into the Dokploy host server:

    ```bash
    # Install Tailscale
    curl -fsSL https://tailscale.com/install.sh | sh

    # Find the Docker internal network subnet
    docker network inspect dokploy-network | grep Subnet
    # Example: "Subnet": "10.254.0.0/24"

    # Get an auth key from: https://login.tailscale.com/admin/settings/keys
    # Bring up Tailscale with subnet routing
    sudo tailscale up \
      --ssh \
      --advertise-routes=10.254.0.0/24 \
      --auth-key=<TAILSCALE_AUTH_KEY> \
      --hostname=spectra-prod

    # Get and save the server's Tailscale IP (used in admin service env vars)
    tailscale ip -4
    # Example output: 100.64.0.5
    ```

    In the Tailscale admin console (https://login.tailscale.com/admin/machines):
    - Find "spectra-prod" → "..." → "Edit route settings"
    - Enable the advertised subnet route

    On each admin client device:
    ```bash
    tailscale up --accept-routes
    ```

    ## Step 4: Configure VPS Cloud Firewall

    ## Redeployment

    To redeploy any service with updated code:
    1. Push changes to the `main` branch
    2. In Dokploy UI → click the service → click Deploy

    The backend runs `alembic upgrade head` on every startup. New migrations are applied; already-applied migrations are skipped automatically.

    ## Troubleshooting

    **Let's Encrypt certificate fails:**
    DNS not yet propagated. Delete the domain entry in Dokploy and re-add after `dig` confirms propagation.

    **Backend crashes at startup with SQLAlchemy driver error:**
    `DATABASE_URL` uses `postgresql://` prefix. Change to `postgresql+asyncpg://`.

    **Admin frontend CORS errors in browser:**
    `CORS_ORIGINS` and `ADMIN_CORS_ORIGIN` in the admin backend must include the Tailscale IP and port 3001.
    Update both env vars and redeploy the admin backend.

    **Tailscale clients can't reach Docker containers (only the server Tailscale IP):**
    Run `tailscale up --accept-routes` on the client device. Also verify the subnet route is approved in the Tailscale admin console (https://login.tailscale.com/admin/machines).

    **Uploads disappear after redeploy:**
    The volume mount was not configured before the first deploy. Configure the volume in Advanced → Mounts and redeploy. Data from the initial deployment (in the container's writable layer) cannot be automatically recovered.

    **Admin ports reachable from public internet:**
    UFW is not sufficient — Docker bypasses it. Configure your VPS provider's cloud firewall to block ports 8001 and 3001 from `0.0.0.0/0`.
