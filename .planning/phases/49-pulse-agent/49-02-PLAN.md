       ```python
       class PulseAgentState(TypedDict):
           collection_id: str
           user_id: str
           pulse_run_id: str
           user_context: str           # Optional user guidance
           file_data: list[dict]       # [{file_id, filename, file_bytes, file_type, data_summary, deep_profile}]
           file_profiles: list[dict]   # Deep profile JSON per file (from profiling node)
           signal_candidates: list[dict]  # Hypotheses from Pulse Agent brain

           report_content: str            # Generated markdown report
           error: str                     # Error message if failed
       ```

    2. **`async def profile_data_node(state: PulseAgentState) -> dict`**:
       - Import and call `profile_files_in_sandbox(state["file_data"], get_settings())`
       - Return `{"file_profiles": profiles}`
       - On error, return `{"error": str(e), "file_profiles": []}`

    3. **`async def run_analyses_node(state: PulseAgentState) -> dict`**:
       - If `state["error"]`, short-circuit return
       - **IMPORTANT: Stateless pipeline.** Unlike the chat module, the Pulse Agent does NOT maintain message history between runs. Each `ainvoke()` starts with a fresh `PulseAgentState` — no `messages[]` list, no prior run context. Do NOT copy the chat agent's stateful conversation pattern. Every LLM call is a one-shot request with the current run's data only.
       - **Step A: Pulse Agent brain** -- Use `get_llm()` with `get_agent_provider("pulse_agent")` / `get_agent_model("pulse_agent")` to call the Pulse Agent LLM. Pass file_profiles + data_summaries + user_context as a structured HumanMessage. Ask it to return JSON array of signal candidates (title, severity_hint, category, code_instructions, chart_hint).
       - Parse the LLM response JSON into `signal_candidates` list
       - **Step B: Fan-out validation** -- For each candidate, dispatch `_validate_single_candidate(candidate, file_data, settings)` via `asyncio.gather(*tasks, return_exceptions=True)`
       - `_validate_single_candidate`: calls `coding_agent()` with Pulse-specific instructions from `get_agent_prompt("pulse_agent")` context + candidate's `code_instructions`, then `validate_code()`, then executes in `E2BSandboxRuntime(timeout_seconds=settings.pulse_sandbox_timeout_seconds)`. Returns validated result dict or None on failure.
       - Filter out exceptions and None results
       - Return `{"signal_candidates": candidates, "validated_signals": valid_results}`
       - If 0 valid results, return `{"error": "No signals could be validated from the analysis"}`

    4. **`async def generate_signals_node(state: PulseAgentState) -> dict`**:
       - If `state["error"]`, short-circuit return
       - For each validated signal: construct `SignalOutput` dict with severity/chartType normalized to lowercase, build `SignalEvidence` from execution results, include analysis_text
       - Validate all signals through `PulseAgentOutput(signals=signal_list)` -- reject any that fail validation (log warning, continue with valid ones)
       - If after validation 0 signals remain, set error
       - **Generate report**: Call the Pulse Agent LLM again with all validated signals to produce a markdown detection summary report (executive summary, signal-by-signal breakdown, methodology notes)
       - Return `{"signals_output": [s.model_dump() for s in validated], "report_content": report_markdown}`

    5. **`def build_pulse_graph() -> CompiledGraph`**:
       ```python
       graph = StateGraph(PulseAgentState)
       graph.add_node("profile_data", profile_data_node)
       graph.add_node("run_analyses", run_analyses_node)
       graph.add_node("generate_signals", generate_signals_node)
       graph.set_entry_point("profile_data")
       graph.add_edge("profile_data", "run_analyses")
       graph.add_edge("run_analyses", "generate_signals")
       graph.set_finish_point("generate_signals")
       return graph.compile()
       ```

       - Atomic deduction: `result = await CreditService.deduct_credit(db, user_id, cost)` -- if `not result.success`, raise `HTTPException(status_code=402, detail=result.error_message)`

       - Return pulse_run

    3. **`@staticmethod async def _run_pipeline(pulse_run_id, collection_id, user_id, cost, file_ids, user_context)`**:
       - Create fresh DB session: `async with async_session_maker() as db:`
       - `try:` block:
         - Load PulseRun, update `status="profiling"`, `started_at=now()`, commit
         - Load files by file_ids, prepare file_data dicts (read file bytes from disk via `file.file_path`)
         - Invoke `graph = build_pulse_graph()`, `result = await graph.ainvoke(initial_state)`
         - If `result.get("error")`: raise RuntimeError with error message
         - Update `status="analyzing"`, commit
         - **Persist signals**: For each signal in `result["signals_output"]`, create Signal model row (collection_id, pulse_run_id, title, severity, category, analysis=analysis_text, evidence=statistical_evidence dict, chart_data, chart_type=chartType)
         - **Persist deep profiles**: For files whose deep_profile was None, update `file.deep_profile = profile_dict` from pipeline result
         - **Generate report**: Create Report row with `report_type="pulse_detection"`, `title="Detection Report - {collection_name}"`, `content=result["report_content"]`, `pulse_run_id=pulse_run_id`, `collection_id=collection_id`
         - Update PulseRun `status="completed"`, `completed_at=now()`
         - `await db.commit()`
       - `except Exception as e:` block:
         - `try:` refund: `await CreditService.refund(db, user_id, cost, reason=f"Pulse detection failed: {str(e)[:200]}")`
         - Update PulseRun `status="failed"`, `error_message=str(e)[:1000]`
         - `await db.commit()`
         - `except Exception as refund_err:` log refund failure
       - Use `logger = logging.getLogger(__name__)` throughout
