The SSE stream headers and body are unchanged — only the URL changes to use the proxy instead of directly hitting the backend.


    Rationale: Same proxy rule applies. `/api/auth/signup-status` → `http://localhost:8000/auth/signup-status`. This is an unauthenticated fetch (no token), so the raw `fetch` call stays as-is (no need to use `apiClient`).
